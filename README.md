# 똑똑한 가계부 V22.8.17 카카오 수정 플로우 V4 + 복구 핫픽스

V22.8.15의 화면·테마·보안·영수증·분석 보호 기준을 유지하면서, 카카오 챗봇의 수정(edit) 플로우를 검증된 V4 파서·상태머신으로 전면 교체한 버전입니다. 실사용에서 재현된 "같은 메뉴가 반복 출력되는" 무한루프를 구조적으로 차단합니다. V22.8.17은 V22.8.16 운영 확인에서 발견된 `복구 NN번` 실패(복구 버퍼의 `household_id` 누락으로 재삽입이 항상 거부)를 고친 핫픽스를 포함합니다.

부분 패치가 아닙니다. 운영에 적용할 때는 검증된 `src/index.js`를 기존 Cloudflare Worker 코드와 전체 교체합니다.

## 이번 버전 핵심

- 재현 버그 수정: `수정 03번` → `지출자 변경`/`지출자` 입력에 같은 메뉴가 재출력되던 경로 제거
- 라우팅 순서 재구성: 수정 세션 중 메시지는 다른 어떤 핸들러(기록 저장 포함)로도 가지 않음
- 무한루프 원천 차단: 실패 문구 회전, 실패 3회 종료, 총 8턴 상한, 전송 직전 동일응답 가드
- 선택형 값은 전부 숫자: 지출자=구성원 목록, 결제수단=등록 카드, 분류=카테고리, 날짜=오늘/어제/그제
- 한 줄 지름길 상시 동작: `수정 03번 지출자 엄마`, 세션 중 `지출자 엄마`
- 오타 허용(편집거리 1)과 값 자동 추론(`엄마`→지출자, `13000`→금액, `어제`→날짜)
- `취소`·`그만`·`됐어` 상시 탈출, `도움말`·`?` 상시 안내, 세션 TTL 5분
- `삭제 NN번` 즉시 삭제 + `복구 NN번` 24시간 되돌리기 (V22.8.17: 복구 재삽입 실패 수정, `restore_error` 텔레메트리 추가)
- 세션 키 = 방+사용자 조합: 같은 방 가족 동시 수정 간섭 없음
- 봇 문구의 `결제자`를 `지출자`로 통일, 봇 안내 예시 전부 selfTest로 고정
- 미인식 입력 텔레메트리(`unrecognized_inputs`, 선택 SQL) 적재

## 수정 플로우 사용법

```
저장 응답 하단:
수정: "수정 03번" 또는 "수정 03번 금액 13000"
삭제: "삭제 03번"

수정 메뉴:
03번 - 저녁 마라탕 / 30,000원 / 현대카드 / 식비
무엇을 바꿀까요? 번호를 보내주세요.
1. 금액  2. 분류  3. 결제수단  4. 내용  5. 날짜  6. 지출자  7. 삭제
한 줄로도 돼요: 지출자 엄마 / 금액 13000
(그만하려면: 취소)
```

## 성능 기준

| 항목 | 기준 |
|---|---:|
| `/my` 데이터 요청 | 4회 이하 |
| `/app` 데이터 요청 | 9회 이하 |
| 개인 홈 HTML | 35KB 미만 |
| 정적 자원 DB 요청 | 0회 |

정적 자원 경로와 캐시 정책은 V22.8.15와 동일합니다. 개인 데이터 HTML은 계속 `no-store`입니다.

## 적용 후 확인

1. `/health` 버전이 `V22.8.17-KAKAO-EDIT-RESTORE-FIX`인지 확인합니다.
2. 챗봇에서 기록 저장 후 `수정 01번` → `지출자 변경` → 번호 선택이 메뉴 반복 없이 완료되는지 확인합니다.
3. `수정 01번 금액 13000` 한 줄 수정과 `삭제 01번` → `복구 01번`을 확인합니다.
4. 세션 중 `취소` 즉시 종료와, 이해 못 할 입력 3회 후 종료 안내를 확인합니다.
5. 그룹챗에서 두 사용자가 동시에 서로 다른 번호를 수정해 간섭이 없는지 확인합니다.
6. 기록 저장, 요약, 예산, 정산, 영수증 OCR, 웹 화면이 기존과 동일한지 확인합니다.

## 롤백

필수 DB 변경이 없으므로 문제가 생기면 직전 버전 `src/index.js`로 Worker 코드만 전체 롤백합니다. 단 V22.8.16에는 복구(undo) 실패 버그가 있으므로 복구 기능이 필요하면 V22.8.17 이상을 유지합니다. 선택 생성한 `unrecognized_inputs` 테이블은 남아 있어도 무해합니다.
