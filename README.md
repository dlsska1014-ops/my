## V22.8.5 모바일 접근·메뉴 위계

- 모바일 로그인 첫 화면에 기존 계정 로그인과 실제 복구 경로를 우선 배치했습니다. 새 계정·초대코드는 필요할 때만 펼칩니다.
- 카카오 로그인이 비활성 또는 설정 불완전일 때 오류 버튼을 숨기고 `PC 전체 메뉴 → 개인 비밀번호` 경로를 바로 안내합니다.
- 전체 메뉴의 이중 카드 테두리를 제거하고, 매일 쓰는 4개 기능·보조 목록·개인 설정으로 크기와 밀도를 차등화했습니다.
- 사이드바는 아이콘과 활성 상태를 강화하고, 한 번에 한 그룹만 열리는 아코디언으로 바꿨습니다.
- 320·360·390·768·1365·1860px에서 가로 넘침 없이 확인했으며 로그인 포함 자동 명암비 실패는 0건입니다.
- 사용자가 만족한 분석 본문은 V22.8.4 렌더 결과와 SHA-256이 완전히 동일하도록 고정 검증합니다.
- 홈 조회 병렬화와 OCR·엑셀 지연 로딩은 그대로 유지되며, 동일 모의 조건에서 V22.8.3 대비 약 60% 빠른 상태입니다.
- 데이터베이스 마이그레이션은 없습니다. V22.8.0 스키마를 그대로 사용합니다.

운영 적용 전에는 `FINAL_APPLY_CHECKLIST_V22_8_5.md`와 `V22_8_5_MOBILE_ACCESS_MENU_HIERARCHY_REPORT.md`를 확인하세요.

## V22.8.2 인증·UI 안정화

- 카카오 로그인은 `KAKAO_LOGIN_ENABLED=1`, REST API 키, 공개 기준 주소, 고정 Redirect URI가 모두 맞을 때만 표시합니다.
- OAuth Redirect URI를 현재 접속 호스트에서 추정하지 않습니다. 미리보기·workers.dev·www/non-www 주소가 섞여 KOE006 화면으로 이동하는 경로를 차단했습니다.
- OAuth state와 계정 연결 임시 쿠키를 성공·취소·실패 경로에서 모두 정리하고, 설정 오류 시 다른 로그인 방법을 유지합니다.
- 카카오 로그인 점검 화면이 공개 주소·Redirect URI 호스트 불일치와 Client Secret 필수 여부를 구분해 표시합니다. 카카오 관리자센터의 실제 등록 여부는 외부 확인 항목으로 명시합니다.
- 확인창에서 취소한 POST 버튼이 `저장 중…`으로 잠기는 문제를 수정하고, 삭제·방출·탈퇴·계정 통합 버튼을 위험 작업으로 일관되게 표시합니다.
- 통합 내비게이션과 전체 메뉴의 백업 경로를 `/my/backup`으로 맞추고 영수증·자동 리포트 메뉴 누락을 보완했습니다.
- 첫 사용 필수 흐름을 `가계부 준비 → 첫 기록 → 저장 결과 확인`으로 통일했습니다. 예산·자산·정기지출은 선택 설정으로 유지합니다.
- 데이터베이스 마이그레이션은 없습니다. V22.8.0 스키마를 그대로 사용합니다.

운영 적용 전에는 `FINAL_APPLY_CHECKLIST_V22_8_2.md`와 `STABILITY_REPORT_V22_8_2.md`를 확인하세요. 카카오 설정을 바꾸기 전에는 `KAKAO_LOGIN_ENABLED=0`으로 유지하는 것이 안전합니다.

# V22.7 인증·원자성·전체 안정화 변경 이력

## V22.7 핵심

- 보안 설정과 개인 비밀번호 인증 뒤 사용자가 보던 정확한 내부 경로·쿼리·앵커로 복귀
- 관리자·개인 비밀번호를 PBKDF2-SHA256 기반으로 저장하고 세션 버전 갱신·로그인 시도 제한 적용
- 존재하지 않는 로그인 이름이나 틀린 비밀번호가 새 계정을 만들지 않도록 인증과 가입을 분리
- 가계부 삭제, 예산 일괄 저장, 정기거래 반영, 거래 수정·삭제·일괄처리, 가져오기, 계정 병합을 DB 트랜잭션 RPC로 보호
- 가계부 삭제 시 거래뿐 아니라 정산·예산·리포트·멤버 별칭·카카오 임시 상태까지 해당 가계부 범위에서 정리
- 기록 목록과 수정 화면에 수입자·지출자를 표시하고 현재 가계부 참여자만 선택 가능하도록 재검증
- 백업 CSV에 지출자 표시명을 포함하되 사용자 UUID는 제외하고, 가져오기 때 이름이 정확히 한 명과 일치할 때만 연결
- CSV·TSV·TXT·JSON·XLS/XLSX, 여러 시트, 제목 유사어, 설명행·반복 제목·자연어 행을 미리보기 후 선택 저장
- 완료된 영수증·정기거래·정산·리포트·분석·예산·백업 기능을 무료 공개하고 결제·사용량 제한 제거
- 미완성 소비 카드/밈과 카드 실적/혜택은 메뉴·공유·직접 URL에서 모두 비노출
- 일반 화면에서 내부 상태 코드·고유 사용자 ID·원시 서버 오류를 숨기고 한국어 해결 안내 제공
- 모바일 하단 메뉴를 `홈 / 기록 / 입력 / 정산 / 메뉴`로 통일하고 인증 화면의 사업자 푸터 겹침 제거
