# 똑똑한 가계부 V22.8.18 웹앱 제출 위치 인라인 결과

V22.8.17까지의 카카오 수정·복구 로직과 화면·테마·보안·영수증·분석 보호 기준을 유지하면서, 웹앱-속도피드백-개선지침서의 (B) 결과 피드백과 (B-0) 응답 규약을 반영한 버전입니다. 저장·수정·삭제 결과가 화면 상단이 아니라 **사용자가 제출한 그 자리**에 보이게 하고, 실패 사유가 화면 밖이면 그 자리로 스크롤합니다. 기존 기능 삭제 없이 점진 개선입니다.

부분 패치가 아닙니다. 운영에 적용할 때는 검증된 `src/index.js`를 기존 Cloudflare Worker 코드와 전체 교체합니다.

## 이번 버전 핵심

- (B) 제출 위치 인라인 결과: redirect 뒤 돌아온 상단 결과 배너(`.ok`/`.error`/`.notice`)를 제출한 폼의 버튼 바로 아래로 이동. 실패가 화면 밖이면 그 자리로 스크롤
- 같은 `action` 폼이 여러 개면 이동하지 않고 상단 유지(오배치 방지), JavaScript·`sessionStorage` 미지원 시 상단 표시로 폴백
- 서버가 `escapeHtml`로 렌더한 노드를 그대로 이동 — 새 마크업 주입 없음(`innerHTML` 미사용, XSS 안전)
- `prefers-reduced-motion` 존중, 성공은 3.5초 후 옅게 정리·실패는 유지, 셸 CSS를 마지막 캐스케이드로 보존(다크모드 보호)
- (B-0) 응답 규약: `message`가 없던 실패 JSON 13곳에 사용자용 한국어 `message` + 분기용 `reason` 추가
- (A) 속도·(C-1) 로딩: 정적 자산 immutable 분리, 병렬 조회(`Promise.all`), 목록 페이지네이션, 제출 버튼 잠금·스피너·`aria-busy`는 기존 구현을 확인(추가 변경 없음)

## 이전 버전 유지 기능 (V22.8.16~17 카카오 수정 플로우 V4)

- `수정 NN번` → `지출자 변경`/번호 선택이 메뉴 반복 없이 완료(무한루프 구조적 차단)
- 실패 문구 회전·3회 종료·총 8턴 상한·전송 직전 동일응답 가드, 세션 TTL 5분
- `삭제 NN번` 즉시 삭제 + `복구 NN번` 24시간 되돌리기(복구 재삽입 실패 핫픽스 포함)
- 봇 문구 `결제자`→`지출자` 통일, 미인식 입력 텔레메트리(`unrecognized_inputs`, 선택 SQL)

## 적용 후 확인

1. `/health` 버전이 `V22.8.18-WEBAPP-INLINE-FEEDBACK`인지 확인합니다.
2. 웹앱에서 기록을 저장하면 초록 결과가 저장 버튼 자리 근처에 보이는지 확인합니다.
3. 금액을 비운 채 저장하면 빨강 사유가 그 자리에 보이고, 화면 밖이면 스크롤되어 보이는지 확인합니다.
4. 챗봇 `수정 01번` → `지출자 변경` → 번호 선택, `삭제 01번` → `복구 01번`이 정상인지 확인합니다.
5. JavaScript를 끈 상태에서도 결과가 화면 상단에 그대로 표시되어 기능 손실이 없는지 확인합니다.
6. 기록 저장, 요약, 예산, 정산, 영수증 OCR, 카카오 1:1·그룹이 기존과 동일한지 확인합니다.

## 롤백

DB 변경이 없으므로 문제가 생기면 직전 V22.8.17 `src/index.js`로 Worker 코드만 전체 롤백합니다. 클라이언트 개선은 인라인 주입이므로 코드 롤백만으로 완전히 되돌아갑니다.
