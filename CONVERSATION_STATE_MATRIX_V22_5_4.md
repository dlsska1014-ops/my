# V22.5.4 대화 상태·권한 검증 매트릭스

## 공통 처리 원칙

1. 현재 활성화된 단계형 흐름의 응답을 먼저 판정합니다.
2. 흐름과 무관한 명시적 상위 명령은 기존 흐름을 안전하게 종료하고 새 명령을 실행합니다.
3. 자유 자연어 NLU는 단계 응답과 명시적 명령 판정 뒤에만 사용합니다.
4. 직접채팅과 각 단톡방은 서로 다른 상태 키를 사용합니다.
5. 저장·수정·삭제 응답에는 처리한 가계부 이름을 표시합니다.

| 영역 | 위험 사례 | V22.5.4 처리 | 자동 검증 |
|---|---|---|---|
| 신규 생성 | `5번`이 이름으로 저장 | 직접입력 종류 선택으로 처리 | 숫자·서수 20개 변형 |
| 신규 생성 | `응`, `만들어줘`가 이름으로 저장 | 확인 단계에서만 생성 승인 | 확인 표현 14개 |
| 신규 생성 | `아니`가 취소 또는 새 이름으로 오인 | 이름 재입력 단계로 복귀 | 부정 표현 7개 |
| 신규 생성 | URL·이메일·금액·초대코드·명령어가 이름으로 저장 | 예약·위험 입력 공통 차단 | 위험 입력 15개 이상 |
| 한국어 이름 | `테스트 가계부`가 기호로 오인 | Unicode 문자·숫자 기준 검증 | 한국어 이름 생성 |
| 중복 생성 | 같은 사용자가 같은 이름 재생성 | 기존 가계부 안내, 신규 생성 차단 | 정확 일치 중복 |
| 상태 손상 | 잘못된 flow/step JSON | 즉시 초기화 | 손상 상태 주입 |
| 상태 만료 | 오래된 생성·예산 상태가 재사용 | 만료 시 자동 초기화 | 과거 expires_at |
| 명령 중단 | 생성 중 `남은 예산`, `도움말`이 이름이 됨 | 상위 명령으로 전환 | 흐름 중단 명령 |
| 예산 | `100만원`이 거래로 저장 | 예산 금액 단계가 우선 소유 | 금액·확인·부정 흐름 |
| 직접채팅 | 선택하지 않은 첫 가계부를 자동 사용 | 명시적 선택 요구 | 0·1·다중 가계부 |
| 미연결 단톡방 | 개인 가계부로 저장 | 모든 쓰기 차단 | 미연결 그룹 거래 0건 |
| 단톡방 연결 | 일반 구성원이 연결 | target owner/admin만 허용 | member·pending 거부 |
| 단톡방 재연결 | 다른 가계부 owner가 방을 가로챔 | 현재와 새 가계부 양쪽 owner/admin 필요 | 방 탈취 시나리오 |
| 단톡방 재연결 | 바로 덮어씀 | 이전·변경 대상 표시 후 확인 | 확인·취소 |
| 동시 연결 | 서로 다른 방 연결이 JSON map에서 유실 | 방별 독립 설정을 권위 데이터로 사용 | 병렬 두 방 연결 |
| 거래 반복 | 같은 문장을 연속 저장 | 대화 범위별 메모리 방어 + DB 최근 중복검사 | 동일·다른 방 반복 |

| 대표 명령어 | 표시와 전송 문구 불일치 | 10개 모두 label=command=messageText | JSON 계약 검사 |
