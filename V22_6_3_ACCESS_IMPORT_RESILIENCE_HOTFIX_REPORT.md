## 1. 이탈 방지 안내

다음 상태에서 단순 오류 코드 대신 사용자가 할 수 있는 행동을 표시한다.

- `pending`: 참여 요청은 접수되었으며 반복 입력하지 말고 관리자 승인을 요청하도록 안내
- `blocked`: 초대코드 재입력으로 해제되지 않으며 관리자에게 제한 해제를 요청하도록 안내
- `viewer`: 조회 전용임을 표시하고 쓰기·가져오기·설정 변경을 제한
- 생성 실패: 중간 데이터 정리 여부와 한 번만 재시도할 것을 안내
- 참여 실패: 권한이 자동으로 열리지 않았음을 명시
- 가져오기 실패: 행 번호, 실패 이유, 수정 방법, 원문 미리보기를 제공
- 빈 파일·대용량·엑셀 변환 실패·파일 읽기 실패: 원본이 바뀌지 않았음을 알리고 대체 경로 제시

시작 순서는 `가계부 생성·참여 → 초대·승인 → 단톡방 연결(선택) → 기록 방법 → 첫 기록`으로 통일했다. 예산·분류·결제수단·정기지출은 첫 기록 이후 선택 사항이다.

## 2. 생성·관리 안정성

### 반영 사항

- 가계부 레코드 생성 뒤 owner 멤버십 생성에 실패하면 새 가계부를 즉시 보상 삭제한다.
- 같은 Worker 인스턴스에서 같은 사용자·같은 이름으로 들어온 동시 생성 요청을 직렬화한다.
- 사용자가 이미 같은 이름의 가계부에 참여 중이면 새로 만들지 않고 기존 가계부를 선택한다.
- 생성·참여 폼은 첫 제출 후 버튼을 비활성화하고 진행 상태를 표시한다.
- 조회 화면이나 거래 저장 과정에서는 가계부를 묵시적으로 만들지 않는다.
- pending·blocked 가계부를 기본 선택하거나 데이터 화면에 전달하지 않는다.

### 남은 경계

서로 다른 Cloudflare 인스턴스에서 완전히 동시에 같은 이름을 생성하는 경우를 DB 고유 제약 없이 100% 막을 수는 없다. 이번 버전은 스키마를 바꾸지 않는 핫픽스이므로 인스턴스 내부 잠금과 기존 가계부 재조회로 위험을 낮췄다. 정식 버전에서는 사용자별 생성 요청 키 또는 서버 트랜잭션/RPC를 권장한다.

## 3. 한 ID·여러 가계부·여러 단톡방

- 한 사용자는 owner/admin/member/viewer 역할로 여러 가계부에 참여할 수 있다.
- 직접채팅은 사용자가 명시적으로 선택한 가계부만 사용한다.
- 단톡방은 방 식별키별로 연결된 가계부만 사용하며 개인 선택 가계부로 자동 대체하지 않는다.
- 서로 다른 두 단톡방을 동시에 연결해도 각 방의 설정이 덮어써지지 않는 기존 회귀시험을 유지했다.
- 단톡방 재연결은 현재 가계부와 새 가계부 양쪽의 owner/admin 권한 및 명시적 확인을 요구한다.
- pending·blocked는 연결된 방에서도 조회·저장이 차단되고 viewer는 조회 명령만 허용된다.

## 4. 참여 흐름

- 초대코드가 없거나 잘못되면 최신 코드를 다시 요청하도록 안내한다.
- 새 참여는 `pending`으로만 저장한다.
- pending 저장이 실패했을 때 기존처럼 member로 우회하지 않는다.
- 재전송 또는 동시 요청으로 역할이 이미 생겼다면 한 번 다시 읽어 실제 역할만 반환한다.
- 역할이 없으면 `approval_request_failed`로 안전하게 중단한다.
- 관리자 승인 전에는 거래·예산·분석·초대코드가 노출되지 않는다.

## 5. 다른 가계부 자료 가져오기

### 지원 범위

- 파일: CSV, TSV, TXT, JSON, XLS, XLSX
- 구분자: 쉼표, 탭, 세미콜론, 파이프
- 표: 제목 위 설명행, 반복 제목, 여러 엑셀 시트, 제목 없는 표
- 문장: 키-값 형식과 자연어 한 줄 기록
- 인코딩: UTF-8 BOM, UTF-16 LE/BE, 깨짐이 많은 경우 EUC-KR 후보 비교
- 날짜: ISO, 점·슬래시·한국어 날짜, 날짜·시간, 엑셀 일련번호
- 금액: 작은 정수, 천 단위 쉼표, 원/KRW/won, 괄호·음수, 한국어 단위
- 수입·지출: 구분 열 또는 분리된 입금/출금 열

천 단위 쉼표가 큰따옴표 없이 들어와 열이 하나 늘어난 경우 금액 열을 복원한다. 내용 안 쉼표 때문에 늘어난 경우 내용 열을 합치고 결과에 자동 보정 사실을 표시한다. 안전하게 정렬할 수 없으면 임의 저장하지 않고 제목 수와 값 수 불일치로 분리한다.

### 실패 설명

다음 사유를 행별로 구분한다.

- 날짜 없음 또는 날짜 형식 오류
- 금액 없음 또는 금액 형식 오류
- 금액 후보가 여러 개라 확정 불가
- 한 행에 수입·지출 금액이 동시에 존재
- 제목 수와 값 수 불일치
- 합계·잔액·설명·반복 제목 행
- 기존 기록과 중복
- 처리 한도 초과
- 저장 중 오류

결과 화면은 원본 파일을 수정하지 않으며, 저장하지 못한 행만 고쳐 재시도할 수 있도록 안내한다.

## 6. UI·UX 추가 판단

이번 버전에서 개선한 항목:

- 역할 제한을 빈 화면이나 일반 오류가 아닌 전용 상태 화면으로 표시
- 초대코드는 owner/admin에게만 표시
- viewer에게 편집 가능한 것처럼 보이던 설정 진입을 제한
- 생성·참여 중복 클릭 방지와 진행 문구
- 가져오기 화면에 제목 유사어, 자연어 예시, 원본 보존 원칙 표시
- 결과 화면에 성공·중복·설명/합계 제외·미인식·저장 오류·한도 초과를 분리

다음 정식 버전 권장 항목:

- 1.6MB 단일 Worker 파일의 화면·권한·가져오기 모듈 분리
- 문자열 치환 중심 UI 정규화를 공통 컴포넌트로 전환
- 첫 기록 완료율, 참여 승인 대기시간, 가져오기 성공률·사유별 실패율 계측
- 실제 iPhone Safari와 Android Chrome에서 키보드·안전영역·파일 선택 QA
- 긴 가계부명, 긴 상호명, 5,000행 자료의 모바일 결과 화면 QA

## 7. 프리미엄 준비 상태

현재 스마트 분석 무료 베타에는 사용할 수 있다. 프리미엄 데이터가 다른 가계부로 섞이지 않도록 역할·가계부 범위를 강화했고, 예상값은 근사치와 기준을 표시하며 결제 기능은 노출하지 않는다.

유료 출시 전 필수 작업:

1. 사용자/가계부 단위 entitlement와 플랜 정책
2. 결제 승인·실패·취소·환불·재시도·웹훅 중복 방어
3. 무료/유료 기능 서버 측 강제와 만료 시 읽기 정책
4. 예측·반복지출·이상지출의 실제 정확도 및 설명 가능성 검증
5. 가격·자동갱신·해지·환불·데이터 보관 고지
6. 결제 장애와 권한 불일치 운영 대시보드 및 복구 절차

따라서 판정은 **무료 베타 준비 완료, 유료 판매 준비 미완료**다.
