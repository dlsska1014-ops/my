# V22.8.16 사용자 화면 대비 보고서

## 확인된 원인

기존 화면별 CSS의 입력 선택자가 공통 다크모드 규칙보다 우선해 흰 입력창이 남았고, 전역 글자색 강제가 밝은 의미색 카드와 주요 버튼의 글자를 덮었습니다. 인증 화면 밖의 메뉴·자산·적립계획·영수증·가계부 관리·키워드·백업·시작 안내에도 화면별 표면 규칙이 충분하지 않았습니다.

## 적용

- 실제 CSS 우선순위에 맞춘 다크 입력·선택·텍스트 영역 규칙
- 전역 `*` 글자색 강제 제거와 링크·버튼 규칙 분리
- 18개 사용자 경로의 카드, 보조문구, 빈 상태, 배지, 안내·경고 표면 보정
- 가계부 관리·참여자·키워드·백업·시작 안내 화면 marker 추가
- 새 CSS를 `/assets/accountbook-shell-v22816.css`로 분리해 immutable 캐시 충돌 방지

## 검증 결과

- 18개 사용자 경로 라이트·다크 표시 텍스트 대비 미달 0건
- 예산과 가계부 관리의 입력 계산 스타일이 다크 배경·밝은 글자로 적용됨을 확인
- 주요 버튼 글자와 의미색 카드가 4.5:1 이상 대비를 유지
- 자동 검사 305개와 ESM 진입점 통과
