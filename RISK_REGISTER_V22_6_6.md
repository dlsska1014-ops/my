# V22.6.6 위험 등록부

| 위험 | 영향 | 대응 | 상태 |
|---|---|---|---|
| 브라우저 OCR 모듈 로딩 실패 | 영수증 자동 인식 불가 | 영수증 문자 붙여넣기와 직접 수정 경로 유지 | 완화 |
| OCR 오인식 | 잘못된 금액·날짜 저장 | 저장 전 사용자 확인 필수, 확인 없는 POST 거부 | 완화 |
| 반복 거래 중복 실행 | 같은 지출이 여러 번 저장 | 월 적용 상태와 동일 거래 중복 키 검사 | 완화 |
| 자동 리포트 데이터 혼합 | 다른 가계부 정보 노출 | 가계부 ID별 조회·스냅샷 키, 교차 가계부 테스트 | 완화 |
| 카드 혜택 조건 노후화 | 예상 혜택 오안내 | 예상값 표기, 카드사 약관 확인 안내, 카탈로그 정기 갱신 | 운영 필요 |
| 무료 개방 후 트래픽 증가 | Worker·DB 비용 및 지연 증가 | 기능별 응답시간·호출량 관찰, 장애 시 긴급 중지 변수 | 운영 필요 |
| 사용자 기능과 내부 콘텐츠 도구 혼동 | 운영 화면 외부 노출 | 내부 제작·검수 경로는 기본 비공개 유지 | 완화 |
| 공개 소비 카드의 의도치 않은 공유 | 개인 소비 문구 노출 | 개별 공유 동작이 있어야 공개 URL 생성, 가계부 원본 목록 비공개 | 완화 |
| 리포트 자동 발송 오해 | 원치 않는 메시지 기대·불만 | 자동 생성과 외부 발송을 분리하고 화면에 명시 | 완화 |

## 출시 보류 조건

- 가계부 간 데이터 혼합이 한 건이라도 재현되는 경우
- viewer가 변경 작업을 수행할 수 있는 경우
- 영수증 확인 없이 거래가 저장되는 경우
- Cron 재실행으로 같은 거래나 같은 기간 리포트가 중복 생성되는 경우
- 카드 혜택을 확정 금액처럼 표시하거나 최신 약관 확인 안내가 사라지는 경우
