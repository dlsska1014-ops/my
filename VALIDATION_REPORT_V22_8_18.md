## 추가된 검증 (V22.8.17 대비 +15)

UX·분석 보호 (+14):
- 인라인 결과 런타임 존재와 `sessionStorage` 기반 제출 폼 기억
- 오프스크린 결과의 `scrollIntoView` 동작
- `prefers-reduced-motion` 존중
- 같은 action 폼이 모호할 때 배너를 이동하지 않음(`count === 1 ? hit : null`)
- 런타임이 `innerHTML`을 절대 대입하지 않음(XSS 안전, 노드 이동만)
- 모바일 홈이 인라인 결과 런타임·스타일을 포함하고 35KB 예산 유지
- 인라인 스타일이 셸 스타일시트 캐스케이드를 넘지 않음(다크모드 보호)
- B-0 응답 규약: 실패 JSON의 한국어 `message`(권한·인증·미발견 등) 포함, 코드 전용 실패 제거

영수증 (+1):
- 영수증 페이지에 추가된 인라인 런타임이 브라우저 문법으로 정상 컴파일

## 지침서 대비 반영 범위

- (A) 속도: 정적 자산 immutable 분리·병렬 조회(`Promise.all`)·목록 페이지네이션 기존 구현 확인
- (B) 결과 피드백: 제출 위치 인라인 결과 신규 구현
- (B-0) 응답 규약: 실패 JSON `message`+`reason` 신규 반영
- (C-1) 로딩 표시: 제출 버튼 잠금·스피너·`aria-busy` 기존 구현 확인

## 수동 확인 필요

- 운영 배포 후 저장·검증 실패 시 인라인 결과가 제출 자리에 보이는지 실기기 확인
- JavaScript 비활성 환경에서 상단 표시 폴백 확인
- 느린 회선에서 버튼 잠금·스피너 체감 확인

운영 배포와 실제 사용자 세션·실기기 검증은 자동 통과로 간주하지 않습니다.
