# V22.1 비용 통제

## 기본 상태
- 외부 LLM/Workers AI 호출 없음
- Event API 미사용
- 원문 발화 영구 저장 기본 비활성
- 성공 로그 영구 샘플링 0%
- Supabase 스키마 변경 필수 없음

## 선택 기능
`schema_v22_1_nlu_metrics_optional.sql`은 시간대별 의도·결과·지연 합계만 저장하며, 원문 발화와 사용자 키는 저장하지 않습니다. 초기 배포에는 적용하지 않아도 됩니다.

## 비용 증가 조건
- 선택 집계 테이블을 활성화하고 장기 보관할 때
- 성공 로그 샘플링을 올릴 때
- 실제 사용량 증가로 Workers 요청·Supabase DB 부하가 늘 때

자연어 의도 엔진 자체는 Worker 내부 문자열 처리이므로 별도 AI 사용료가 발생하지 않습니다.
