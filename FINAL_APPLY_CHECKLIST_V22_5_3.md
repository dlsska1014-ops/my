# V22.5.3 최종 적용 체크리스트

## 배포

1. 현재 Worker를 백업합니다.
2. 이 패키지의 `src/index.js`를 배포합니다.
3. `https://ttokttok-accountbook.com/health`에서 아래 버전을 확인합니다.

```text
V22.5.4-CONVERSATION-STATE-INTEGRITY-HARDENING
```

OpenBuilder와 Supabase SQL은 변경하지 않습니다.

## 생성 흐름 확인

```text
새 가계부 만들기
5번
테스트 가계부
응
```

확인 기준:

- `5번`이 이름으로 표시되지 않음
- `테스트 가계부` 생성 전 확인 문구가 표시됨
- `응`을 보내면 `테스트 가계부`로 생성됨
- `응`, `만들어줘`, `5번`이라는 이름의 가계부가 생성되지 않음

추가 확인:

```text
새 가계부 만들기
5번
두번째 테스트
만들어줘
```

`두번째 테스트`로 생성되어야 합니다.

## 지출자 변경 확인

```text
점심 180000원 현금
지출자 변경
엄마
```

확인 기준:

- 일반 기록 예시가 아니라 구성원 목록이 표시됨
- 변경 완료 응답에 가계부 이름과 `지출자: 엄마`가 표시됨
- 요약·정산에서 엄마 지출로 집계됨

직접 문법도 확인합니다.

```text
01번 지출자 엄마
방금 지출자 엄마로 변경
```

## 회귀 확인

- 연결되지 않은 단톡방에서 거래가 저장되지 않음
- 직접 채팅에서 선택하지 않은 가계부가 임의 사용되지 않음
- 초대코드는 선택 가계부 기준으로만 표시됨
- 거래 저장 응답에 가계부 이름 표시
- `/시작`, `/기록`, `/남은예산`, `/요약` 기존 명령 정상
- 공개 홈페이지와 `/sitemap.xml` 정상
