# 변경 이력

최신 ZIP 하나만으로도 이전 작업의 흐름과 보호해야 할 기준을 확인할 수 있도록 유지합니다.

## V22.8.10 · 2026-07-20

### 홈 데이터 조회 병목 제거

- 사용자 가계부 목록의 가계부별 순차 조회를 `id=in.(...)` 묶음 조회로 변경
- 가계부 참여자 프로필의 사용자별 조회를 묶음 조회로 변경
- 선택한 역할을 다시 읽던 중복 membership 조회 제거
- 홈의 예산 fallback, 표시 이름, 정기지출 준비, 결제수단 설정을 `accountbook_settings` 한 번으로 통합
- 예산 설정 fallback과 예산 테이블을 병렬 조회하도록 변경
- 실제 반환 행이 페이지 크기보다 적을 때 빈 다음 페이지를 다시 조회하지 않도록 종료 조건 보강
- `/my`가 선택 가계부로 바로 이동하는 일반 경로에서 사용하지 않는 사용자 프로필 재조회 제거

### 홈 응답과 브라우저 캐시 최적화

- 개인 홈에 반복 삽입되던 공통 CSS·JavaScript를 `/assets/mobile-home-v22810.css`, `/assets/mobile-home-v22810.js`로 분리
- 정적 자원에 `public, max-age=31536000, immutable` 적용
- 개인 홈 HTML을 106,841 bytes에서 24,490 bytes로 축소
- 홈 데이터 요청을 16회에서 9회, `/my` 진입 요청을 7회에서 4회로 축소
- 공통 자원에는 DB를 전혀 조회하지 않으며 HTML 본문은 기존처럼 `no-store` 유지

### 경량 누적 ZIP과 검증

- 374개 항목이던 과거 누적 ZIP에서 오래된 개별 로그·중복 체크섬·원본 학습 ZIP·이미지·빌드 폴더 제외
- 버전 이력은 `CHANGELOG.md`, `HISTORY_INDEX.md`, SQL 판정은 `SQL_HISTORY.md`로 통합
- 카카오 원본 학습 파일 대신 검증된 프로젝트 메모와 적용 체크리스트만 포함
- 홈 요청 수·묶음 조회·응답 크기·정적 캐시를 확인하는 32개 성능 검증 추가
- 기존 171개 회귀 검증과 합쳐 총 203개 자동 확인
- 분석 클라이언트·렌더러 해시와 영수증 구현은 변경하지 않음
- 신규 SQL·환경변수·Kakao Developers·OpenBuilder 변경 없음, `src/index.js` 전체 교체만 필요

## V22.8.9 · 2026-07-20

### 계정 보안과 가계부 관리 분리

- 가계부 생성 form에서 로그인 이름·비밀번호·비밀번호 확인 입력을 모두 제거
- 가계부 생성 시 기존 계정 비밀번호 확인 또는 새 로컬 로그인 identity 생성을 하지 않도록 서버 처리 분리
- 가계부에는 비밀번호가 없고, 초대코드는 참여·단톡방 연결에만 쓰인다는 설명을 생성·초대·참여자 화면에 명시
- 가계부별 옵션의 `개인 비밀번호` 항목을 제거하고 전역 `내 계정·보안` 진입점으로 이동
- 비소유자의 가계부 나가기는 현재 로그인 세션과 명시적 이력 보존 확인만 사용하도록 단순화

### 영구 삭제 본인 확인

- 로컬 로그인 사용자는 모든 가계부에 공통인 `내 계정 로그인 비밀번호`로 확인
- 카카오 로그인 사용자는 기존 카카오 OAuth 콜백에서 `prompt=login` 재인증 후 삭제 가능
- 카카오 재인증 결과가 현재 사용자에게 연결된 카카오 ID와 일치하는지 다시 확인
- 재인증 토큰을 사용자·가계부·삭제 목적에 묶고 HttpOnly 쿠키로 5분만 유지하며 삭제 성공 시 즉시 제거
- 가계부 이름 정확히 입력, 전체 기록·참여자 삭제 확인, 소유자 권한, 기존 원자적 purge RPC를 그대로 유지

### 삭제·연결해제 단톡방 안전 안내

- 연결된 가계부가 없거나 삭제된 그룹 입력은 다른 개인 가계부로 fallback하지 않고 저장 0건 유지
- 사용자가 다른 가계부를 가진 경우와 하나도 없는 경우 모두 `방금 입력한 기록은 어디에도 저장하지 않았어요`를 명시
- 이미 삭제된 가계부를 가리키는 오래된 그룹 연결은 다음 요청에서 자동 정리
- 그룹 응답은 V22.8.7 카카오 매뉴얼 기준의 지원 output·번호형 선택지 정책을 계속 유지

### 배포·문서·검증

- 영수증 55개, 카카오 18개, 가계부·계정 보안 43개, UX·보호 화면 55개로 총 171개 자동 확인
- 분석 클라이언트·렌더러 해시와 영수증 구현을 변경하지 않음
- 이번 변경은 `src/index.js` 전체 교체만 필요하며 신규 SQL·환경변수·OpenBuilder 변경 없음
- 기존 카카오 로그인 설정과 동일한 Redirect URI를 재사용하므로 Kakao Developers 콘솔 추가 작업 없음

## V22.8.8 · 2026-07-20

### 예측 가능한 행동 피드백

- 모든 POST 동작을 일괄 `저장 중…`으로 표시하던 공통 런타임을 행동별 진행 문구로 변경
- 로그인·새 계정·가계부 생성·참여·삭제·탈퇴·가져오기·일반 저장을 서로 다른 문구로 안내
- 제출 중 form과 button에 `aria-busy`를 적용하고 화면 읽기 도구용 `aria-live` 상태 추가
- 뒤로 가기 캐시에서 원래 버튼 문구와 제출 상태를 복원하고 비밀번호 유효성도 다시 동기화
- 삭제·탈퇴·계정 통합의 공통 확인창을 실제 결과가 드러나는 문구로 세분화

### 비밀번호 확인과 화면 보호

- 새 계정 만들기와 개인 복구 비밀번호 설정에 8자리·불일치·일치 상태의 즉시 피드백 추가
- 서버의 기존 검증을 유지하고 브라우저 사전 검증을 보강
- 분석 클라이언트와 분석 렌더러를 V22.8.7 소스 해시로 고정해 변경하지 않았음을 자동 검증
- 영수증 UI는 변경하지 않고 기존 55개 회귀 검증으로 보호

### 배포·문서·검증

- Brunch UX 원칙을 프로젝트 판단 기준으로 요약한 `docs/ux/UX_PRINCIPLES_BRUNCH_110_PROJECT_MEMORY.md` 추가
- UX 원칙·진행 문구·비밀번호 피드백·분석 고정 55개 자동 검증 추가
- 전체 자동 확인은 영수증 55개, 카카오 18개, UX 55개로 총 128개
- 이번 변경은 `src/index.js` 전체 교체만 필요하며 신규 SQL·환경변수·OpenBuilder 변경 없음

## V22.8.7 · 2026-07-20

### 카카오 그룹 채팅 응답 매뉴얼 정합성

- `botGroupKey`가 있는 그룹 요청을 중앙 응답 단계에서 판별하도록 추가
- 그룹 챗봇 가이드 v1.11.1에서 미지원인 `quickReplies`를 그룹 응답에서 제거
- 제거된 선택지는 최대 5개의 번호형 입력 문장으로 `SimpleText`에 보존해 단계형 흐름 유지
- 1:1 채팅의 기존 단계형 `quickReplies`는 그대로 유지
- 그룹 응답 output을 SimpleText, SimpleImage, TextCard, BasicCard, ListCard, ItemCard로 제한하고 최대 3개로 정규화
- 알 수 없는 SkillRequest 필드가 포함된 그룹 요청을 허용하는 회귀 테스트 추가
- `botUserKey` → `appUserId` → `plusfriendUserKey` 순서를 자동 검증해 레거시 키가 신규 기본 식별자가 되지 않도록 고정

### 배포·문서·검증

- `DEPLOYMENT_MATRIX.md`, `NEXT_UPDATE_PLAN.md`, V22.8.7 상세 보고서와 검증 보고서 추가
- 빌드 단계에 카카오 그룹 응답 18개 자동 검증을 추가
- 이번 변경은 Worker 코드만 교체하며 신규 SQL·환경변수·OpenBuilder 수정 없음

## V22.8.6 · 2026-07-17

### 영수증 등록 UX

- 앨범 선택과 카메라 촬영 분리 유지
- PC 사진 드래그 앤 드롭과 클립보드 이미지 붙여넣기 추가
- 사진 미리보기·선택 취소·25MB 제한·이미지 형식 검사 유지
- OCR 실행 중 `분석 취소`와 전체 과정 90초 제한 추가
- 인식 금액과 직접 입력 금액에 천 단위 쉼표 표시
- 가계부와 기록 월을 변경하면 즉시 적용하고 기존 `선택 적용` 버튼은 무스크립트 대비용으로 유지
- 상태 안내에 스크린리더용 실시간 알림 적용

### 안정화 보강

- 취소 후 늦게 완료된 OCR 작업자가 새 실행에 섞이거나 메모리에 남지 않도록 실행 세대 가드 추가
- OCR 시간 제한을 이미지 축소 단계부터 적용해 화면이 잠긴 채 남는 경우 차단
- 드래그 진입 깊이를 추적해 자식 요소 위에서 강조 표시가 깜빡이는 현상 방지
- 조회 전용 가계부에서 드래그·붙여넣기로 등록 흐름이 우회되지 않도록 차단
- 숨김 요소가 다른 CSS에 의해 노출되지 않도록 `[hidden]` 우선 규칙 적용
- 영수증 렌더러의 특정 버전명을 제거해 다음 버전에서도 같은 구현을 재사용하도록 정리

### 성능 유지

- Tesseract 5.1.1 지연 로딩
- 기기 성능에 따른 1400px 또는 1800px 사전 축소
- 여러 영수증을 연속 처리할 때 OCR 작업자 재사용, 유휴 시 메모리 해제
- 영수증 화면 전용 최소 데이터 조회와 요청 단위 인증 캐시 유지

### 자동 검증

- 영수증 화면·파서·저장·중복 차단·조회 예산 검증을 빌드 단계에 포함
- 모바일 홈·전체 메뉴·분석·로그인 화면 회귀와 모바일 확대 허용 여부 검증
