# 변경 이력

최신 ZIP 하나만으로도 이전 작업의 흐름과 보호해야 할 기준을 확인할 수 있도록 유지합니다.

## V22.8.8 · 2026-07-20

### 예측 가능한 행동 피드백

- 모든 POST 동작을 일괄 `저장 중…`으로 표시하던 공통 런타임을 행동별 진행 문구로 변경
- 로그인·새 계정·가계부 생성·참여·삭제·탈퇴·가져오기·일반 저장을 서로 다른 문구로 안내
- 제출 중 form과 button에 `aria-busy`를 적용하고 화면 읽기 도구용 `aria-live` 상태 추가
- 뒤로 가기 캐시에서 원래 버튼 문구와 제출 상태를 복원하고 비밀번호 유효성도 다시 동기화
- 삭제·탈퇴·계정 통합의 공통 확인창을 실제 결과가 드러나는 문구로 세분화

### 비밀번호 확인과 화면 보호

- 새 계정 만들기와 개인 복구 비밀번호 설정에 8자리·불일치·일치 상태의 즉시 피드백 추가
- 서버의 기존 검증을 유지하고 브라우저 사전 검증을 보강
- 분석 클라이언트와 분석 렌더러를 V22.8.7 소스 해시로 고정해 변경하지 않았음을 자동 검증
- 영수증 UI는 변경하지 않고 기존 55개 회귀 검증으로 보호

### 배포·문서·검증

- Brunch UX 원칙을 프로젝트 판단 기준으로 요약한 `docs/ux/UX_PRINCIPLES_BRUNCH_110_PROJECT_MEMORY.md` 추가
- UX 원칙·진행 문구·비밀번호 피드백·분석 고정 55개 자동 검증 추가
- 전체 자동 확인은 영수증 55개, 카카오 18개, UX 55개로 총 128개
- 이번 변경은 `src/index.js` 전체 교체만 필요하며 신규 SQL·환경변수·OpenBuilder 변경 없음

## V22.8.7 · 2026-07-20

### 카카오 그룹 채팅 응답 매뉴얼 정합성

- `botGroupKey`가 있는 그룹 요청을 중앙 응답 단계에서 판별하도록 추가
- 그룹 챗봇 가이드 v1.11.1에서 미지원인 `quickReplies`를 그룹 응답에서 제거
- 제거된 선택지는 최대 5개의 번호형 입력 문장으로 `SimpleText`에 보존해 단계형 흐름 유지
- 1:1 채팅의 기존 단계형 `quickReplies`는 그대로 유지
- 그룹 응답 output을 SimpleText, SimpleImage, TextCard, BasicCard, ListCard, ItemCard로 제한하고 최대 3개로 정규화
- 알 수 없는 SkillRequest 필드가 포함된 그룹 요청을 허용하는 회귀 테스트 추가
- `botUserKey` → `appUserId` → `plusfriendUserKey` 순서를 자동 검증해 레거시 키가 신규 기본 식별자가 되지 않도록 고정

### 배포·문서·검증

- 카카오 원본 매뉴얼, 학습 요약, 정본 충돌 정리를 누적 ZIP에 복원
- `DEPLOYMENT_MATRIX.md`, `NEXT_UPDATE_PLAN.md`, V22.8.7 상세 보고서와 검증 보고서 추가
- 빌드 단계에 카카오 그룹 응답 18개 자동 검증을 추가
- 이번 변경은 Worker 코드만 교체하며 신규 SQL·환경변수·OpenBuilder 수정 없음
- 영수증 등록, 모바일 홈, 전체 메뉴, 분석 화면, 로그인 기준은 V22.8.6 그대로 보호

## V22.8.6 · 2026-07-17

### 영수증 등록 UX

- 앨범 선택과 카메라 촬영 분리 유지
- PC 사진 드래그 앤 드롭과 클립보드 이미지 붙여넣기 추가
- 사진 미리보기·선택 취소·25MB 제한·이미지 형식 검사 유지
- OCR 실행 중 `분석 취소`와 전체 과정 90초 제한 추가
- 인식 금액과 직접 입력 금액에 천 단위 쉼표 표시
- 가계부와 기록 월을 변경하면 즉시 적용하고 기존 `선택 적용` 버튼은 무스크립트 대비용으로 유지
- 상태 안내에 스크린리더용 실시간 알림 적용

### 안정화 보강

- 취소 후 늦게 완료된 OCR 작업자가 새 실행에 섞이거나 메모리에 남지 않도록 실행 세대 가드 추가
- OCR 시간 제한을 이미지 축소 단계부터 적용해 화면이 잠긴 채 남는 경우 차단
- 드래그 진입 깊이를 추적해 자식 요소 위에서 강조 표시가 깜빡이는 현상 방지
- 조회 전용 가계부에서 드래그·붙여넣기로 등록 흐름이 우회되지 않도록 차단
- 숨김 요소가 다른 CSS에 의해 노출되지 않도록 `[hidden]` 우선 규칙 적용
- 영수증 렌더러의 특정 버전명을 제거해 다음 버전에서도 같은 구현을 재사용하도록 정리

### 성능 유지

- Tesseract 5.1.1 지연 로딩
- 기기 성능에 따른 1400px 또는 1800px 사전 축소
- 여러 영수증을 연속 처리할 때 OCR 작업자 재사용, 유휴 시 메모리 해제
- 영수증 화면 전용 최소 데이터 조회와 요청 단위 인증 캐시 유지

### 자동 검증

- 영수증 화면·파서·저장·중복 차단·조회 예산 검증을 빌드 단계에 포함
- 모바일 홈·전체 메뉴·분석·로그인 화면 회귀와 모바일 확대 허용 여부 검증
