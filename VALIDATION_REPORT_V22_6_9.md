## V22.6.9 전용 확인

- 로그아웃 상태 `/settings`는 홈 리디렉션 없이 인증 폼을 표시
- 잘못된 관리자 비밀번호도 요청 경로 유지
- 올바른 비밀번호 인증 후 정확한 보안 설정 경로 복귀
- 관리자 비밀번호 변경 후 세션 갱신 및 설정 화면 유지
- 새로 저장한 관리자 비밀번호 인증 성공
- 개인 비밀번호 저장 후 가계부 ID·관리 패널·앵커 유지
- 기록 목록의 `지출자 WIFI♥`, `지출자 Bin`, `수입자 Bin` 표시
- PC 기록 수정 모달의 필수 수입자·지출자 선택 확인
- 소유자 지출자 변경 저장 성공
- 가계부 외 사용자 ID 지정 거부 및 기존 값 보존
- 참여자 이름 변경 완료의 한국어 메시지 표시
- 시작가이드 요청 문구 제거
- 참여자·정산·관리자 가계부 표의 사용자 ID/키 열 제거
- 밈 연구소·밈 보관함·카드 실적 경로 404 유지

## UI·UX 확인

- 사용자가 첨부한 390px 전후 화면에서 문제 지점을 식별해 해당 경로의 DOM·반응형 CSS를 직접 수정했습니다.
- V22.6.8에서 통과한 360×780·390×844, 18개 경로/108개 캡처를 회귀 기준으로 대조했습니다.
- V22.6.9 HTML 회귀에서는 모바일 홈의 `feed`, 빠른 입력, 하단 메뉴, 수정/삭제 영역이 유지되고 지출자 선택에 라벨과 참여자 옵션이 존재함을 검사했습니다.
- 이번 변경은 기록 카드에 한 줄을 추가하므로 긴 닉네임은 기존 카드 폭 안에서 줄바꿈되며 문서 폭을 확장하지 않는 구조입니다.

## 권한·개인정보 확인

- 지출자 변경은 서버가 현재 가계부 참여자를 재조회해 검증합니다.
- 일반 구성원은 자신이 아닌 지출자로 변경할 수 없습니다.
- 화면에 보이는 UUID 열은 제거했습니다.
- 숨은 폼 식별자는 인증된 관리 작업에만 사용되고, 조작해도 서버의 가계부 권한 검사를 통과해야 합니다.
- 개인정보 안내의 식별키 처리 고지는 삭제하지 않았습니다.

## 검증 한계

- 운영 Cloudflare Worker와 실제 Supabase에는 배포·쓰기하지 않았습니다.
- 운영 도메인의 실제 쿠키·카카오 로그인·인앱브라우저 키보드는 배포 후 실기기 확인이 필요합니다.
- 이 환경에서는 새 Chromium 실행 파일을 사용할 수 없어 V22.6.9 신규 캡처를 만들지 못했고, 기존 108개 캡처 기준·첨부 화면 수동 대조·HTML 회귀 검사로 확인했습니다.
