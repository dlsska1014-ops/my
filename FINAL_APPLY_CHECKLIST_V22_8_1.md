# V22.8.1 최종 적용 체크리스트

## 1. 적용 전

- [ ] 현재 Cloudflare Worker 소스와 환경변수 백업
- [ ] 현재 Supabase 데이터 백업
- [ ] 기존 V22.8.0 SQL 적용 상태 확인
- [ ] `PUBLIC_BASE_URL=https://ttokttok-accountbook.com` 확인
- [ ] 카카오 Redirect URI와 OpenBuilder Skill URL 변경 없음 확인

## 2. 배포

- [ ] `src/index.js`를 기존 Worker에 반영
- [ ] Worker 배포 버전이 `V22.8.1-GUIDED-UIUX-FULL-OVERHAUL`인지 확인
- [ ] `/health`가 200을 반환하는지 확인
- [ ] Supabase URL·키·세션 시크릿 환경변수가 기존 값으로 유지되는지 확인

## 3. 역할별 확인

- [ ] 소유자: 자산 추가·수정·삭제
- [ ] 일반 참여자: 자산 조회 가능, 수정 버튼 비노출
- [ ] 소유자/관리자: 정기지출 추가·삭제
- [ ] 일반 참여자: 정기지출 관리 버튼 비노출
- [ ] 소유자/관리자: 예산 저장
- [ ] 다른 가계부로 전환했을 때 자산·예산·기록이 섞이지 않음

## 4. 화면 확인

- [ ] 모바일 320px에서 가로 스크롤 없음
- [ ] 모바일 390px에서 하단 내비게이션이 콘텐츠를 가리지 않음
- [ ] 데스크톱에서 사이드바 활성 메뉴가 정확함
- [ ] 자산 종류는 기본 6개만 먼저 보임
- [ ] 정기지출 납부월은 반복주기에 맞게 표시됨
- [ ] 예산 하단의 불필요한 링크가 보이지 않음
- [ ] 전체 메뉴의 추가 도구는 접혀 있음
- [ ] 시작가이드가 처음 3단계를 표시함

## 5. 핵심 기능 회귀

- [ ] 카카오 한 줄 지출 입력
- [ ] 카카오 수입 입력
- [ ] 최근 기록 조회·수정·취소
- [ ] WebView POST 저장
- [ ] CSRF 차단
- [ ] 백업·복구 미리보기
- [ ] 미완성 카드 혜택·밈 직접 경로 비노출

## 6. 이상 시 복구

1. 이전 Worker 버전 V22.8.0으로 롤백한다.
2. 데이터베이스 스키마는 변경하지 않았으므로 별도 SQL 롤백은 하지 않는다.
3. 환경변수와 카카오 설정은 변경 전 값을 유지한다.
4. 문제 화면·계정 역할·가계부 ID·발생 시각을 기록한 뒤 재현한다.
