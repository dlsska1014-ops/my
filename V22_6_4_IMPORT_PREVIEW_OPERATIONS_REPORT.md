# V22.6.4 가져오기 미리보기·운영 핫픽스 보고서

적용 버전: `V22.6.4-IMPORT-PREVIEW-OPERATIONS-HOTFIX`

## 결론

V22.6.3의 자연어·다양한 파일 형식 인식은 유지하고, 실제 저장 전에 사용자가 인식 결과를 검토하고 저장할 행을 선택하는 2단계 흐름을 추가했다. 자동 검증 기준으로 무료 베타 배포 후보이며, 운영 Supabase와 실제 모바일 브라우저 확인은 배포 전 남아 있다.

## 사용자 흐름

1. CSV·TSV·TXT·JSON·XLS/XLSX 또는 자연어 목록을 입력한다.
2. `1. 분석 미리보기`를 누른다. 이 단계에서는 거래를 저장하지 않는다.
3. 인식 행, 중복 제외, 예상 수입·지출, 제목 매핑, 자동 보정과 제외 이유를 확인한다.
4. 실제 저장할 행만 체크한다.
5. `선택한 항목 저장`을 눌러 선택 행만 저장한다.
6. 저장 완료·중복 제외·저장 오류와 행별 해결 방법을 확인한다.

## 저장 안전장치

- 미리보기 데이터는 HMAC 서명 토큰으로 보호한다.
- 토큰은 사용자 ID·가계부 ID·작업키·만료시간에 묶이며 30분 후 만료된다.
- 토큰 변조, 다른 계정·가계부 사용, 만료 토큰은 저장하지 않고 다시 미리보기를 안내한다.
- 저장 직전에 가계부 권한과 참여자 상태를 다시 확인한다.
- 각 후보 거래에 작업키 기반 결정적 UUID와 `source_user_key`를 부여한다.
- 같은 저장 요청이 재전송되면 기존 거래 ID를 확인해 중복으로 제외한다.
- 동시에 같은 요청이 들어와도 거래 기본키가 마지막 중복 방어선으로 작동한다.
- 같은 날짜·금액·내용 중복 제외 옵션은 저장 직전 다시 검사한다.

## UI·UX 개선

- 첫 버튼을 `분석하고 가져오기`에서 `1. 분석 미리보기`로 변경했다.
- 미리보기 상단에 “아직 거래는 저장되지 않았습니다”를 명시했다.
- 원본 행, 인식 행, 저장 후보, 중복 제외, 예상 수입·지출을 한눈에 표시한다.
- 전체 선택·전체 해제와 현재 선택 건수를 제공한다.
- 모바일에서 버튼을 전체 너비로 배치하고 표는 가로 스크롤을 허용한다.
- 행 선택 체크박스에 접근성 이름을 추가하고 화면 확대 제한을 두지 않았다.
- 저장 중 버튼을 비활성화하고 처리 상태를 표시한다.
- 아무 행도 선택하지 않으면 데이터는 저장되지 않고 같은 미리보기를 유지한다.

## 운영 계측

원문 거래 내용은 운영 이벤트에 넣지 않고 다음 집계만 기록한다.

- `my_import_preview`: 원본·인식·저장 후보·중복·한도·사유별 건수
- `my_import_commit`: 선택·저장·중복·오류·사유별 건수
- `my_import_row_failed`: 저장 오류 행 번호와 안전 처리된 내부 오류
- `my_import_idempotency_check_failed`: 재전송 확인 조회 실패

이벤트는 기존 운영 대시보드의 인스턴스 메모리 로그에서 확인한다. 장기 통계가 필요하면 별도 집계 테이블과 보관 정책이 필요하다.

## 프리미엄 판단

무료 베타 기능 검증에는 문제가 없다. 유료 출시는 entitlement 원장, 결제·웹훅 멱등성, 환불·해지, 약관·개인정보 검토, 만료 후 읽기 정책이 구현되기 전 진행하지 않는다.

- iPhone Safari·Android Chrome에서 미리보기 표와 선택 저장 확인

- 운영 대시보드에서 가져오기 집계 이벤트 노출 확인
