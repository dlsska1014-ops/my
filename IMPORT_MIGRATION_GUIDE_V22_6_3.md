# V22.6.3 다른 가계부 자료 가져오기 가이드

## 가장 안전한 순서

1. 원본 파일을 복사해 보관한다.
2. 다른 가계부에서 CSV 또는 XLSX로 내보낸다.
3. 똑똑한가계부의 `백업·가져오기`에서 대상 가계부를 다시 확인한다.
4. 처음에는 5~20행만 가져온다.
5. 결과 화면에서 저장 완료, 중복, 미인식, 자동 보정 내용을 확인한다.
6. 정상일 때 전체 파일을 처리한다.
7. 가져온 기록 화면에서 날짜·금액·내용·결제수단·지출자를 표본 확인한다.

원본 파일은 수정되지 않는다. 미인식 행은 저장하지 않으므로 사유를 확인한 뒤 해당 행만 다시 가져오면 된다.

## 지원 입력

| 종류 | 지원 방식 | 참고 |
|---|---|---|
| CSV | 쉼표 자동 인식 | 값 안 쉼표는 큰따옴표 권장 |
| TSV | 탭 자동 인식 | 엑셀/스프레드시트에서 복사·붙여넣기에 적합 |
| TXT | 표 또는 자연어 | 한 줄에 한 거래 권장 |
| JSON | 배열 또는 records/items/data/rows 등 | 객체의 한 단계 중첩 키 지원 |
| XLS/XLSX | 브라우저에서 모든 시트를 TSV로 변환 | 변환 완료 문구가 나온 뒤 제출 |
| 자연어 | `어제 점심 12000원 국민카드` | 날짜가 없으면 오늘로 보정하고 경고 |
| 키-값 | `날짜: ... | 금액: ... | 내용: ...` | `:` 또는 `=` 사용 가능 |

## 행 제목 유사어

| 인식 필드 | 주요 한국어 제목 | 주요 영문 제목 |
|---|---|---|
| 날짜 | 날짜, 일자, 일시, 기준일, 거래일시, 사용일시, 이용일시, 승인일시, 결제일시 | date, datetime, transactionDate, approvalDate, postingDate |
| 구분 | 구분, 유형, 거래구분, 수입지출, 입출금, 차변대변 | type, transactionType, inout, debitCredit |
| 수입 | 수입, 입금, 입금액, 수입금액, 대변 | income, deposit, credit, moneyIn |
| 지출 | 지출, 출금, 출금액, 사용금액, 이용액, 승인금액, 결제금액, 차변 | expense, withdrawal, debit, moneyOut |
| 금액 | 금액, 거래금액, 거래액, 원화금액, 이용금액, 승인액, 청구금액 | amount, transactionAmount, value, price, total |
| 내용 | 내용, 내역, 내역명, 적요, 메모, 사용처, 가맹점명, 거래처명, 품명 | memo, note, description, merchant, payee, item |
| 분류 | 분류, 카테고리, 항목, 대분류, 중분류, 소분류, 태그 | category, classification, class, tag |
| 결제수단 | 결제수단, 결제계정, 카드명, 계좌명, 은행, 금융기관, 자산 | paymentMethod, method, card, account, bank, wallet |
| 지출자 | 지출자, 결제자, 사용자, 구성원, 담당자, 작성자, 닉네임 | payer, spender, member, user, owner |

괄호 안 단위도 허용한다. 예: `이용일시(KST)`, `승인금액(KRW)`, `금액(원)`.

## 자동 보정과 거부 기준

| 상황 | 처리 |
|---|---|
| 표 위 제목·설명 | 거래가 아닌 preamble로 제외 |
| 여러 시트의 반복 제목 | 새 열 구성으로 전환하고 제목 행 제외 |
| 합계·소계·잔액 | 개별 거래가 아니므로 제외 |
| 자연어에 날짜 없음 | 오늘 날짜로 보정하고 경고 |
| 제목이 있는 표에 날짜 없음 | 임의 저장하지 않고 날짜 없음 처리 |
| 금액 열에 문자 | 다른 숫자를 대신 쓰지 않고 금액 형식 오류 처리 |
| 수입·지출 열에 모두 금액 | 어느 쪽인지 임의 선택하지 않고 거부 |
| 따옴표 없는 `12,300` | 금액 셀로 복원하고 경고 |
| 내용의 따옴표 없는 쉼표 | 내용 셀을 합치고 경고 |
| 열 수를 안전하게 맞출 수 없음 | 제목 수와 값 수 불일치로 거부 |
| 같은 날짜·금액·내용 | 기본 설정에서 중복 건너뛰기 |
| 지출자 이름이 한 명과 정확히 일치 | owner/admin은 해당 구성원으로 연결 |
| 지출자 불명확 또는 member 권한 | 가져온 사용자 본인 기록으로 저장하고 경고 |

## 인식하지 못했을 때 확인할 것

- 날짜가 실제 거래일인지, 단순 문서 작성일인지 확인한다.
- 잔액·누계·승인번호 같은 숫자 열은 제거한다.
- 실제 금액 열 제목을 `금액`, `지출`, `수입`, `출금액`, `입금액` 중 하나로 바꾼다.
- 한 행에 거래 하나만 둔다.
- 쉼표가 들어간 내용은 큰따옴표로 감싸거나 TSV로 붙여넣는다.
- 병합 셀, 수식 오류, 이미지 안의 표는 값으로 바꾼다.
- 암호가 걸린 엑셀은 잠금을 해제한 복사본 또는 CSV로 저장한다.
- 엑셀 변환 라이브러리를 불러오지 못하면 CSV로 저장하거나 표를 복사해 붙여넣는다.

## 운영 한도

- 기본 업로드 크기: 5MB (`MY_IMPORT_MAX_BYTES`, 최대 15MB)
- 파서 분석 행: 최대 5,000행
- 기본 1회 저장: 120행 (`MY_IMPORT_LIMIT`, 최대 1,000행)
- 큰 파일은 월별 또는 시트별로 나누는 것을 권장한다.

처리 한도를 넘은 행은 저장되지 않으며 원인 목록에 표시된다.
