# V22.8.2 인증·UI 안정화 보고서

검토일: 2026-07-16 (Asia/Seoul)

## 결과

카카오 KOE006 오류로 사용자를 보내던 구조적 원인을 제거하고, 설정이 불완전한 상태에서는 카카오 로그인을 자동 차단하는 fail-closed 정책을 적용했습니다. 기존 로컬 로그인, 가계부 데이터, V22.8.0 자산 스키마는 유지합니다.

## 확인한 핵심 원인

이전 코드는 `KAKAO_REDIRECT_URI`가 없으면 현재 요청의 `url.origin`으로 callback 주소를 만들었습니다. 운영 도메인 외에 미리보기, workers.dev, www/non-www 주소로 접속하면 카카오 Developers에 등록한 Redirect URI와 달라질 수 있어 KOE006이 발생합니다. REST API 키만 있어도 카카오 로그인 버튼이 자동 노출되어 설정 중인 상태가 사용자 오류로 이어질 수도 있었습니다.

## 변경 내용

### 카카오 OAuth

- `KAKAO_LOGIN_ENABLED`를 명시적으로 켠 경우에만 기능 활성화
- `PUBLIC_BASE_URL`과 `KAKAO_REDIRECT_URI`를 필수값으로 사용
- callback을 `/auth/kakao/callback`의 고정 HTTPS URI로 검증
- 공개 기준 주소와 Redirect URI origin 불일치 차단
- 다른 호스트의 OAuth 시작 요청은 공개 기준 호스트로 먼저 정규화
- Client Secret 필수 모드 지원
- state 불일치, 사용자 취소, 설정 실패, 성공 경로에서 임시 쿠키 정리
- 설정 실패 시 카카오로 리다이렉트하지 않고 로컬 로그인 안내 제공
- 점검 화면과 운영 진단에 카카오 설정 상태 반영

### UI·UX 및 작업 안정성

- 확인창·브라우저 검증이 제출을 취소한 경우 버튼 잠금 금지
- 삭제·방출·탈퇴·계정 통합 작업의 위험 색상과 확인 절차 통일
- 백업 링크를 사용자용 `/my/backup`으로 통일
- 전체 메뉴에 영수증 기록·자동 리포트 추가
- 첫 사용 완료 조건을 첫 기록과 결과 확인으로 분리
- 개인 비밀번호 상태 조회 실패가 저장 오류 안내까지 중단하지 않도록 보호
- 카카오 로그인 복구 안내의 미정의 origin과 잘못된 현재 호스트 callback 표시 수정

소스 코드에서는 카카오 Developers 관리자센터의 실제 등록값을 읽을 수 없습니다. 운영자가 동일한 카카오 앱에 정확한 Web domain과 Redirect URI를 등록해야 실제 로그인 성공이 완성됩니다. 실제 카카오 계정으로 성공 callback까지 확인하기 전에는 “운영 로그인 정상”으로 간주하지 않습니다.
