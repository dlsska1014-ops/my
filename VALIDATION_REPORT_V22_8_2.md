## 신규 회귀 시험이 보장하는 계약

- 인가 URL: `https://kauth.kakao.com/oauth/authorize` + client_id + **명시** redirect_uri + response_type=code + 시도마다 고유한 state
- `KAKAO_REDIRECT_URI` 미설정 시 로그인 버튼 비노출·인가 요청 차단(fail-closed)
- 다른 호스트에서 시작한 로그인은 canonical 호스트로 이동, stale 호스트에 state 쿠키 미발급
- callback 취소/state 불일치/토큰 실패/프로필 실패의 개별 안내문·상태코드·state 쿠키 만료
- 모의 카카오 서버로 전체 로그인 왕복(인가→토큰→프로필→세션 쿠키→가계부 생성 유도) 성공
- `/kakao-login-recovery` 200 응답(V22.8.1까지는 미정의 변수로 500)
- 소스 어디에도 `url.origin` 기반 kakao callback 파생 없음

## 검증 중 발견·수정한 추가 결함

- `/kakao-login-recovery` 페이지가 정의되지 않은 `origin` 변수를 참조해 항상 500(안전모드)으로 떨어지는 실버그를 발견하고 수정했습니다. 기존 54개 시험 어느 것도 이 경로를 실행하지 않아 V22.8.1 검증에서 잡히지 않았습니다.
