# V22.7 인증·원자성·전체 안정화 보고서

## 결과

릴리스 식별자는 `V22.7.0-AUTH-ATOMICITY-STABILITY`입니다. 제보된 보안 설정 이탈, 지출자 수정·표시 누락, 내부 완료 코드와 사용자 UUID 노출, 불필요한 안내 문구를 수정했고, 전체 파일 교차 검토에서 발견된 인증·동시성·가계부 삭제 잔여 데이터·백업 지출자 누락도 함께 보완했습니다.

## 사용자 흐름

- 보안 설정에서 비밀번호를 입력한 뒤 `/`로 이동하지 않고 원래 경로·쿼리·앵커로 복귀합니다.
- 새 가계부는 이름과 내 표시명, 개인 비밀번호를 확인한 뒤 생성하고 초대 단계로 이어집니다.
- 카카오 로그인 사용자는 최초 표시명을 카카오 닉네임으로 채우며 나중에 변경할 수 있습니다.
- 템플릿은 날짜 이름으로 즉시 생성하지 않고 추천 이름을 사용자가 확인·수정한 뒤 생성합니다.
- 홈·기록·입력·정산·메뉴의 하단 내비게이션을 인증 화면에서 일관되게 사용합니다.
- 실패 문구는 내부 코드나 원시 DB 오류 대신 현재 상태, 데이터 보존 여부, 다시 시도할 방법을 설명합니다.

## 인증·보안

- 관리자와 로컬 개인 비밀번호는 PBKDF2-SHA256 210,000회, 사용자별 salt로 저장합니다.
- 존재하지 않는 로그인 이름이나 틀린 비밀번호는 계정을 만들지 않습니다. 가입과 로그인 경로를 분리했습니다.
- 관리자·사용자 세션에 세션 버전을 포함하고 비밀번호 변경 시 이전 세션을 만료합니다.
- 인증 실패 횟수와 잠금 시간을 DB에서 관리하며 Worker 인스턴스가 달라도 제한이 유지됩니다.
- 쓰기 요청은 같은 출처 검사를 통과해야 하며, 외부 출처의 POST·PATCH·PUT·DELETE를 거부합니다.
- 운영 로그는 세션·토큰·원문 비밀번호·사용자 UUID를 그대로 남기지 않고 범위를 축소하거나 해시합니다.
- `/ready`는 필수 설정 누락 개수만 반환하고 Secret 이름이나 값은 외부에 공개하지 않습니다.

## 데이터 원자성

`schema_v22_7_0_auth_atomicity.sql`은 아래 작업을 PostgreSQL 트랜잭션 함수로 처리합니다.

- 로컬 계정 생성·개인 로그인 연결·카카오 계정 연결
- 가계부 전체 삭제
- 월 수입 분류와 분류별 예산 일괄 교체
- 정기거래 월 반영
- 계정 병합
- 거래 수정·삭제·일괄 변경·가져오기
- 가계부 탈퇴

가계부 삭제는 거래·참여자·가계부 행 외에도 예산, 정기거래, 분류, 거래 감사, 사용자 분류·키워드·결제수단·정기계획·별칭·수정 이력·정산 이력·무료 리포트·선택 가계부·카카오 편집/대화 임시 상태를 해당 가계부 범위에서 제거합니다. 단톡방 연결 미러는 Worker가 삭제 직후 정리합니다.

## 지출자·개인정보

- 모바일과 PC 기록 목록에 수입자·지출자 이름을 표시합니다.
- 기록 수정, 일괄 수정, 정기거래, 가져오기에서 현재 가계부의 활성 참여자만 선택할 수 있습니다.
- 거래 수정·삭제는 변경 전·후 데이터를 감사 테이블에 남기되 일반 화면에는 내부 ID를 표시하지 않습니다.
- 참여자·정산·가계부 관리 화면은 이름과 권한 중심으로 표시하고 고유 사용자 UUID 열을 제거했습니다.
- 백업 CSV에는 지출자 표시명만 포함합니다. 가져오는 가계부에서 같은 이름이 정확히 한 명일 때만 연결하고, 없거나 중복이면 미리보기 경고 후 가져온 사용자로 저장합니다.

## 가져오기

- CSV, TSV, TXT, JSON, XLS/XLSX, 여러 시트, 쉼표·세미콜론·탭·파이프 구분을 처리합니다.
- 날짜·금액·수입·지출·내용·분류·결제수단·지출자의 한국어·영문 제목 유사어를 지원합니다.
- 설명행, 반복 제목, 제목 없는 표, 키-값 문장, 자연어 거래, 엑셀 날짜, UTF-8·UTF-16·EUC-KR 계열을 보정합니다.
- 저장 전 미리보기에서 인식 열, 자동 보정, 중복 후보, 실패 행의 원인과 수정 방법을 보여줍니다.
- 선택한 행만 서명 토큰으로 저장하며 같은 확인 요청을 반복해도 거래가 중복되지 않습니다.

## 무료·숨김 정책

- 완료된 영수증, 정기거래, 고급 정산, 자동 리포트, 스마트 분석, 예산, 결제수단, 백업·가져오기는 일반 무료 기능입니다.
- 결제·구독·프리미엄 사용량 제한은 연결하지 않았습니다.
- 미완성 소비 카드/밈과 카드 실적/혜택은 메뉴·공유·직접 URL에서 모두 숨기고 `404`와 `noindex`를 적용합니다.

## UI·UX 점검

- 인증 후 화면에 사업자 정보 푸터를 삽입하지 않아 고정 하단 메뉴와 겹치지 않습니다.
- 320·360·390px 기준으로 23개 핵심 화면의 viewport, 폼 레이블, 중복 ID, 터치 크기, 가로 넘침 방어 CSS를 정적 검증했습니다.
- 내부 상태 코드는 한국어 완료·오류 문구로 변환하고, 빈 화면에는 첫 기록 또는 설정으로 가는 다음 행동을 제공합니다.
- 이 실행 환경에는 Chromium 실행 파일이 없어 V22.7 픽셀 캡처를 새로 만들지 못했습니다. 배포 전 실제 iPhone·Android·카카오 인앱브라우저 확인은 필수입니다.

## 남은 운영 확인

1. 운영 DB 백업 후 마이그레이션을 적용하고 `/ready`가 200인지 확인합니다.
2. 실제 카카오 OAuth와 서로 다른 단톡방 두 곳의 가계부 연결을 확인합니다.
3. 참여자 2명 이상의 테스트 가계부에서 지출자 변경, 백업, 다른 가계부 가져오기, 정산 반영을 확인합니다.
4. 테스트 가계부 삭제 전후로 관련 거래·리포트·카카오 연결이 남지 않는지 확인합니다.
5. iPhone Safari·Android Chrome·카카오 인앱브라우저에서 키보드가 열린 상태까지 점검합니다.
