# 오픈챗봇 채팅 우선 구성 기준 — V21.5

## 목표

처음 사용자가 별도 설명서를 읽지 않아도 카카오톡 안에서 다음 과정을 완료할 수 있어야 합니다.

```text
가계부 만들기 또는 참여
→ 구성원 초대
→ 단톡방 연결
→ 예산 설정
→ 내 이름 설정
→ 첫 기록
→ 날짜별 요약
```

## 역할 분리

- OpenBuilder: 대표 명령과 폴백을 `/skill`로 전달
- Worker: 단계 판단, 파싱, 권한, 저장, 응답
- Supabase: 단계 상태, 가계부, 구성원, 예산, 거래 저장

OpenBuilder 컨텍스트는 영구 상태로 사용하지 않습니다.

## quickReplies 사용 정책

단계형 설정에서만 사용합니다.

- 시작
- 새 가계부 만들기
- 초대코드 참여
- 예산 설정
- 내 이름 설정
- 단톡방 연결

일반 거래 저장, 남은예산, 요약, 오류 안내에는 기본적으로 사용하지 않습니다.

## 홈페이지 전환 정책

카카오톡:

- 빠른 기록
- 예산 설정과 확인
- 날짜별 요약
- 구성원·결제자 구분

홈페이지:

- 상세 그래프
- 전체 거래 검색·수정
- 데이터 내보내기
- 고급 분석·관리

거래를 저장할 때마다 링크를 붙이지 않습니다. 가계부 생성 완료, 예산 설정 완료, 상세 분석 요청 같은 자연스러운 지점에서만 대표 링크 1개를 제공합니다.
