# V21.4-KAKAO-COMMANDS

## 배경
- 카카오 그룹챗봇 beta 가이드(슬라이드 25 대표 명령어, 유의사항 슬라이드)에 맞춰 채팅창 `/` 드롭다운 대표 명령어를 도입합니다.
- 드롭다운으로 명령어를 선택하면 `기록 `, `수정 ` 같은 접두어가 입력창에 남은 채 이어서 입력되므로, `/skill` 파서가 접두어를 이해하지 못하면 기존 저장/수정 흐름이 깨집니다.
- A) OpenBuilder 관리자센터 설정(코드 수정 없음)과 B) `/skill` 파서 보강(코드 수정)으로 나눠 반영했습니다.

## 반영

### A. OpenBuilder 대표 명령어 등록 (관리자센터 — 수동 작업)
- 등록표(8개, 순서 고정, 설명은 기록·수정·단톡방 연결 3건만)와 규칙 준수 체크리스트를 `KAKAO_OPENBUILDER_UTTERANCES.md`의 "V21.4 대표 명령어(드롭다운) 등록" 섹션에 정리했습니다.
- 블록 발화 병합: 입력 예시 블록에 `기록`, `기록 점심 12000원 국민카드` 추가, 수정 블록에 `수정`, `수정 01번 금액 13000원` 추가, 요약 블록의 `정산` 발화는 이미 존재함을 확인했습니다. `기록` 단독 발화는 최근 기록 블록에서 입력 예시 블록으로 이동했습니다. 폴백 블록의 Skill 연결 원칙은 그대로입니다.

### B. src/index.js `/skill` 파서 보강
- **B-1 선행 명령어 스트리핑**: `stripLeadingCommand(text)`를 추가하고 `handleKakaoSkill`의 utterance 정규화 직후에 적용했습니다.
  - `기록 점심 12000원` → `점심 12000원`, `수정 01번 금액 13000원` → `01번 금액 13000원`
  - `기록`/`수정` 단독 입력은 정규식이 뒤에 내용이 있을 때만 매칭하므로 스트리핑되지 않습니다.
  - `오늘 기록`은 `기록`으로 시작하지 않으므로 스트리핑 대상이 아니며 조회 그대로입니다.
- **`기록` 단독의 역할 변경**: 기존에는 `isRecentCommand`(최근 조회)에 매칭됐으나, 대표 명령어 "기록"은 입력 버튼이므로 `isInputExampleCommand`로 옮겨 입력 예시 안내로 응답합니다. `오늘 기록`/`오늘 내역`/`최근` 계열 조회는 그대로입니다.
- **반복 발화 방어(dedup) 정합**: `checkKakaoRepeatGuard`가 스트리핑 후 텍스트로 키를 생성하도록 변경했습니다. `점심 12000원` 직후 `기록 점심 12000원`을 보내도 같은 내용으로 판정해 중복 저장을 막습니다. 가이드성 명령(입력 예시·수정가이드)은 조회 명령과 동일하게 반복 방어에서 제외해 드롭다운 연타 시 불필요한 차단이 없도록 했습니다.
- **B-2 도움말 보강**: helpText에 `"/" 를 입력하면 명령어 목록이 열립니다.` 한 줄을 추가했습니다. 개인정보/봇 입장 안내의 기존 안전 문구("봇에게 직접 보낸 명령어만 처리")는 유지했습니다.
- **B-3 응답 형식**: simpleText 중심·요약형 조회 응답·KAKAO_RETRY_DEDUP(트랜잭션 단위 중복 방어)은 변경하지 않았습니다. 트랜잭션 중복 방어는 파싱된 row 기준이므로 접두어 유무와 무관하게 동작합니다.
