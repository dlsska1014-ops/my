## 변경 내용

### 1. 성능 최적화
- **OCR 지연 로딩**: Tesseract.js를 페이지 진입 시가 아니라 사용자가 사진을 선택한 순간에만 내려받습니다. 문자만 붙여넣는 사용자는 아무것도 다운로드하지 않습니다.
- **서버 조회 경량화**: 영수증 화면 전용 `getReceiptPageContext`가 사용자·가계부 조회를 병렬로 수행하고, 분류 자동완성용으로 `category` 컬럼만 읽습니다. 동일 조건에서 **Supabase 호출 17회 → 8회(재방문 7회)**, 불필요한 멤버·캘린더·예산·분석 조회 제거.
- **이미지 축소 후 인식**: OCR 전에 긴 변 1600px로 축소(EXIF 회전 반영)해 인식 시간을 크게 줄입니다. 실측: 3200×2400 → 1600×1200, 53KB → 14KB.
- **워커 재사용**: 한 번 초기화한 인식 엔진을 재사용해 두 번째 사진부터는 다운로드·초기화 없이 바로 인식합니다.

### 2. 안정화
- **자동 인식 복구(핵심 버그 수정)**: 클라이언트 코드를 실제 함수(`receiptCaptureClientMain`)로 유지하고 `toString()`으로 직렬화(`<script id="receiptCaptureRuntime">`)해 정규식 파괴 문제를 원천 차단. 파서 정확성을 스모크 테스트로 고정.
- 90초 타임아웃 + **분석 취소** 버튼, 취소·실패 시 버튼·상태 완전 복구.
- CDN 로드 실패 시 명확한 안내와 문자 붙여넣기 대안 제시(브라우저 테스트로 실경로 검증).
- 이미지 아닌 파일·20MB 초과 파일 거부, 같은 사진 재선택 가능, 중복 실행 방지(busy 잠금).
- `hidden` 속성이 페이지 CSS에 밀려 미리보기·취소 버튼이 항상 노출되던 문제 수정(`[hidden]{display:none!important}`).
- 저장 실패 시 원인별 오류 메시지(권한/확인 체크/날짜·금액/일시 오류) 구분 표시.

### 3. 앨범 접근·UX 개선
- **🖼️ 앨범에서 선택 / 📷 카메라로 촬영 버튼 분리** — 앨범 버튼은 `capture` 없이 갤러리를 열고, 카메라 버튼은 기존처럼 후면 카메라를 엽니다.
- 선택한 사진 **미리보기 + 파일명·용량 표시 + 사진 지우기** — 확인 체크 전에 원본과 비교할 수 있습니다.
- PC에서 **드래그&드롭, 클립보드 붙여넣기(Ctrl+V)** 지원.
- 인식 단계별 상태 문구(모듈 다운로드 → 언어 데이터 준비 → 인식 %)와 **진행률 바**, 상태 영역 `aria-live="polite"`(스크린리더 안내).
- 인식 완료 시 결과 카드로 자동 스크롤 + 상호 입력란 포커스, 인식 실패 항목(상호·날짜·총액)을 구체적으로 안내.
- 총액 입력 시 쉼표 자동 표기(`35,400`), 서버는 쉼표 포함 값도 정상 저장.
- 가계부·월 변경 시 자동 적용(버튼은 JS 미동작 환경 대비 유지).
- 보라색 저명도 UI를 중립 색상으로 교체, 취소·지우기는 공용 가이드 디자인의 위험 동작 스타일 적용, `abPageReceipts` 스코프 클래스 부여.
- 320·360·390·768·1365px에서 가로 넘침 0 확인.

### 4. 서버 저장 로직
