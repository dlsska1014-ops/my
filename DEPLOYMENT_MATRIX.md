## V22.8.8 실제 적용 판단

### Cloudflare Worker

- 적용 파일: `src/index.js`
- 방식: 기존 Worker 코드 전체 교체
- 부분 붙여넣기: 금지
- 최종 ZIP의 배포 소스는 `src/index.js` 한 개입니다.

### Supabase SQL

- 신규 SQL 파일: 없음
- 실행 작업: 없음
- 코드 확인 결과: 새 테이블·컬럼·인덱스·RLS·RPC 참조가 없습니다.
- 이유: 브라우저 제출 상태, 버튼 문구, 확인창, 비밀번호 입력 피드백만 변경했습니다.
- 전제: 운영 DB에는 기존 V22.8.0까지 필요한 스키마가 이미 적용되어 있어야 합니다.
- 주의: 누적 ZIP의 과거 SQL 파일은 이력과 신규 설치 참고용이며 V22.8.8 배포 때 다시 실행하지 않습니다.

### Cloudflare 설정

- 신규 환경변수·Secret: 없음
- 기존 `PUBLIC_BASE_URL`, Supabase 키, 로그인·세션 Secret을 그대로 유지합니다.
- 실제 Secret은 ZIP에 포함하지 않습니다.

### 카카오 설정

- Skill URL 변경 없음: `https://ttokttok-accountbook.com/skill`
- 블록·엔티티·파라미터 변경 없음
- Event API·선톡·자동 푸시 추가 없음
- V22.8.7의 1:1·그룹 응답 정책을 그대로 유지합니다.

### 보호 화면

- 분석 클라이언트 소스 SHA-256: `b73386dfddd66aa42000b7b34b6c03b7deeda3ac5824b75468db3aa269087a5d`
- 분석 렌더러 소스 SHA-256: `0d03ce468e0203e1eb7bc344e0fc24876f74c92de492389246f5dd80ea0de58d`
- 두 해시는 V22.8.7과 동일합니다.
- 영수증 화면은 구현을 변경하지 않고 기존 회귀 검증을 유지했습니다.

## 다음 버전 판단 규칙

1. 새 테이블·컬럼·인덱스·RLS·RPC 또는 기존 쿼리의 새 필드 참조가 있는지 검색합니다.
2. DB 변경이 있으면 누적·멱등 SQL, 적용 순서, 검증 쿼리, 롤백 한계를 먼저 작성합니다.
3. DB 참조가 없으면 SQL을 만들지 않고 `src/index.js` 교체만 명시합니다.
4. 새 환경변수나 카카오 설정은 이름, 필수 여부, Secret 여부, 운영 콘솔 작업을 분리합니다.
5. 보호 화면은 소스 해시와 경로 회귀 검증으로 변경 여부를 확인합니다.
6. 최종 ZIP에서 매트릭스, 실제 파일, 검증 보고서, 체크섬이 서로 일치하는지 검사합니다.
