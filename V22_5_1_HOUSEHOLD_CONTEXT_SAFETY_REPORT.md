# V22.5.1 가계부 컨텍스트 안정화 보고서

## 1. 확인된 원인

웹 `/my/households`는 현재 웹 로그인 사용자 ID의 참여 가계부만 표시합니다. 카카오 Skill은 `botUserKey`로 별도의 사용자를 찾습니다. 두 신원이 아직 통합되지 않았다면 웹에는 가계부가 없는데 카카오톡에는 과거 챗봇 계정의 가계부가 보일 수 있습니다.

더 중요한 코드 문제는 연결되지 않은 단톡방에서도 카카오 챗봇 사용자의 첫 참여 가계부를 fallback으로 선택했다는 점입니다. 그 결과 다음 현상이 가능했습니다.

- 단톡방을 연결하지 않았는데 특정 개인 가계부 이름 표시
- 사용자가 고르지 않은 가계부의 초대코드 노출
- 자연어 거래가 의도하지 않은 개인 가계부에 저장될 위험
- 여러 가계부 중 조회 순서상 첫 가계부가 암묵적으로 선택

거래 저장이 새 가계부나 초대코드를 생성한 것은 아닙니다. 이미 존재하던 챗봇 계정의 참여 가계부를 잘못 현재 컨텍스트로 사용한 문제였습니다.

## 2. 수정 원칙

### 직접 채팅

- 가계부 없음: 생성·초대코드 참여만 안내
- 선택 상태 없음: 참여 가계부 목록을 먼저 표시하고 번호 또는 이름으로 확인
- 선택 완료: 이후 기록·예산·요약·초대코드는 선택 가계부 기준
- 여러 가계부: `가계부 전환`으로 언제든 변경

### 단톡방

- 연결 전: 기록·조회·초대코드를 개인 가계부에 fallback하지 않음
- 연결 가능한 목록: 현재 사용자가 owner/admin인 가계부만 표시
- 연결 후: 해당 방은 연결된 가계부 하나만 사용
- 연결 가계부 삭제: 저장된 그룹 연결도 함께 제거

### 응답 투명성

거래 저장 응답 등에 `가계부: 이름`을 표시해 사용자가 처리 위치를 즉시 확인할 수 있도록 했습니다.

## 3. 데이터 변경

신규 테이블이나 컬럼은 없습니다. 기존 `accountbook_settings`에 다음 선택 포인터를 저장합니다.

```text
kakao_selected_household_v2251:<user_id>
```

단톡방 연결은 기존 `kakao_group_links` 설정을 그대로 사용합니다.

## 4. 계정 분리 처리

이 패치는 웹 로그인 계정과 카카오 챗봇 계정을 자동으로 합치지 않습니다. 자동 통합은 다른 사용자의 데이터를 잘못 결합할 위험이 있기 때문입니다.

관리자는 `/identity-audit`에서 다음을 확인해야 합니다.

- 카카오 챗봇 사용자와 카카오 웹 로그인 사용자의 계정 유형
- 각 계정이 참여한 가계부와 역할
- 각 계정의 거래 건수
- 유지할 주 계정과 병합할 보조 계정

통합 전 백업이 필요하며, 기억하지 못하는 가계부도 거래 건수와 연결 정보를 확인하기 전 삭제하지 않습니다.
