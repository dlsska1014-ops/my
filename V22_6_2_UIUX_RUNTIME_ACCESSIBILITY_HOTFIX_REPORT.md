# V22.6.2 UI·UX 런타임·접근성 핫픽스 보고서

## 검토 기준

- 기준 파일: `kakao-accountbook-cloudflare-v22.6.1-identity-session-household-list-recovery-hotfix (1).zip`
- 기준 파일 SHA-256: `a8c4fdaf2be7d9a16991911476a291e3eec3e91547869ad524f42ab3802e8eac`
- 적용 버전: `V22.6.2-UIUX-RUNTIME-ACCESSIBILITY-HOTFIX`
- 원칙: V22.6.1 계정·세션 복구와 V22.5.4 상태·권한·동시성 규칙을 보존하고, 사용자 화면과 클라이언트 동작만 선택적으로 보완

## 결론

V22.6.1의 계정 통합 세션 복구 방식은 타당했다. 보조 계정 쿠키를 주 계정으로 해석하고, GET/HEAD에서는 쿠키를 재발급하며, POST 본문은 리다이렉트로 잃지 않도록 처리한다. 이 로직은 V22.6.2에 그대로 유지했다.

UI·UX에서는 실제 동작 결함, 흐름 불일치, 모바일 탐색·접근성 문제를 확인했다. 주요 결함을 수정하고 Chromium 데스크톱·모바일 18개 렌더링, 모바일 실제 클릭, 기존 대화·권한·NLU 회귀 검증을 통과했다. 운영 배포 전에는 실제 문제 계정과 실데이터를 이용한 최종 확인이 남아 있다.

## 실제 결함 — 수정 완료

1. **모바일 한 줄 금액 해석 불안정**
   - 템플릿에서 생성되는 기존 정규식 파서가 축약 금액을 안정적으로 처리하지 못할 수 있었다.
   - `5천`, `12000`, `250만원`, `1만 5천원`을 공통 파서로 처리하고 금액·수입/지출·결제수단·분류·메모를 채우도록 수정했다.

2. **홈 거래 클릭 후 수정 진입 불가**
   - 홈 타임라인은 `openEdit(...)`를 호출했지만 실제 함수가 없어 클릭이 동작하지 않았다.
   - 편집 가능한 행만 버튼으로 만들고, 해당 거래의 최근 내역 카드·수정 영역을 열어 포커스를 이동하도록 연결했다.

3. **온보딩 순서 불일치**
   - 공개 안내와 로그인 후 시작가이드가 예산·결제수단·정기지출을 첫 기록보다 먼저 요구했다.
   - 모든 시작 안내를 `가계부 만들기 → 초대·참여 → 단톡방 연결 → 기록 튜토리얼 → 첫 기록`으로 통일했다.
   - 첫 기록 완료 여부는 선택 월이 아닌 해당 가계부 전체 기록 수를 기준으로 판단한다.

4. **모바일 하단 메뉴 불일치**
   - 화면별로 홈·분석·캘린더·예산·분류 등 구성이 달라 현재 위치와 다음 행동을 예측하기 어려웠다.
   - `모임 / 기록 / 입력 / 정산 / 내정보` 5개로 통일하고, 중앙 입력 버튼을 노란 핵심 액션으로 표시했다.
   - 기본 `/app`은 기록, `#add`는 입력을 활성화하며 해시 변경 후에도 현재 탭을 갱신한다.

5. **모바일 확대·키보드 접근성 제한**
   - 일부 화면에 `maximum-scale=1`이 있었고 포커스 표시, 폼의 접근 가능한 이름, 제목 계층이 부족했다.
   - 확대 제한을 제거하고 `focus-visible`, `prefers-reduced-motion`, 폼 이름, `h1`, 내비게이션 ARIA 상태를 보완했다.
   - 접근성 이름 주입 시 self-closing 태그 형식을 보존하도록 렌더링 단계도 검증했다.

6. **사용자 화면에 내부 구현 용어 노출**
   - 로그인 화면에 Cloudflare 환경변수·API 키·Skill 용어가 노출되고, 메뉴와 프리미엄 화면에 운영용 설명 및 숨김 기능명이 남아 있었다.
   - 사용자 행동 중심의 짧은 문구로 교체하고 기술 오류 상세는 운영 로그에만 남겼다.

7. **프리미엄 수치의 과도한 정밀도**
   - 단순 예측이 확정 금액처럼 보일 수 있었다.
   - 근사 금액, 기준일, 단순 일평균이라는 계산 근거, 베타 기간 무료 범위를 표시했다.

8. **버전 파일 불일치**
   - 첨부 V22.6.1의 `VERSION.txt`가 V22.6으로 남아 있었다.
   - 릴리스 식별자를 V22.6.2로 통일했다.

## UI·UX 개선 사항

- 공개 페이지 모바일 메뉴와 현재 페이지 표시 추가
- 공개 기능 카드 제목을 `h2`로 정리해 `h1 → h3` 건너뜀 제거
- `/app`의 중복 예산·고정지출 섹션 제거, 전용 관리 화면 유지
- 입력 결과 안내에 `aria-live` 적용
- 터치가 잦은 메뉴·칩·날짜 버튼의 최소 높이 보완
- 편집 권한이 없는 거래는 클릭 가능한 요소로 오인되지 않게 일반 행으로 표시
- 프리미엄·메뉴·로그인 문구를 한 화면 한 결정 원칙에 맞게 단순화

## 정상 확인 — 변경하지 않음

- V22.6.1의 `verifyRawUserSession`, `resolveEffectiveUserId`, 병합 포인터·감사 기록 복구, GET/HEAD 쿠키 재발급, POST 본문 보존
- V22.5.4 대화 상태 검증, 만료·손상 상태 정리, 직접채팅/단톡방 범위 분리
- 거래 생성자와 지출자 권한 분리, 삭제 확인, 단톡방 재연결 권한 검사
- quickReplies 기본 미노출과 단계형 흐름에서만 선택지 제공하는 정책
- 미완성 밈·카드 실적 기능 404·noindex 정책
- Supabase 테이블·컬럼·RLS·저장소 및 OpenBuilder 구성

## 잠재 위험 — 배포 전·후 확인 필요

1. 자동시험은 모의 Supabase 기준이므로 실제 문제 계정에서 가계부 수·역할·거래 수를 대조해야 한다.
2. UI 보완은 기존 대형 단일 Worker의 렌더링 결과를 안전하게 정규화하는 핫픽스 방식이다. 다음 정식 버전에서는 공통 레이아웃·폼·내비게이션 컴포넌트로 분리하는 것이 좋다.
3. 매우 긴 가계부명·분류명, 대량 거래, iOS Safari 안전영역은 실제 기기 추가 확인이 필요하다.
4. 프리미엄 예상치는 단순 일평균 기반이므로 금융 조언이나 확정 지출로 사용하면 안 된다.
5. 과거 계정 통합 감사 기록이 없으면 주·보조 계정을 자동 추정하지 말고 관리자 확인이 필요하다.
