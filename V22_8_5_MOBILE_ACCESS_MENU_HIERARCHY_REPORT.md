# V22.8.5 모바일 접근·메뉴 위계 안정화 보고서

## 결론

PC에서 이미 로그인된 사용자가 모바일에서는 카카오 `KOE006` 또는 비활성 상태 때문에 진입하지 못하는 상황을 코드와 화면으로 분리해 처리했습니다. 카카오 설정이 완성되기 전에도 기존 계정 로그인을 첫 화면에서 바로 사용할 수 있고, PC에서 개인 비밀번호를 만드는 정확한 경로를 안내합니다.

전체 메뉴와 통합 사이드바는 색을 늘리는 방식이 아니라 크기·밀도·반복 리듬을 차등화했습니다. 사용자가 만족한 분석 본문은 수정하지 않았습니다.

## 모바일 접근 원인

모바일 기기만 차단하는 사용자 에이전트 분기는 확인되지 않았습니다. PC는 기존 세션 쿠키로 계속 접속하지만 모바일은 새 로그인이 필요하고, 카카오 로그인이 꺼져 있거나 Kakao Developers 등록값과 요청 Redirect URI가 다르면 카카오 경로를 사용할 수 없습니다.

즉시 사용할 수 있는 경로는 다음과 같습니다.

1. 로그인된 PC에서 `전체 메뉴 → 개인 비밀번호`를 엽니다.
2. 로그인 이름과 개인 비밀번호를 설정합니다.
3. 모바일 `/my`에서 같은 값으로 기존 계정 로그인을 합니다.

카카오 로그인을 다시 켤 때는 Web domain, Redirect URI, REST API 키, 공개 기준 주소를 같은 카카오 앱 기준으로 대조해야 합니다.

## 성능

홈의 병렬 조회, 중복 런타임 축소, OCR·엑셀 모듈 지연 로딩은 유지됩니다. 동일 QA 데이터에서 백엔드 호출마다 20ms 지연을 준 비교 결과 V22.8.3의 약 576.5ms에서 V22.8.5의 약 228.7ms로 줄어 약 60% 개선 상태를 확인했습니다.

이 값은 같은 조건에서 코드 대기 구조를 비교한 수치이며 실제 운영 네트워크의 절대 시간을 보장하지는 않습니다.

## 적용과 롤백

- 신규 데이터베이스 마이그레이션은 없습니다.
- `src/index.js`만 교체하고 기존 환경변수와 Secret을 유지합니다.
- 문제 발생 시 직전 Worker 코드로 롤백하며 SQL은 롤백하지 않습니다.
- Kakao Developers의 Web domain·Redirect URI 등록은 운영자가 별도로 완료해야 합니다.
