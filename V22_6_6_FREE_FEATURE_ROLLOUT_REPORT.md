# V22.6.6 전체 스마트 기능 무료 개방 보고서

> 이 문서는 V22.6.6 변경 이력입니다. V22.6.7에서는 영수증·반복 거래·정산·리포트·스마트 분석의 무료 제공은 유지하지만, 미완성인 소비 카드/밈과 카드 실적/혜택은 다시 숨겼습니다. 최신 기준은 `V22_6_7_STABILIZATION_HIDDEN_INCOMPLETE_REPORT.md`입니다.

## 결정

초기 사용자 확보 단계에서는 별도 프리미엄 등급을 운영하지 않습니다. 기존 프리미엄 후보로 준비하던 사용자 기능을 모두 일반 기능으로 제공하며 결제, 구독, 사용량 제한, 잠금 화면을 연결하지 않습니다.

과거 주소인 `/my/premium`은 기존 링크가 깨지지 않도록 남겨 두되 `/smart-tools`와 같은 무료 화면을 표시합니다.

## 무료로 개방한 기능

| 기능 | 사용자 경로 | 구현 내용 | 변경 권한 |
|---|---|---|---|
| 스마트 분석 | `/smart-tools` | 월말 예상, 반복지출 후보, 이상지출, 절약 후보, 스마트 예산·미션 | 참여자 조회 |
| 영수증 기록 | `/receipts` | 브라우저 OCR·문자 붙여넣기, 사용자 확인 후 저장, 중복 방지 | member 이상 저장 |
| 반복 거래 자동화 | `/smart-tools` | 후보를 명시적으로 확정한 뒤 월 1회 자동 반영 | owner/admin 확정 |
| 고급 정산 | `/settlement-summary` | 동일·비율·인원수·품목별 분배, 최소 송금 제안, 완료 이력 | 참여자 조회, owner/admin 완료 |
| 자동 리포트 | `/reports` | 주간·월간 스냅샷, 공유 문구, 인쇄·PDF 저장 | 참여자 조회, owner/admin 설정 |
| 카드 혜택 | `/card-benefits` | 등록 카드 실적과 예상 혜택 계산 | 참여자 조회 |
| 소비 카드 | `/meme-lab`, `/meme-archive` | 카드 생성·저장·공유·보관함·랭킹 | member 이상 저장, owner/admin 삭제 |
| 가족별 분석 | `/my/analysis` | 참여자·분류·결제수단별 비교 | 참여자 조회 |

## UI·UX 반영

- 메뉴에서 `무료 스마트 도구`, `영수증 기록`, `자동 리포트`, `카드 혜택`, `소비 카드`, `소비 도감`을 바로 찾을 수 있습니다.
- 화면에서 프리미엄 등급, 유료 전환, 결제 유도, 잠금 해제 문구를 제거했습니다.
- 스마트 도구의 모바일 지표를 2열 카드로 정리했습니다.
- 영수증 파일 선택기를 한글 버튼과 선택 파일명으로 표시합니다.
- 소비 카드 제목의 어두운 배경 대비와 보관함 빈 상태의 줄바꿈을 보완했습니다.
- 360px과 390px 폭에서 가로 넘침, 잘린 컨트롤, 하단 메뉴 겹침, 라벨 없는 입력 요소를 검사했습니다.

## 데이터·권한 안전성

- 무료 개방은 과금 정책만 바꾸며 가계부 역할과 데이터 격리를 완화하지 않습니다.
- 요청한 `household_id`가 로그인 사용자의 참여 가계부가 아니면 조회·저장을 거부합니다.
- viewer는 조회만 가능하고 자동화 설정, 정산 완료, 소비 카드 삭제를 할 수 없습니다.
- 영수증 이미지는 서버에 올리지 않고 브라우저 안에서 읽습니다. 인식 결과는 사용자가 확인해야 저장됩니다.
- 반복 거래와 자동 리포트는 가계부·기간 기준 멱등 키로 중복 실행을 방지합니다.
- 공개 소비 카드는 사용자가 저장·공유한 개별 카드만 공개 URL로 접근할 수 있습니다.

## 운영 정책

- 사용자 기능은 기본값으로 열립니다. 기능 환경변수는 장애 시 긴급 중지용이며 유료 등급 판별에 사용하지 않습니다.
- 내부 콘텐츠 제작·검수 도구는 사용자용 프리미엄 기능이 아니라 운영 도구이므로 기본적으로 외부에 노출하지 않습니다.
- 자동 리포트는 생성만 수행하며 카카오톡으로 임의 발송하지 않습니다. 사용자가 복사·공유합니다.
- 카드 혜택 금액은 입력 기록과 내장 조건에 따른 예상값입니다. 운영 전 카드사 최신 약관을 확인해야 합니다.

## 호환성

- 기존 가계부 생성·참여·초대·다중 단톡방·가져오기·예산·비밀번호·삭제 흐름을 유지합니다.
- Supabase 신규 필수 마이그레이션은 없습니다. 기존 `transactions`, `accountbook_recurring`, `accountbook_settings`, 소비 카드 저장 구조를 사용합니다.
- OpenBuilder Skill URL과 기존 `/my/premium` 링크는 변경 없이 동작합니다.
