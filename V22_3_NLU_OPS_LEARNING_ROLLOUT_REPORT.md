# V22.3 NLU 운영·학습·점진배포 번들

## 목적
V22.2의 NLU와 분석 UI를 유지하면서, 실제 운영에서 실패 표현과 응답 지연을 확인하고 반복 개선할 수 있는 관측성 계층을 추가한다.

## 추가 기능
- `/nlu-ops`: 관리자용 자연어 운영·학습 센터
- `/nlu-ops.json`: 런타임·영구 집계 JSON
- `/nlu-failures.csv`: 비식별 실패 샘플 CSV
- `/skill` 응답 헤더: 버전·intent·NLU result·request id·latency
- 현재 Worker 인스턴스의 성공/폴백/확인/오류·p50/p95/p99 집계
- 선택형 Supabase 시간대별 집계와 실패 샘플 중복 집계
- 정기 스케줄 실행 시 실패 샘플 보존기간 정리

## 개인정보·비용 기본값
모든 영구 NLU 로그는 기본 비활성이다. 메모리 집계는 Worker 인스턴스 재시작 시 초기화된다. 영구 집계에는 사용자 키를 저장하지 않는다. 실패 샘플은 선택적으로 비식별 텍스트만 저장한다.

## 변경하지 않은 것
- OpenBuilder 시나리오·블록·발화
- `/skill` URL
- 기존 거래·예산·권한·중복방지
- V22.1 Intent Registry와 4,738건 평가셋
- V22.2 분석 스튜디오
- 기존 Supabase 스키마(선택 SQL 미적용 시)
