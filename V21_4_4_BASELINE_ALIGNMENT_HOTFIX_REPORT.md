# V21.4.4-BASELINE-ALIGNMENT-HOTFIX

## 배경
- `KAKAO_CHATBOT_IMPROVEMENT_ROADMAP_V1.md`가 확정한 P0(운영 배포 전 필수) 항목을 V21.4.1 기준본 위에 최소 변경으로 반영했습니다.
- 추가 요구사항: **처음 시작 흐름을 "가계부 생성 → 참여 → 기록" 순서로 재구성.** 기존에는 가계부가 없는 신규 사용자에게도 "점심 12000 체크카드" 같은 기록 예시부터 안내했고, 기록을 시도하면 자동으로 가계부가 만들어졌습니다. 가계부가 없는 상태에서 기록부터 안내하는 순서 문제를 온보딩 단계화로 해결했습니다.

## 반영

### 온보딩 재구성 — 생성 → 참여 → 기록 (핵심)
- **자동 가계부 생성 중단**: `ensurePrimaryHousehold`(없으면 자동 생성)를 조회 전용 `findPrimaryHousehold`로 교체했습니다. 가계부가 없는 사용자가 기록·조회를 시도하면 `kakaoOnboardingGuideText`가 단계별 안내를 반환합니다.
  - 1단계 가계부 만들기(웹 `/my/households`, URL 1개) → 2단계 함께 쓰기(그룹방 `단톡방 연결 초대코드` / 개인은 웹 초대코드 입력) → 3단계 기록(`점심 12000원 국민카드`)
  - 그룹방/개인방 여부에 따라 2단계 문구가 달라집니다. DB 쓰기는 발생하지 않습니다.
  - `KAKAO_DEFAULT_HOUSEHOLD_ID` 강제 지정 배포 구성은 기존대로 동작합니다.
- **도움말/메뉴/입력예시 재배치**: helpText가 "처음 시작 순서 1.만들기 2.참여 3.기록"으로 시작하고, `메뉴`(사용 가능한 명령)와 `/kakao-command-menu.json` 카탈로그를 시작 → 기록 → 조회 → 관리 순서로 재정렬했습니다. 입력 예시에도 "가계부가 없다면 가계부 만들기 먼저" 힌트를 추가했습니다.
- **대표 명령어 드롭다운 재구성** (관리자센터 수동 반영): 1 가계부 만들기 → 2 단톡방 연결 → 3 기록 → 4 오늘 기록 → 5 요약 → 6 남은예산 → 7 수정 → 8 도움말. 8개 제한으로 `정산`은 드롭다운에서 제외(발화·파서는 유지). 등록표는 `KAKAO_OPENBUILDER_UTTERANCES.md`에 갱신했습니다.

### 로드맵 P0 정렬
- **P0-1 비정상 요청 차단**: `/skill`의 비 JSON 본문은 운영 기본값에서 저장·수정·참여 로직에 진입하지 않고 HTTP 200 안전 응답만 반환합니다. 원문 테스트는 `ALLOW_RAW_SKILL_TEST=1` 개발 환경 설정에서만 허용합니다.
- **P0-2 quickReplies**: V21.4.1에서 기본 미사용으로 전환된 상태를 유지하고, `/health`·명령어 JSON 라벨을 `disabled_by_baseline`으로 통일했습니다. 미사용 자동 생성 함수(`kakaoQuickRepliesForText`, `kakaoDefaultQuickReplies` 노출부)를 정리했습니다.
- **P0-3 정식 용어 남은예산**: 사용자 노출 영역(명령 카탈로그·OpenBuilder 안내 페이지·예산 가이드 문구·웹 설명)에서 `오늘예산`을 0건으로 만들었습니다. 파서의 호환 별칭(입력 시 예산 응답)은 노출 없이 유지합니다.
- **P0-4 가계부 관리 안내형 제한**: `가계부 만들기(+이름)`, `가계부 참여(+코드)`, `가계부 전환/목록`은 데이터 변경 없이 `kakaoHouseholdManageGuideText`로 안내만 반환하며 대표 주소 `/my/households` 1개만 포함합니다. `단톡방 연결 + 초대코드`는 기존 지원 기능으로 유지했습니다. 직접 처리 코드(웹 `/my/households`의 생성·참여)는 웹 경로에서 그대로 사용합니다. 단독 `가계부 만들기`/`가계부 참여` 발화 인식 매처(`isHouseholdCreateGuideCommand`/`isHouseholdJoinGuideCommand`)를 추가했습니다.
- **P0-5 URL 정책**: V21.4.1의 "일반 응답 URL 0개, 링크 요청만 대표 주소 1개"를 유지하고, 가계부 관리 안내·온보딩 안내는 웹 이동 요청 성격이므로 `/my/households` 1개를 허용합니다. `초대코드` 응답에서 직접 참여 명령(`가계부 참여 코드`) 안내를 제거하고 단톡방 연결·웹 참여로 교체했습니다.
- **P0-6/P0-9 XLSX 재작성**: `KAKAO_OPENBUILDER_BLOCK_MAP_V21_4_4.xlsx`를 새로 생성했습니다.
  - 업무 블록 12개 + 시스템 블록 3개 분리 집계, 온보딩 순서 배치
  - 발화 111개, 한 행에 한 발화, 중복 0건 (생성 시 자동 검사)
  - 전 발화에 대응 파서 함수·예상 응답·부작용(DB 쓰기) 명시 (P0-7 합격 조건)
  - 파서가 없는 별칭(`모임비`, `여행 정산`, `카테고리별 지출`, `항목별 지출`, `오늘예산`) 제거
  - `적용 상태`/`테스트 결과`/`우선순위` 드롭다운, 배포 스냅샷 ID·작업자·확인 일시·실제 결과 열, 맑은 고딕 통일
- **P0-7 파서 정합**: XLSX 등록 발화 전부가 코드 매처와 1:1 대응합니다. `기록예시`, `가계부 연결` 등 문서에 있던 미인식 발화는 매처에 추가했습니다. `정산`은 V21.4.1의 1/N 정산 응답을 유지합니다.
- **P0-8 버전 정리**: `APP_VERSION`=`VERSION.txt`=보고서=XLSX = `V21.4.4-BASELINE-ALIGNMENT-HOTFIX`. 로드맵 문서(`KAKAO_CHATBOT_IMPROVEMENT_ROADMAP_V1.md`)를 패키지에 포함했습니다.

## 추가 — V21.4.1-CHAT-RESPONSE-POLICY-HOTFIX 번들 검토·병합
업로드된 `chat-response-policy` 번들을 기존 기준본(`kakao-response-routing` 번들)과 전수 대조했습니다.

- **이미 반영되어 있던 것** (routing 번들이 같은 문제를 더 넓게 수정한 병렬 변종):
  - `내 예산`/`내예산` → 예산 조회 라우팅 (기대 응답 `💰 {월} 남은 예산 / 월 예산 / 사용 금액 / 남은 예산` 포맷 일치 확인)
  - `\n` 문자 노출 수정, `정산`의 월 요약 분리(1/N), 일반 응답 URL 제거, quickReplies 기본 미부착
- **이번에 포팅한 것** (policy 번들 쪽 문구가 확정 기대 응답):
  - 미연결 단톡방 안내: 예시 `단톡방 연결 ABC123` 한 줄 + "초대코드는 가계부 관리자에게 확인해주세요." 문구로 정렬
  - `정산` 구성원 표기: "이름: N원 냄 · 받을 예정" → **"이름: N원 받기 / 보내기"** 간결 포맷, `구성원별` 헤더 제거, 송금 제안은 송금이 있을 때만 표시
- **반영하지 않은 것** (policy 번들이 더 오래된 부분 — 회귀 방지):
  - quickReplies `enabled` 라벨/기본 부착, `오늘예산` 노출, 요약·미인식 응답의 URL 자동 첨부, `링크` 응답의 다중 URL(4개), URL 스트리핑 엔진 부재
- 검증: 모킹 시나리오(백인남 120,000원 지출·2인)에서 세 기대 응답이 예시와 동일하게 출력됨을 확인. smoke_v2144(32건)·smoke_v2141 기능 검사 전체 통과.
