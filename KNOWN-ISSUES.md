## V22.8.18 UI·UX 4단계

- 첨부 v4는 독립 데모이므로 거래·예산·참여자를 localStorage에 저장합니다. 운영본은 이 저장 방식을 적용하지 않고 기존 서버 DB와 권한 검사를 유지합니다.
- 홈은 빠른 입력과 최근 기록을 한 화면에서 제공하므로 다른 사용자 화면의 그룹형 사이드바 대신 간결한 홈 전용 사이드바를 사용합니다. 폭·색·상태 규칙은 4단계 체계와 동일합니다.
- `/assets/accountbook-shell-v22818.css`는 immutable입니다. 중간 개발본을 같은 주소로 연 로컬 브라우저에서는 캐시 삭제 또는 새 시크릿 창으로 확인합니다.
- 자동 브라우저 점검은 1440px 데스크톱과 390px 모바일의 라이트·다크를 확인했지만 iPhone Safari와 Android Chrome 실기기를 대신하지 않습니다.

## V22.8.10 홈 성능 측정 범위

- 자동 측정값은 고정 QA 데이터에서 DB 요청 수와 생성 HTML 크기를 비교한 결과입니다. 실제 체감 시간은 Supabase 리전, 운영 데이터 행 수, Cloudflare 위치, 기기와 네트워크에 따라 달라집니다.
- 공통 CSS·JavaScript는 첫 방문에 한 번 내려받아야 합니다. 이후에는 버전형 immutable 캐시를 재사용하므로 반복 진입과 새로고침에서 이득이 커집니다.
- 개인 데이터 HTML은 보안을 위해 계속 `no-store`입니다. 브라우저 캐시로 가계부 내용을 재사용하지 않습니다.
- 이번 버전에는 추측성 DB 인덱스를 추가하지 않았습니다. 운영 배포 뒤에도 느리다면 Supabase 실행 계획과 느린 쿼리 로그를 확인해야 합니다.
- 브라우저·프록시가 `/assets/mobile-home-v22810.css` 또는 `.js`를 차단하면 화면 스타일이나 입력 보조가 정상 동작하지 않을 수 있습니다. 배포 후 두 경로의 200 응답과 Content-Type을 확인합니다.

## 카카오 그룹 챗봇 beta 규격

- 제공된 그룹 챗봇 스킬 서버 개발 가이드 v1.11.1은 그룹 응답에서 QuickReplies, CommerceCard, Carousel을 지원하지 않는 것으로 정리되어 있습니다. V22.8.7부터 그룹 요청에서 이를 제거하고 V22.8.11도 같은 정책을 유지하지만, 1:1 챗봇의 일반 응답 규격과는 차이가 있으므로 카카오가 beta 규격을 갱신하면 출시 전에 다시 확인해야 합니다.
- 그룹 챗봇 사용에는 카카오의 제휴·권한 부여와 운영 채널 심사가 필요할 수 있으며 코드 배포만으로 활성화되지 않습니다.
- 챗봇은 그룹방 전체 대화를 읽지 않습니다. 챗봇에게 직접 전달된 요청만 처리합니다.
- Event API 기반 선톡과 자동 푸시는 이번 버전에 포함하지 않았습니다.
- `plusfriendUserKey`는 신규 식별자로 사용하지 않지만 기존 저장 사용자 연결을 보호하기 위해 레거시 폴백이 남아 있습니다. 실제 기존 데이터 마이그레이션 계획 없이 제거하면 안 됩니다.

## 카카오톡 사진 영수증

카카오톡 대화방에 올린 사진을 챗봇이 직접 OCR 하는 기능은 현재 제공하지 않습니다. 사진을 기기에 저장한 뒤 웹의 `영수증 등록`에서 `앨범에서 선택`을 사용하거나, 전자영수증 문자를 복사해 붙여넣어야 합니다.

## OCR 네트워크와 기기 차이

- 첫 OCR 실행은 고정 버전 Tesseract와 한국어 데이터를 내려받기 때문에 이후 실행보다 느릴 수 있습니다.
- 네트워크 차단 또는 오프라인 환경에서는 사진 인식이 시작되지 않으며 직접 입력으로 대체할 수 있습니다.
- HEIC 디코딩 지원은 브라우저와 OS에 따라 다릅니다. 실패하면 사진 화면을 캡처해 JPG·PNG로 다시 선택합니다.
- 90초 안에 완료되지 않으면 자동 중단되며 사진을 더 가까이 찍거나 해상도를 낮춰 다시 시도합니다.

## 앨범·카메라 권한

권한 창과 앨범 선택 화면은 운영체제와 브라우저가 제공합니다. 서비스는 사용자가 해당 버튼을 눌렀을 때만 접근을 요청하며, 거부된 권한은 기기 설정에서 다시 허용해야 할 수 있습니다.

## 카카오 로그인 KOE006

KOE006은 배포 코드만으로 해결되지 않을 수 있습니다. 카카오 개발자 콘솔의 앱, REST API 키, 정확한 Redirect URI, 활성화 상태가 실제 인가 요청과 일치하는지 확인해야 합니다. Client Secret 값은 소스나 ZIP에 포함하지 않습니다.

## 카카오 영구 삭제 재인증

- V22.8.9는 기존 카카오 로그인 설정과 같은 Redirect URI를 사용하고 `prompt=login`으로 로그인 화면을 다시 요청합니다.
- 카카오 계정 세션·브라우저 정책·인앱 브라우저 상태에 따라 화면 전환 방식이 달라질 수 있으므로 실제 PC·iPhone·Android에서 확인해야 합니다.
- 다른 카카오 계정으로 인증하면 현재 계정과 ID가 달라 삭제가 차단됩니다.
- 취소·만료·네트워크 실패 시 가계부는 변경되지 않으며 다시 본인 확인해야 합니다.
- 카카오 로그인이 운영에서 꺼져 있거나 설정이 불완전하면 `내 계정·보안`에서 계정 로그인 비밀번호를 설정해 삭제를 확인할 수 있습니다.

## 삭제와 수동 연결해제 안내 구분

V22.8.9는 새 개인정보·감사 데이터를 저장하지 않기 위해 삭제된 연결과 사용자가 직접 해제한 연결을 서버 응답에서 하나의 안전 문구로 안내합니다. 두 경우 모두 기록은 저장하지 않습니다. 운영 문의에서 정확한 사유 구분이 실제로 필요하다고 확인되기 전에는 장기 이력 저장을 추가하지 않습니다.

## 배포 주소

Sites 배포 주소와 `ttokttok-accountbook.com` Cloudflare 운영 도메인은 별도 배포 대상입니다. Sites에서 검증된 코드가 운영 도메인에 자동 반영된다고 가정하지 않습니다.
