# V21.5.1 카카오 스킬 지연 핫픽스 보고서

## 버전
- 내부 버전: `V21.5.1-KAKAO-SKILL-LATENCY-HOTFIX`
- 기준 버전: `V21.5-GUIDED-ONBOARDING-BUDGET-BUNDLE`

## 확인된 실제 장애
운영 채널의 `폴백 블록`에서 카카오 오류 코드 `(1001)`과 함께 스킬 서버 타임아웃이 발생했다.
`/health`는 V21.5로 정상 응답했고 동일 스킬의 대표 블록은 동작했으므로 URL·DNS·인증서보다는 자연어 거래 저장 경로의 처리시간이 원인이었다.

V21.5 자연어 거래 경로는 사용자·가계부·권한·그룹 연결·단계 상태·수정 상태·분류·결제수단·중복 검사·저장·번호·별칭·예산 피드백·예비비 알림을 순차 조회했다. 지연 모의시험에서 기존 코드는 Supabase 요청당 300ms 조건에서 단일 자연어 거래 응답에 약 6.1초가 걸려 카카오 제한시간을 넘었다.

## 변경 내용
1. 그룹 멘션 `@똑똑한가계부`를 Worker에서도 안전하게 제거한다.
2. `기록`, `/기록`, `입력`, `/입력` 접두어 뒤의 거래 내용을 동일하게 처리한다.
3. 금액과 거래 내용이 명확한 자연어 입력은 단계 상태·수정 상태 DB 조회를 생략하는 빠른 거래 경로를 사용한다.
4. 회원 가계부 목록과 그룹방 연결 가계부를 병렬 조회하고 기존 회원 목록의 권한을 재사용한다.
5. 사용자 카테고리와 결제수단 설정을 병렬 조회한다.
6. 저장 후 번호·표시명 조회를 제한시간 안에서 병렬 처리한다.
7. 예산 피드백·예비비 알림을 거래 저장 응답의 필수 경로에서 제거한다. 해당 정보는 `남은예산`과 요약에서 조회한다.
8. 여러 건 저장 시 건별 번호 조회를 순차 실행하지 않고 저장 결과를 즉시 반환한다.
9. `/skill-ops` 이벤트에 요청 수신과 `latency_ms`를 기록하고 응답 헤더 `x-kakao-skill-latency-ms`를 추가한다.

## 유지한 범위
- OpenBuilder 시나리오명·블록명·발화 변경 없음
- `/skill` URL 변경 없음
- Supabase 테이블·컬럼 변경 없음
- 가계부 권한·그룹 연결·중복 저장 방어 유지
- 단계형 온보딩·예산 설정·내 이름 설정 유지
- 일반 응답 링크 및 quickReplies 정책 유지

## 지연 모의시험
- 기존 V21.5, Supabase 요청당 300ms: 단일 자연어 거래 약 `6,151ms` → 카카오 타임아웃 위험
- V21.5.1, 요청당 300ms: 직접채팅 약 `2,772ms`, 그룹채팅 약 `3,316ms`
- V21.5.1, 요청당 450ms: 직접·그룹·`/기록` 입력 모두 약 `3.9초`

지연이 큰 상황에서는 저장은 완료하고 번호·별칭 보강이 제한시간 안에 끝나지 않으면 `오늘 기록`에서 수정 번호를 확인하도록 안내한다.
