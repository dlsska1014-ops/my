# V22.7 최종 적용 체크리스트

## 배포 전

- [ ] Worker 코드와 Supabase 운영 데이터를 백업했다.
- [ ] V22.6.8 조회 전용 무결성 감사를 실행했다.
- [ ] 중복 설정·참여행, 고아 거래, 소유자 없음·다수를 검토했다.
- [ ] `schema_v22_6_8_operations_integrity.sql` 적용 상태를 확인했다.
- [ ] 밈·카드 실적 관련 공개 플래그와 `INCOMPLETE_FEATURE_QA_ENABLED`를 운영에서 제거하거나 0으로 유지했다.

## DB와 Secret

- [ ] `schema_v22_7_0_auth_atomicity.sql`이 오류 없이 COMMIT됐다.
- [ ] 5개 보안·감사 테이블이 생성됐다.
- [ ] 13개 함수가 존재하고 `service_role`만 실행할 수 있다.
- [ ] `SUPABASE_URL`과 `SUPABASE_SERVICE_ROLE_KEY`를 확인했다.
- [ ] `ADMIN_SESSION_SECRET`, `USER_SESSION_SECRET`, `ADMIN_API_TOKEN`, `MY_IMPORT_TOKEN_SECRET`을 서로 다른 운영 고유값으로 설정했다.
- [ ] `ADMIN_PASSWORD`는 최초 DB 비밀번호 설정 전 부트스트랩 용도로만 사용한다.

## 기본 상태

- [ ] `/health`가 `V22.7.0-AUTH-ATOMICITY-STABILITY`를 반환한다.
- [ ] `/ready`가 200과 `ready: true`를 반환한다.
- [ ] `/final-release`와 `/ops-audit`의 필수 게이트가 정상이다.
- [ ] `/meme-lab`, `/meme-archive`, `/card-benefits`가 404다.

## 인증·복귀

- [ ] 로그아웃 상태에서 보안 설정을 열면 홈으로 튕기지 않고 인증 화면이 보인다.
- [ ] 틀린 관리자 비밀번호 뒤에도 대상 경로가 유지된다.
- [ ] 올바른 비밀번호 뒤 원래 보안 설정의 쿼리와 앵커까지 복귀한다.
- [ ] 개인 비밀번호 저장 뒤 원래 가계부 옵션으로 복귀한다.
- [ ] 존재하지 않는 로그인 이름과 틀린 비밀번호로 사용자 행이 생성되지 않는다.
- [ ] 비밀번호 변경 후 이전 세션이 만료되고 현재 세션은 갱신된다.

## 가계부 생성·참여·삭제

- [ ] 새 가계부에서 이름·내 표시명·개인 비밀번호를 확인한 뒤 초대 단계로 이어진다.
- [ ] 카카오 최초 표시명이 카카오 닉네임으로 채워진다.
- [ ] 한 계정이 여러 가계부를 생성하고 전환해도 목록·현재 선택이 섞이지 않는다.
- [ ] 서로 다른 단톡방 두 곳을 서로 다른 가계부에 연결해도 기록 범위가 섞이지 않는다.
- [ ] 초대코드 참여가 pending/승인/차단 권한을 올바르게 적용한다.
- [ ] 틀린 가계부 이름이나 비밀번호로 삭제되지 않는다.
- [ ] 올바른 확인 뒤 테스트 가계부와 관련 데이터만 삭제된다.

## 기록·지출자·정산

- [ ] 기록 목록에 수입자·지출자 이름이 보인다.
- [ ] 수정/삭제 영역에서 현재 가계부 참여자를 선택할 수 있다.
- [ ] 다른 가계부 사용자 ID를 제출하면 거부된다.
- [ ] 일반 구성원은 권한 밖의 다른 사람 기록을 수정·삭제하지 못한다.
- [ ] 지출자 변경이 분석·예산·정산에 같은 이름으로 반영된다.
- [ ] 정산 표와 공유 문구에 사용자 UUID가 보이지 않는다.

## 예산·정기거래·무료 기능

- [ ] 수입 분류와 직접입력 수입 항목의 합계가 월 수입 기준으로 표시된다.
- [ ] 분류별 예산 합계가 이번 달 예산으로 자동 산정된다.
- [ ] 월 전체 예산 직접 설정은 선택 항목으로만 동작한다.
- [ ] 정기거래에는 유효한 지출자가 필요하고 같은 월 중복 반영이 없다.
- [ ] 영수증·정기거래·고급 정산·리포트·분석·백업 기능에 결제 제한이 없다.

## 백업·가져오기

- [ ] 백업 CSV에 `지출자` 표시명이 있고 사용자 UUID는 없다.
- [ ] CSV·TSV·TXT·XLS/XLSX와 여러 시트를 소량으로 미리보기했다.
- [ ] 제목 유사어와 자연어 행이 인식되고 자동 보정 내용이 표시된다.
- [ ] 인식 실패 행마다 이유와 수정 방법이 보인다.
- [ ] 지출자 이름이 정확히 한 명과 일치할 때만 연결된다.
- [ ] 미리보기 전에는 거래가 저장되지 않는다.
- [ ] 선택한 행만 저장되고 같은 확인을 반복해도 중복되지 않는다.

## UI·실기기

- [ ] 홈·기록·입력·정산·메뉴가 모든 핵심 화면에서 같은 순서다.
- [ ] 내부 코드, 원시 DB 오류, 사용자 UUID가 일반 화면에 보이지 않는다.
- [ ] 참여자 이름 저장 완료 문구가 자연스러운 한국어다.
- [ ] 시작가이드 하단의 불필요한 비교 문구가 없다.
- [ ] iPhone Safari와 카카오 인앱브라우저에서 320~390px 화면을 확인했다.
- [ ] Android Chrome과 카카오 인앱브라우저에서 같은 흐름을 확인했다.
- [ ] 키보드가 열린 상태에서 선택·저장 버튼과 하단 메뉴가 가려지지 않는다.
- [ ] 긴 가계부 이름·참여자 이름으로 가로 넘침이 없다.
