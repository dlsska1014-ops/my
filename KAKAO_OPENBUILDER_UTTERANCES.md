# V21.5.2 적용 메모

- 시나리오명·블록명·기존 발화를 변경하지 않습니다.
- `닉네임수정`, `구성원 초대`, `예산 바꾸기`, `이번 달 얼마 썼어` 같은 자연어는 폴백 블록을 통해 `/skill`에서 처리합니다.
- 별도 OpenBuilder 블록 추가가 필요하지 않습니다.
- 폴백 블록은 `똑똑한가계부_스킬_v21`과 `스킬데이터 사용`을 유지합니다.

# Kakao OpenBuilder 적용 가이드 — V21.5

## 1. 확정 구조

- OpenBuilder: 시작·도움말·대표 명령을 `/skill`로 전달하는 얇은 라우터
- 폴백 블록: 모든 자유 발화를 반드시 `/skill`로 전달
- Cloudflare Worker: 단계 진행, 금액·날짜·분류 파싱, 권한 확인, 저장과 응답 생성
- Supabase: 가계부·구성원·거래·예산·단계 상태 저장

Skill URL:

```text
https://ttokttok-accountbook.com/skill
```

## 2. 봇 입장 블록

봇 입장 블록에서는 긴 기능 목록이나 여러 웹 주소를 노출하지 않습니다. 사용자가 `시작하기`를 누르면 Worker의 단계형 온보딩이 시작되게 구성합니다.

### 권장 고정 응답

```text
안녕하세요 😊
카카오톡에서 함께 쓰는 똑똑한가계부입니다.

처음 사용한다면 가계부 만들기부터 시작해 보세요.
이미 초대코드가 있다면 기존 가계부에 참여할 수 있어요.
```

### 버튼

```text
버튼명: 시작하기
동작: 메시지 전송
전송 문구: 시작
```

주의:

- 봇 입장 블록에는 `/my`, `/start-guide` 등 링크를 반복해서 넣지 않습니다.
- “봇만 추가하면 자동 연결된다”라고 안내하지 않습니다.
- 단톡방에서는 봇 추가 후 관리자 한 명이 초대코드로 가계부를 한 번 연결해야 합니다.

## 3. 시작 블록

블록명 예시:

```text
01_시작_온보딩
```

발화 예시:

```text
시작
시작하기
처음
처음 사용
가계부 시작
온보딩
```

설정:

- Skill 사용: 켬
- Skill URL: `https://ttokttok-accountbook.com/skill`
- OpenBuilder 고정 답변: 없음
- 파라미터·엔티티·컨텍스트: 없음

신규 사용자에게 Worker가 아래 바로연결을 제공합니다.

```text
가계부 만들기
초대코드 참여
사용법 보기
```

## 4. 도움말 블록

블록명 예시:

```text
10_도움말
```

발화 예시:

```text
도움말
/도움말
사용법
사용 방법
명령어 알려줘
뭐 할 수 있어
처음 사용하는 법
```

설정:

- Skill 사용: 켬
- OpenBuilder 고정 답변: 없음
- 카드·웹 링크·바로연결을 OpenBuilder에 직접 중복 작성하지 않음
- Worker가 사용자의 가계부 유무를 확인해 맞춤 도움말과 필요한 단계 버튼만 반환

## 5. 가계부 생성·참여 블록

### 가계부 만들기

발화 예시:

```text
새 가계부 만들기
가계부 만들기
새 장부 만들기
```

### 초대코드 참여

발화 예시:

```text
초대코드로 참여
초대코드 참여
가계부 참여
참여하기
```

두 블록 모두 `/skill`에 연결합니다. 종류 선택, 이름 입력, 초대코드 입력 등의 다음 단계는 Worker가 처리합니다.

## 6. 예산 설정 블록

블록명 예시:

```text
40_예산설정
```

발화 예시:

```text
예산 설정
/예산설정
예산설정
월 예산 설정
카테고리 예산 설정
```

Worker 단계:

```text
전체 월 예산 / 카테고리별 예산 / 지난달 예산 복사
→ 카테고리 선택
→ 금액 선택 또는 직접 입력
→ 설정 확인
→ 저장
```

자연어 직접 설정도 폴백을 통해 `/skill`로 전달합니다.

```text
식비 예산설정 100만원
교통비 예산 설정 30만원
전체 예산설정 300만원
8월 식비 예산설정 80만원
```

## 8. 사용자 이름 설정 블록

발화 예시:

```text
내 이름 설정
사용자 이름 설정
결제자 설정
지출자 설정
```

사용자별 지출 합계를 위해 카카오 사용자를 가계부 표시명과 한 번 매핑합니다. 상태와 결과는 Supabase에 저장합니다.

## 9. 단톡방 연결 블록

발화 예시:

```text
단톡방 연결
단톡방 연결 ABC123
그룹방 연결 ABC123
```

안내 기준:

```text
1. 단톡방에 봇 추가
2. 가계부 관리자 한 명이 ‘단톡방 연결 초대코드’ 입력
3. 이후 해당 방에서 구성원들이 기록·조회
```

## 10. 대표 명령어 드롭다운

권장 순서:

| 순서 | 명령어 | 설명 |
|---:|---|---|
| 1 | 시작 | 가계부 만들기 또는 참여부터 안내 |
| 2 | 기록 | 내용과 금액을 이어서 입력. 예: 기록 점심 12000원 |
| 3 | 예산 설정 | 전체·카테고리별 예산을 단계형으로 설정 |
| 4 | 남은예산 | 사용액·잔여액·카테고리별 사용률 확인 |
| 5 | 오늘 요약 | 오늘 수입·지출·분류·결제자 요약 |
| 6 | 오늘 기록 | 오늘 기록 번호 확인 및 수정 준비 |
| 7 | 단톡방 연결 | 초대코드를 이어서 입력 |
| 8 | 도움말 | 현재 사용 가능한 기능 안내 |

## 11. 폴백 블록

블록명 예시:

```text
99_폴백_통합스킬
```

설정:

- 모든 미매칭 자유 발화 대상
- Skill 사용: 반드시 켬
- `/skill` 연결 필수
- OpenBuilder 자체 장문 답변 없음

다음 자유 문장이 폴백을 통해 정상 저장되는지 확인합니다.

```text
점심 12000원 국민카드
어제 병원 15000원
월급 250만원
식비 예산설정 백만원
```

## 12. quickReplies와 링크 정책

quickReplies 허용:

- 시작
- 가계부 만들기·참여
- 예산 설정
- 내 이름 설정
- 단톡방 연결 같은 명시적 설정 과정

quickReplies 미사용:

- 일반 거래 저장
- 일반 요약·남은예산 조회
- 미인식 안내
- 오류·중복 방어 안내

홈페이지 링크는 가계부 생성 완료, 예산 설정 완료, 첫 상세 요약처럼 가치가 분명한 전환 지점에서만 1개 노출합니다. 같은 종류의 링크는 사용자·가계부별 24시간 동안 반복하지 않습니다.

## 13. 배포 후 필수 테스트

```text
시작
새 가계부 만들기
초대코드로 참여
예산 설정
식비 예산설정 100만원
남은예산
내 이름 설정
점심 12000원 국민카드
오늘 요약
어제 요약
단톡방 연결
도움말
```

확인 항목:

- 실제 줄바꿈이 보이고 `\\n` 문자가 노출되지 않음
- 일반 거래 저장 응답에 웹 링크·quickReplies가 없음
- 미인식 문장이 일반 도움말로 잘못 라우팅되지 않음
- `/정산`이 월 요약이 아닌 정산 응답을 반환
- 폴백 블록이 모든 자유 발화를 `/skill`로 전달
