# Kakao OpenBuilder 발화/블록 설정 가이드

## Skill URL
```text
${PUBLIC_BASE_URL}/skill
```

## 가장 중요한 원칙
카카오톡에서 `이해하기 어려워요`가 나오면 대부분 Worker 코드가 아니라 OpenBuilder에서 해당 발화가 `/skill`로 전달되지 않는 상태입니다.

특히 **폴백 블록**도 반드시 Skill URL로 연결해야 합니다.

## 응답 링크 정책 (V21.4.1)
- 일반 응답에는 URL을 자동으로 넣지 않습니다.
- 사용자가 `링크` 또는 `주소`를 명시적으로 요청한 경우에만 대표 주소 1개를 보냅니다.
```text
https://ttokttok-accountbook.com
```
- `/my`, `/start-guide` 등 여러 URL을 한 응답에 반복하지 않습니다.
- quickReplies는 기본 미사용이며 특정 응답에서 꼭 필요할 때만 명시적으로 추가합니다.

## V21.4.4 대표 명령어(드롭다운) 등록 — 관리자센터

경로: 카카오 챗봇 관리자센터 > 그룹 챗봇 설정 > 대표 명령어/설명
등록 순서 그대로 노출되므로 아래 순서를 지킵니다. **처음 시작 순서(가계부 만들기 → 참여 → 기록 → 조회 → 수정 → 도움말)대로 배치합니다.** 설명은 3건만 등록합니다.

| # | 명령어(버튼명) | 설명(버튼 사용 설명) |
|---|---|---|
| 1 | 가계부 만들기 | (설명 없음) |
| 2 | 단톡방 연결 | 초대코드를 이어서 입력 예) 단톡방 연결 ABC123 |
| 3 | 기록 | 내용·금액을 이어서 입력 예) 기록 점심 12000원 국민카드 |
| 4 | 오늘 기록 | (설명 없음) |
| 5 | 요약 | (설명 없음) |
| 6 | 남은예산 | (설명 없음) |
| 7 | 수정 | 번호와 바꿀 값을 이어서 입력 예) 수정 01번 금액 13000원 |
| 8 | 도움말 | (설명 없음) |

※ V21.4.4 변경: 온보딩 순서 도입으로 `가계부 만들기`가 1번에 들어가면서 8개 제한에 따라 `정산`은 드롭다운에서 제외했습니다. `정산` 발화 자체(블록·파서)는 그대로 동작합니다.

규칙 준수 체크:
- [ ] 직관적 명령어(가계부 만들기/오늘 기록/요약/남은예산/도움말)에는 설명을 달지 않았다
- [ ] 설명 문구는 "명령어가 이미 입력창에 입력된 상태에서 이어서 쓸 내용" 기준으로 작성했다
- [ ] 명령어 8개 이하로 유지했다 (드롭다운 과밀 방지)
- [ ] 금지 표현(대화 열람/학습 오인 문구) 없음
- [ ] 반영 확인: 채팅방 재진입 후 최대 1시간 대기, 즉시 확인은 봇 제거→재추가
- [ ] 폴백 블록은 기존대로 반드시 Skill 연결 유지

## V21.4.4 처음 시작(온보딩) 정책
- 가계부가 없는 사용자가 기록·조회를 시도하면 예시 대신 **1단계 가계부 만들기 → 2단계 참여 → 3단계 기록** 순서를 안내합니다. (자동 가계부 생성 중단)
- `가계부 만들기` / `가계부 참여` / `가계부 전환`은 카카오톡에서 직접 처리하지 않고 웹 대표 주소 1개(`https://ttokttok-accountbook.com/my/households`)만 안내합니다. (P0-4, V21.5 재설계 전까지)
- `단톡방 연결 + 초대코드`는 기존대로 그룹방에서 직접 처리합니다.
- 블록/발화의 단일 원본은 `KAKAO_OPENBUILDER_BLOCK_MAP_V21_4_4.xlsx`입니다 (12업무 블록 + 시스템 블록 분리 집계).
