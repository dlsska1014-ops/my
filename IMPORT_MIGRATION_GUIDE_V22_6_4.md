# V22.6.4 다른 가계부 자료 옮기기

## 권장 순서

1. 현재 똑똑한가계부 데이터를 CSV로 백업한다.
2. 다른 가계부·은행·카드사에서 CSV 또는 XLSX를 내려받는다.
3. `백업·가져오기`에서 저장 대상 가계부를 다시 확인한다.
4. 파일을 선택하거나 표·자연어 목록을 붙여넣는다.
5. `1. 분석 미리보기`를 누른다. 아직 저장되지 않는다.
6. 제목 매핑, 중복 제외, 예상 합계, 자동 보정과 제외 이유를 확인한다.
7. 저장할 행만 선택하고 `선택한 항목 저장`을 누른다.
8. 저장 결과에서 완료·중복·오류 건수를 확인한다.

## 인식하지 못한 경우

결과에 행 번호, 이유, 해결 방법, 원문 미리보기가 표시된다.

| 이유 | 수정 방법 |
|---|---|
| 날짜 없음 | 날짜·거래일 열을 추가하거나 `2026-07-15`처럼 입력 |
| 날짜 형식 오류 | `2026-07-15`, `2026.7.15`, `7월 15일` 형식 사용 |
| 금액 없음 | 금액·지출·수입·입금·출금 열을 추가 |
| 금액 후보 여러 개 | 잔액·누계 열을 빼고 실제 거래금액 제목을 명확히 지정 |
| 열 개수 불일치 | 쉼표가 든 내용을 큰따옴표로 감싸거나 탭 표로 붙여넣기 |
| 수입·지출 동시 존재 | 한 행에는 수입 또는 지출 하나만 남기기 |
| 중복 | 기존 기록과 같은 날짜·금액·내용인지 확인 |
| 처리 한도 초과 | 월별·시트별로 나누어 다시 분석 |

원본 파일과 미인식 행은 자동 수정하거나 저장하지 않는다. 미리보기는 30분 뒤 만료되므로 오래 검토했다면 파일을 다시 분석한다.

## 자연어 예시

```text
어제 점심 12000원 국민카드
7월 14일 월급 250만원
거래일: 2026-07-10 | 사용액: 9,900원 | 거래처명: 약국 | 결제계정: 삼성카드
```

## 운영자 권장값

```text
MY_IMPORT_LIMIT=120
MY_IMPORT_MAX_BYTES=5242880
USER_SESSION_SECRET=<충분히 긴 운영 고유값>
MY_IMPORT_TOKEN_SECRET=<가져오기 토큰 전용 운영 고유값>
```

`MY_IMPORT_TOKEN_SECRET`을 생략하면 사용자 세션 Secret을 사용한다. 운영에서는 별도 Secret 사용을 권장한다.
