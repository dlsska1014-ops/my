- 성공한 새 참여는 pending 유지
- pending이 기본 가계부로 선택되지 않음
- pending 명시 접근은 403 상태 안내, 초대코드 미노출
- viewer 백업 화면은 읽기 전용이며 가져오기 입력 비활성
- viewer 설정 변경 화면은 403과 권한 요청 안내
- 생성·참여 화면의 중복 제출 방지 스크립트 렌더
- owner/admin/member/viewer/pending/blocked 읽기·쓰기 매트릭스

## 검증 한계

- Supabase는 자동시험용 모의 응답을 사용했다.
- 실제 운영 RLS, 데이터 이상, 네트워크 재시도는 운영 환경에서 확인해야 한다.
- XLS/XLSX는 브라우저 SheetJS 변환에 의존하므로 실제 네트워크와 암호화·손상 파일을 확인해야 한다.
- 서로 다른 Worker 인스턴스의 완전 동시 동명 생성은 DB idempotency 없이 100% 보장할 수 없다.
- 프리미엄 결제·구독·entitlement는 구현하지 않았고 유료 출시 검증 대상이 아니다.
