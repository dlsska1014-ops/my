# V22.4.1 계정·가계부 안정성 핫픽스

## 확인된 원인

1. `/households` 참여자 카드의 8자리 문자열은 초대코드가 아니라 `users.id` 앞 8자리였습니다.
2. 카카오 챗봇 `botUserKey`, 카카오 웹 로그인 ID, 백업 로그인이 별도 사용자 행으로 생성될 수 있었습니다.
3. `ensureSelectedHouseholdRole()`이 화면 조회 중 `ensureOwnerMembership()`을 호출해 참여자 역할을 생성·승격할 수 있었습니다.
4. 구형 `ensurePrimaryHousehold()`에는 명시적 생성 요청 없이 가계부를 자동 생성하는 코드가 남아 있었습니다.
5. 역할 변경 화면은 기존 소유자가 있어도 다른 참여자를 소유자로 바꿀 수 있었습니다.

## 수정 내용

- 일반 사용자 화면에서 내부 사용자 ID 일부를 제거하고 계정 유형만 표시
- 초대코드는 거래 저장과 무관하며, 생성·명시적 재발급에서만 바뀐다는 안내 추가
- 화면 열기·가계부 선택 시 역할을 생성하거나 소유자로 승격하지 않도록 변경
- 가계부 생성은 명시적 사용자 생성 요청에서만 실행
- 사용자 식별값이 없을 때 payload 전체 해시로 임시 사용자를 만드는 동작 제거
- `botUserKey`를 최우선으로 사용하고 `user.type=botUserKey`일 때만 `user.id` 사용
- 기존 로그인 연결 정보가 있으면 신규 사용자 생성 전 연결 계정을 우선 조회
- 두 번째 소유자 지정 및 유일한 소유자 강등·삭제 방지
- 관리자 전용 `/identity-audit` 추가
- 관리자 명시적 계정 통합 기능 추가
  - 거래 지출자 ID 이전
  - 참여 역할 병합
  - 별명·로그인 연결·단톡방 연결자 이전
  - 보조 사용자는 삭제하지 않고 통합 이력 계정으로 보존
- 기존 가계부·거래·초대코드는 계정 통합 중 삭제·재생성하지 않음
