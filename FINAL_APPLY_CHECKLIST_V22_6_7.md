# V22.6.7 최종 적용 체크리스트

## 배포 전

- [ ] 현재 Worker 코드를 별도 백업했다.
- [ ] Supabase 거래·사용자·가계부·구성원·설정·반복 규칙을 백업했다.
- [ ] `USER_SESSION_SECRET`, Supabase URL과 Service Role Key, 관리자 비밀번호를 확인했다.
- [ ] 운영에서 `INCOMPLETE_FEATURE_QA_ENABLED`가 없거나 `0`인지 확인했다.
- [ ] `MEME_CARDS_ENABLED`와 `CARD_PERFORMANCE_ENABLED`를 제거하거나 `0`으로 정리했다.
- [ ] Cloudflare Cron 설정을 기록했다.

## 배포

- [ ] `src/index.js`를 V22.6.7 파일로 교체했다.
- [ ] Worker 저장·배포가 성공했다.
- [ ] `/health`가 `V22.6.7-STABILIZATION-HIDDEN-INCOMPLETE`을 반환한다.

## 숨김 확인

- [ ] `/menu`와 `/smart-tools`에 소비 카드/밈과 카드 실적/혜택이 없다.
- [ ] `/meme-lab`, `/meme-archive`, `/share/meme`, `/card-benefits`가 404다.
- [ ] `/admin/meme/save`와 `/share/meme/like` POST가 404이며 데이터가 변하지 않는다.
- [ ] 카카오에서 `밈`을 보내면 준비 중 안내와 사용할 수 있는 명령만 나온다.

## 공개 기능 확인

- [ ] owner 계정으로 영수증을 확인 저장하고 같은 영수증 재저장을 막는지 확인했다.
- [ ] 다른 구성원이 같은 영수증을 저장해도 중복되지 않는지 확인했다.
- [ ] 반복 거래 후보를 확인 등록하고 cron을 두 번 실행해 월 거래가 한 건만 생기는지 확인했다.
- [ ] 정산 4방식 중 실제 사용하는 방식을 열고 합계와 완료 이력을 확인했다.
- [ ] 주간·월간 리포트 설정·생성·공유 문구·인쇄/PDF를 확인했다.
- [ ] viewer가 영수증 저장, 정산 완료, 리포트 설정을 변경하지 못하는지 확인했다.
- [ ] 두 가계부를 전환해 기록·리포트·정산 이력이 섞이지 않는지 확인했다.

## 모바일·운영 확인

- [ ] 실제 iPhone Safari에서 홈·메뉴·스마트 도구·개인 비밀번호를 확인했다.
- [ ] 실제 Android Chrome에서 영수증·정산·리포트를 확인했다.
- [ ] 개인 비밀번호 저장 후 원래 화면으로 돌아오는지 확인했다.
- [ ] 사업자 정보와 하단 메뉴가 겹치지 않는지 확인했다.
- [ ] 정기 거래 cron의 `failed`와 리포트 cron의 `failed`·`invalid`가 0인지 확인했다.

## 롤백 기준

- [ ] 다른 가계부 데이터가 보이거나 저장되면 즉시 이전 Worker로 롤백한다.
- [ ] 거래 중복이 반복되거나 cron 부분 실패가 지속되면 cron을 일시 중지하고 원인을 확인한다.
- [ ] 숨김 URL이 200으로 열리면 환경 변수를 확인하고 즉시 V22.6.7을 재배포한다.
