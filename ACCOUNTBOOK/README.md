## V19.8에서 새로 개선된 것 (v19.7 → v19.8) — 저장 즉시 예산 알림

카카오 매뉴얼 가드레일(우리는 먼저 신호를 줄 수 없다)에 맞춰, "밀어넣는 푸시"가 아니라
**사용자의 저장 동작에 대한 응답/화면 안에서** 예산 상태를 즉시 보여줍니다.

### 1. 챗봇 응답 예산 알림 3단계 (매뉴얼 §6)
- 챗봇으로 기록하면 그 **응답 안에** 해당 분류(또는 전체) 예산 상태를 실어 보냅니다. (별도 푸시 아님)
- 단계 구분: 📊 정상(80% 미만) · ⚠️ 주의(80~99%) · 🚨 초과(100%+, 초과 금액 표시).
- 전체 예산은 v19.4 자동 산정과 일관되게 직접 설정값 없으면 분류 합계 기준으로 계산.

### 2. 웹 저장 배너
- `/app`에서 기록을 저장하면, 그 분류가 예산 주의/초과에 걸릴 때 화면 상단에 배너 표시
  (⚠️ 주황 / 🚨 빨강). 카카오톡으로 밀어넣지 않습니다.

### 3. 가드레일 준수
- 임의 타이밍의 카카오톡 선톡은 만들지 않습니다. "먼저 알려주기"가 필요한 알림은
  향후 Event API 심사(월 2회 제한) 또는 사용자가 켜는 알림 전용 블록으로만 설계합니다.
  (docs/kakao-manual/LEARNED_SUMMARY.md 최상위 가드레일 참조)

## V19.7에서 개선된 것 (v19.6 → v19.7) — 홈 단일화 · 하단탭 재편

사용자 편의성 로드맵의 첫 단계. 매일 착지하는 홈의 정보구조(IA)를 정리했습니다.

### 1. 단일 홈 확인·정착
- `/my`는 가계부가 있으면 이미 `/app`으로 리다이렉트하고 있어, 사용자가 실제로 보는 홈은 `/app` 하나입니다.
  (뒤따르던 사이드네비 대시보드 렌더 경로는 실사용 경로에서 도달하지 않음.)
- 최근 개선(자주 쓰는 항목 칩·오늘 카드·한 줄 입력)이 모두 이 `/app` 홈에 모여 있어, 여기를 정본 홈으로 정착.

### 2. 하단탭 재편 (금융 우선)
- 기존: 입력 · 기록 · **밈** · 메뉴 (4개, 밈이 예산·분석보다 우선)
- 변경: **홈 · 기록 · ＋입력 · 예산 · 전체** (5개)
  - `＋입력`을 중앙에 원형으로 강조 — 가장 자주 하는 동작을 한 손 엄지 위치에 배치.
  - `예산`을 1급 탭으로 승격(→ /budgets), 밈카드는 하단탭에서 내리고 홈 안 카드로 유지(강등).
  - 스크롤 위치에 따라 홈/기록 탭이 자동 활성화(스크롤 스파이).

### 3. 모바일 이중 네비게이션 제거 (코덱스 지적)
- `/app`이 자체 헤더+하단바에 더해 공통 네비(renderUnifiedNav)까지 렌더해 **상단바 2개·하단바 2개**가 겹치고,
  고정 상단바 자동숨김이 sticky 헤더와 어긋나던 문제를 실제 모바일 렌더로 재현·수정.
- `/app`에서 중복 공통 네비를 제거 → 단일 헤더 + 단일 하단바로 정리. 하단바 5칸 정렬 고정.

## V19.6에서 개선된 것 (v19.5 → v19.6) — 카카오 매뉴얼 컴플라이언스

카카오 그룹 챗봇 공식 매뉴얼(beta 48장 + 스킬 서버 개발 가이드 v1.11.1)을 학습해
`docs/kakao-manual/`에 보존하고, 규칙을 코드에 반영했습니다.

### 1. 카카오 재전송 멱등 처리 (매뉴얼 §5)
- 카카오가 네트워크 타임아웃 등으로 같은 SkillRequest를 재전송해도 거래가 중복 저장되지 않도록,
  `/skill` 저장 시 **최근 120초 이내 동일 거래(재전송)만** 중복으로 합칩니다.
- 이전에는 같은 날짜의 동일 금액·내용이면 시간과 무관하게 무조건 중복 차단되어,
  "하루에 커피를 두 번 마신" 것 같은 **정당한 반복 입력이 오차단**되던 문제가 있었습니다.
  이제 재전송 창(120초)을 벗어난 반복 입력은 정상 저장됩니다.
- CSV 백업 가져오기의 중복 차단은 기존대로 시간 무관 전체 비교를 유지합니다(과거 데이터 안전).

### 2. 개인정보 보관 기간 명시 (매뉴얼 §8·§10)
- 매뉴얼은 "발화를 저장하면 저장 사실과 보관 기간을 도움말에 명시"하도록 요구합니다.
- /privacy 페이지와 챗봇 `개인정보` 응답에 **보관 기준**을 추가했습니다:
  이용 중 보관 · 웹에서 직접 삭제 가능 · 탈퇴/삭제 요청 시 보관 종료.
- 기존의 "단톡방 대화를 읽거나 학습하지 않는다"는 표현은 매뉴얼 금지 표현이 아니라
  권장 방향이므로 유지했습니다.

### 3. 코드 컴플라이언스 점검 결과 (docs/kakao-manual/LEARNED_SUMMARY.md)
- SkillResponse 미지원 타입(QuickReplies/CommerceCard/Carousel) 사용 0건 확인.
- `plusfriendUserKey`는 botUserKey→appUserId가 모두 없을 때만 쓰는 레거시 폴백으로만 존재(신규 사용 없음).
- URL 랜딩 쿼리 키만으로 권한 확정하지 않고 서버 세션 검증 유지.

## V19.5에서 개선된 것 (v19.4 → v19.5)

### 1. /app 기록 저장·수정·삭제가 드디어 일반 사용자에게 열림 (핵심 수정)
- 기존에는 /app의 빠른 입력·기록 수정/삭제·고정지출이 전부 **관리자 세션 전용 라우트**로 연결되어
  일반 로그인 사용자가 저장을 누르면 관리자 로그인으로 튕기는 구조적 문제가 있었습니다.
- 이제 서버에서 권한 매트릭스대로 동작합니다:
  owner/admin = 가계부 전체 기록 관리, member = 자기 기록만 입력/수정/삭제(지출자도 본인으로 강제),
  viewer/pending/blocked = 차단. 수정/삭제는 폼 값이 아니라 **DB의 실제 row 소유권**으로 검증합니다.
- 고정지출 추가/반영/삭제는 owner/admin만 가능. UI에서도 권한 없는 사용자에게 폼을 숨깁니다.

### 2. 한 줄 스마트 입력 (챗봇 문법을 웹에서도)
- /app 빠른 입력 맨 위에 "점심 12000 국민카드"처럼 한 줄로 쓰면
  금액(만/천 단위 축약 지원: 5천, 2만, 1만5천, 250만원), 내용, 결제수단(등록된 결제수단 자동 매칭),
  수입/지출(월급·급여 등 키워드)이 자동으로 채워집니다.
- 분류를 비워두면 서버가 카카오 챗봇과 같은 규칙으로 **분류를 자동 추론**해 저장합니다.

### 3. 분석 → 기록 드릴다운 (뱅크샐러드 스타일)
- /analysis 분류별 지출 랭킹의 분류명 클릭 → 해당 분류의 기록 목록으로 이동
- /analysis·/my/analysis 일별 소비 그래프의 날짜 카드 클릭 → 그날 기록으로 이동
- /payment-methods 결제수단명 클릭 → 해당 결제수단 기록으로 이동
- /calendar는 기존 날짜별 기록 섹션(앵커 이동 + 권한별 수정/삭제) 유지 확인
