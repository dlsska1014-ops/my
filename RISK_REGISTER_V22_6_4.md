# V22.6.4 위험 등록부

| 위험 | 현재 대응 | 잔여 조치 |
|---|---|---|
| 잘못 인식한 행 즉시 저장 | 미리보기와 선택 저장 | 실제 파일 표본 확대 |
| 미리보기 토큰 변조 | HMAC 서명, 사용자·가계부·만료 검증 | 운영 전용 Secret 설정 |
| 저장 버튼 재전송 | 작업키·결정적 거래 UUID·중복 재검사 | 운영 동시 제출 시험 |
| 다른 참여자에게 잘못 귀속 | 정확한 닉네임 한 명만 매핑, 권한 재검사 | 동명이인 UX 개선 |
| 대형 파일로 HTML·토큰 증가 | 파일·행·토큰 크기 제한 | 대규모 이전은 서버 작업 큐 검토 |
| 운영 이벤트에 민감 원문 노출 | 사유 코드와 건수만 집계 | 장기 저장 시 보관 정책 수립 |
| 인스턴스 로그 소실 | 운영 대시보드 실시간 확인 | 장기 지표 테이블 별도 설계 |
| SheetJS CDN 장애 | CSV 저장·붙여넣기 대체 안내 | 자체 호스팅 또는 서버 변환 검토 |
| 브라우저 표 가로 넘침 | 표 컨테이너 스크롤 | iOS·Android 실기기 확인 |
| 운영 RLS 오구성 | 애플리케이션 역할 재검사 | 운영 Supabase RLS 감사 필수 |
| 유료 권한 미구현 | 결제 미연결·무료 베타 유지 | entitlement·결제 운영 별도 개발 |

## 배포 차단 조건

- 미리보기만 수행했는데 거래가 저장됨
- 다른 가계부 또는 다른 사용자 토큰으로 저장 가능
- 같은 저장 요청 재전송으로 중복 거래 생성
- viewer·pending·blocked가 저장 가능
- 미인식 행이 이유 없이 누락되거나 임의 저장
- 운영 RLS에서 다른 가계부 데이터가 노출
