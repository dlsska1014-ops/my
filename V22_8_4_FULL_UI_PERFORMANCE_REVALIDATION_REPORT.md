# V22.8.4 전체 UI·성능 재검증 보고서

## 결론

사용자가 지적한 홈의 과도한 파란 면, 영수증의 보라색·회색 저명도, 키워드 편집기의 거대한 카드 배치를 제거했습니다. 동시에 첫 화면을 느리게 만들던 순차 데이터 조회와 무거운 브라우저 모듈의 선로딩을 줄였고, 영수증 분석이 브라우저에서 실행되지 않던 숨은 스크립트 오류를 수정했습니다.

## 주요 개선

| 영역 | 이전 문제 | V22.8.4 개선 |
|---|---|---|
| 홈 타임라인 | 행 전체가 파란색이고 작은 글자와 금액 구분이 어려움 | 흰색 행, 중립 아이콘, 초록 수입·빨강 지출, 명확한 구분선 |
| 빠른 입력 | 선택지가 거대한 파란 버튼으로 이어짐 | 내용에 맞는 작은 흰색 칩과 명확한 선택 상태 |
| 알림 | 검은 배경과 장식이 정보보다 강함 | 옅은 민트 상태 카드와 읽기 쉬운 본문 |
| 영수증 | 보라색 업로드 영역과 저명도 안내 | 중립 점선 영역, 남색 선택 버튼, 단계별 상태 안내 |
| 영수증 분석 | 템플릿 문자열에서 정규식이 깨져 분석 버튼이 실행되지 않음 | 브라우저 함수를 안전하게 직렬화하고 상호작용 검증 추가 |

| 백업 | 화면 진입 즉시 엑셀 변환기 다운로드 | XLS/XLSX를 실제 선택할 때만 SheetJS 로드 |
| OCR | 화면 진입 즉시 OCR 엔진 다운로드 | 사진 선택 시에만 Tesseract 로드 |
| 데이터 조회 | 핵심 홈 데이터가 여러 차례 순차 대기 | 독립 조회를 병렬 실행하고 중복 호출·런타임 축소 |

## 성능 비교

동일한 QA 데이터와 각 백엔드 호출에 20ms 지연을 부여한 비교에서 홈 응답 준비 시간이 598.2ms에서 242.4ms로 줄어 약 59% 개선되었습니다. 최대 동시 조회 수는 2에서 8로 늘고 백엔드 호출은 22회에서 21회로 줄었습니다.

이 수치는 실제 운영 회선의 사용자 체감 시간을 보장하는 현장 지표가 아니라, 코드 변경 전후의 대기 구조를 같은 조건에서 비교한 결과입니다. 운영에서는 실제 로그와 브라우저 네트워크 패널로 다시 확인해야 합니다.

## 적용과 롤백

- 신규 데이터베이스 마이그레이션은 없습니다.
- `src/index.js`만 교체하며 기존 환경변수와 Secret을 유지합니다.
- 문제 발생 시 직전 Worker 코드로 롤백합니다. V22.8.4 때문에 SQL을 롤백하지 않습니다.
- 카카오 `KOE006`은 카카오 관리자센터의 Web domain·Redirect URI·REST API 키가 실제 요청과 다를 때 외부에서 발생할 수 있으므로 별도 운영 대조가 필요합니다.
