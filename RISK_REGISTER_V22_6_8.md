# V22.6.8 위험 등록부

| 위험 | 영향 | 현재 완화 | 배포 후 확인 |
|---|---|---|---|
| 운영 DB에 기존 중복이 있음 | 고유 인덱스 적용 실패 | 조회 전용 감사와 예외 중단, 자동 삭제 금지 | 감사 결과와 정리 근거 보관 |
| DB RPC 미적용·스키마 캐시 지연 | 여러 Worker 사이 잠금 미작동 | 같은 인스턴스 임시 잠금과 `memory-fallback` 경고 | Cron `lock_mode=database` 확인 |
| Worker가 lease 보유 중 종료 | 일시적 자동화 지연 | 만료 가능한 lease, 기본 15분 뒤 재시도 | `locked_until`과 처리 시간 확인 |
| 실제 작업이 lease보다 오래 걸림 | 만료 뒤 두 번째 실행 시작 | 거래 고유 인덱스·설정 키가 최종 중복 방어 | 처리 시간 측정 후 필요할 때만 lease 조정 |
| 동일 상호·날짜·금액의 별도 영수증 | 정상 거래 오탐 가능 | 원문이 있으면 원문 지문 우선, 원문 없는 경우만 상호 보조 | 실제 영수증 표본 확인 |
| 정산 이력이 설정 JSON 한 행에 저장 | 장기 이력·복잡한 동시 편집 한계 | 가계부 단위 분산 잠금, 최근 30건 제한 | 향후 정산 전용 테이블 전환 검토 |
| 리포트 강제 재생성 | 같은 스냅샷 재계산 | 실행 중 분산 잠금, 설정 키 upsert | 운영에서 `force=1` 상시 사용 금지 |
| 고아 사용자·가계부 데이터 | 목록 누락·권한 이상 | 조회 전용 감사 제공, 자동 삭제하지 않음 | 원인별 계정 통합/소유권 복구 |
| 서비스 역할 키 노출 | 전체 데이터 접근 위험 | 서버 환경변수만 사용, RPC를 service_role 전용으로 제한 | Cloudflare Secret·Supabase 키 회전 정책 확인 |
| 모바일 브라우저·실제 폰트 차이 | 줄바꿈·키보드 가림 | 한글 폰트 360/390 자동 캡처 | iOS/Android 실기기 최종 확인 |
| 미완성 밈/카드 실적 재노출 | 불완전 기능 공개 | 전용 QA 잠금 + 직접 경로 404 | 숨김 URL 감시 |

## 다음 구조 개선 후보

정산 이력과 작업 실행 이력을 장기 운영 데이터로 쓸 경우 JSON 설정이 아니라 전용 정산/작업 테이블, 상태 전이, 실행 ID, 재처리 큐를 도입하는 것이 다음 단계입니다. 현재 버전은 기존 스키마와의 호환성을 우선해 잠금 테이블만 추가합니다.
