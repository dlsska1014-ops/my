# V22.8.1 Guided UI·UX Full Overhaul

## 결과

V22.8.0의 저장·권한·가계부 격리·카카오 NLU·Supabase 구조를 유지하면서 전체 사용자 화면에 하나의 UI 체계를 적용했다. 핵심 방향은 “처음에는 필요한 것만, 상세 기능은 필요할 때”이다.

## 주요 변경

- 공통: 차분한 파랑·민트 디자인 토큰, 얇은 테두리, 약한 그림자, 일관된 입력·버튼·카드 규칙
- 탐색: 목적 기반 사이드바, 모바일 5개 하단 메뉴, 단색 아이콘, 간결한 활성 상태
- 온보딩: 처음 3단계, 첫 기록 우선, 완료 후 자동 축소
- 자산: 자주 쓰는 6종 우선, 이름·금액 중심, 선택 정보와 공개 범위 접기
- 정기지출: 기본 정보·납부 일정·선택 정보로 재배치
- 예산: 반복 빠른 선택 제거, 핵심 지표 4개, 불필요한 하단 링크 제거
- 전체 메뉴: 핵심 3개 그룹, 추가 도구 접기
- 접근성: 자동 라벨, 포커스 표시, 모션 감소, 모바일 표 안내, 제출 중복 방지

전체 50개 감사 항목은 `UIUX_FULL_AUDIT_V22_8_1.md`에 기록했다.

## 호환성

- 데이터베이스 스키마 변경 없음
- V22.8.0에서 사용하던 Cloudflare Worker 환경변수와 Supabase SQL을 그대로 사용
- OpenBuilder Skill URL과 대표 발화 변경 없음
- 기존 공개 도메인 변경 없음
- 미완성 카드 혜택·밈 기능 비노출 유지

## 적용 파일

- `src/index.js`: Worker 전체 소스
- `smoke_v2281_guided_uiux.mjs`: 이번 UI·UX 회귀 검사
- `UIUX_FULL_AUDIT_V22_8_1.md`: 50개 개선 감사
- `FINAL_APPLY_CHECKLIST_V22_8_1.md`: 운영 반영 순서
- `VALIDATION_REPORT_V22_8_1.md`: 검증 결과
