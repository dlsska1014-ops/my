# V22.5.4 대화 상태 무결성 안정화 보고서

## 목적

사용자가 수십~수백 가지 조합을 직접 시험하지 않아도 되도록, 발견된 개별 오류를 공통 원인별로 묶어 상태 전이·권한·동시성 기준으로 수정하고 자동 회귀시험을 확장했습니다.

## 발견한 공통 원인

1. 단계형 대화 응답보다 일반 이름·거래·NLU 파서가 먼저 실행되는 우선순위 오류
2. 한국어 문자를 `\W` 기반으로 검사해 정상 이름을 기호로 오인하는 오류
3. 직접채팅과 여러 단톡방이 같은 수정·반복 상태를 공유할 수 있는 범위 오류
4. 방 연결이 단일 JSON map의 read-modify-write로 저장되어 동시 업데이트를 잃을 수 있는 구조
5. 단톡방 재연결 시 새 가계부 권한만 검사해 현재 연결을 가로챌 수 있는 권한 공백
6. 거래의 현재 지출자와 원 생성자를 동일하게 취급해 지출자 변경 후 수정 권한이 꼬이는 문제
7. 삭제·재연결처럼 영향이 큰 동작에 확인 단계가 부족한 문제

## 주요 수정

- 허용된 flow/step 및 30분 만료 검증
- 상태별 로컬 응답과 상위 명령 분리
- 생성·예산·참여·선택·별명 입력의 예약어/위험값 차단
- 한국어 이름 Unicode 검증
- direct/group별 flow·edit·repeat scope
- 단톡방 연결 전 개인 가계부 fallback 제거
- 재연결 시 현재·대상 가계부 양쪽 owner/admin 재검증
- 단톡방 연결을 `kakao_group_link_v2254:*` 방별 설정에 독립 저장하고 legacy map은 호환 미러로 유지
- 거래 원 생성자를 `source_user_key`로 보존
- 번호 기반 수정과 삭제 확인
- 최종 대표 명령어와 사용자 안내 문구 정렬
