# V22.8.6 영수증 등록 화면 최적화·안정화 보고서

기반 소스: 사용자가 제공한 `V22.8.5-MOBILE-ACCESS-MENU-RECEIPT-PERFORMANCE-STABILITY` Worker 소스(V22.8.2~V22.8.5 변경 전부 포함).

## 배경

- 저장소의 `V22.8.5` 커밋이 문서만 반영하면서 `src/index.js`와 `static_ui_qa_v2270.mjs`를 삭제해 소스가 유실된 상태였습니다.
- 사용자가 보관 중이던 V22.8.5 계열 소스를 업로드해 주어 이를 저장소에 복원하고, 그 위에 영수증 화면 안정화를 적용했습니다.
- 업로드 소스에는 이미 영수증 성능 개선(앨범/카메라 분리, OCR 지연 로딩, 기기 성능별 이미지 축소, 워커 재사용·유휴 해제, 미리보기)이 들어 있었습니다. V22.8.6은 여기에 남아 있던 안정화 공백을 메웠습니다.

## V22.8.6에서 추가된 것

### 안정화
- **OCR 90초 타임아웃 + ‘분석 취소’ 버튼**: 기존에는 인식이 멈추면 화면이 영구히 잠겼습니다(모든 버튼 비활성 유지). 이제 상한 시간 초과나 사용자 취소 시 워커를 종료하고 버튼·상태를 완전히 복구합니다. 실행 ID 가드로 취소 후 늦게 도착한 결과가 화면을 덮어쓰지 않습니다.
- **`[hidden]` 우선 CSS 규칙**: 페이지 CSS가 `hidden` 속성을 덮어써 숨김 요소가 노출되는 계열의 버그를 차단합니다.
- **QA 파서 노출(`window.__receiptParsers`)**: 과거 템플릿 리터럴 이스케이프로 정규식(`\d`, `\s`)이 깨져 자동 인식이 통째로 죽었던 회귀(V22.8.1에서 실재)를 스모크 테스트가 브라우저 문법으로 직접 검증합니다.

### 편의성
- **PC 드래그&드롭·클립보드 붙여넣기(Ctrl+V)**로 사진 선택 지원(선택까지만 — 분석은 기존 흐름대로 ‘선택한 사진 읽기’로 시작).
- **금액 쉼표 자동 표기**: 자동 인식 값과 수기 입력 모두 `35,400` 형태로 표시(커서 위치 유지). 서버는 쉼표 값을 기존대로 숫자로 저장.
- **가계부·기록 월 변경 시 자동 적용**(‘선택 적용’ 버튼은 JS 미동작 환경 대비 유지).
- 상태 영역에 정적 `aria-live="polite"` 부여(스크립트 로드 전에도 스크린리더 안내).

### 저장소 정리
- 유실됐던 `static_ui_qa_v2270.mjs`(23화면 643 단언 정적 UI QA)를 V22.8.1 커밋에서 복원.
- 테스트 16개 파일의 버전 단언을 `V22.8.6-RECEIPT-SCREEN-OPTIMIZATION`으로 갱신.
- 업로드 소스의 실제 구현과 어긋나 있던 세부 단언 2건 현행화(의도 유지):
  - `smoke_v2284` 74행: `/app` 병렬 조회 변수 목록 → 현행 `const [members, rawRows, calendar, budgets] = await Promise.all` 패턴으로 갱신.
  - `smoke_v2267` 90행: 영수증 개인정보·확인 문구 → 현행 문구(“서버에 업로드하지 않고”, “이 내용으로 저장하는 데 동의”)로 갱신.
- 신규 `smoke_receipt_screen_optimization.mjs`(42개 체크)와 `validate_v2286.sh`(신규 스모크 + 기존 V22.8.5 체인 전체) 추가.
