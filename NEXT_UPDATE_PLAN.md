# 다음 업데이트 계획

현재 기준은 `V22.8.8-PREDICTABLE-ACTION-FEEDBACK`입니다. 다음 버전은 기능 수를 늘리기보다 실제 사용 확인, 접근성, 오류 복구를 우선합니다. 사용자가 만족한 분석 화면은 별도 요청이 없는 한 계속 고정합니다.

## V22.8.9 후보 · 실기기 접근성·복귀 흐름 검증

### 목표

- PC Chrome, iPhone Safari, Android Chrome에서 로그인·가입·개인 비밀번호의 실제 입력 흐름 확인
- VoiceOver·TalkBack으로 입력 오류, 진행 중 상태, 완료 후 이동을 읽을 수 있는지 확인
- 브라우저 뒤로 가기와 비밀번호 관리자 자동 채움 뒤 버튼 상태가 정확히 복원되는지 확인
- 삭제 취소, 네트워크 오류, 서버 검증 오류 뒤 같은 form을 다시 제출할 수 있는지 확인
- 영수증의 느린 네트워크 OCR 로드·취소·재시도 상태 실기기 확인

### 예상 변경 판단

| 항목 | 현재 예상 | 변경 조건 |
|---|---|---|
| `src/index.js` | 교체 가능성 있음 | 실기기에서 재현된 UI·복귀 오류를 수정할 때 |
| SQL | 없음 | 사용자 설정을 새로 영구 저장해야 할 때만 재검토 |
| 환경변수 | 없음 | 외부 서비스나 새 운영 토글이 필요할 때만 재검토 |
| OpenBuilder | 없음 | 카카오 블록·파라미터 변경이 확인될 때만 작업 |

### 완료 조건

- 기기·브라우저별 결과를 체크리스트에 기록
- 재현되지 않은 추측성 화면 개편은 하지 않음
- 분석 소스 해시와 영수증 55개 회귀 검증 유지
- 신규 오류가 없으면 문서·실기기 검증만으로 종료 가능

## V22.9.0 후보 · 오류·빈 상태의 다음 행동 정리

- 서버 오류 뒤 해당 입력칸과 해결 방법으로 포커스 이동
- 빈 검색·빈 가계부·권한 없음 상태의 다음 행동 문구 일관성 점검
- 저장 성공 토스트와 중요한 경고 모달의 사용 기준 정리
- 중복된 확인창·상태 문구를 공통 디자인 규칙으로 통합
- 실제 사용자 문의와 운영 이벤트에서 반복되는 경로만 우선 개선

예상 배포는 화면 런타임 변경 시 `src/index.js` 교체, SQL 없음입니다. 사용자별 온보딩 상태나 오류 선호를 새로 영구 저장할 필요가 생길 때만 SQL을 별도 설계합니다.

## V22.9.1 후보 · 카카오 beta 운영 검증

- 카카오 1:1·그룹 실사용 시나리오 결과 누적
- 그룹 응답 변환 여부와 오류 경로를 개인정보 없이 기존 운영 화면에서 구분
- 최신 카카오 beta 문서와 제공된 v1.11.1 매뉴얼 차이 재확인
- 지원 output·길이·번호형 선택지 회귀 픽스처 확대

기존 메모리 기반 운영 이벤트로 충분하면 SQL이 없습니다. 장기 통계가 실제로 필요하다고 확인되면 개인정보 보존 기준과 함께 누적 SQL을 먼저 검토하며 자동으로 포함하지 않습니다.

## V23.0.0 보류 · 명시 승인 후만 진행

- Event API 선톡 또는 정기 알림
- 장기 보존형 사용자 행동·그룹 운영 통계
- A/B 실험 또는 사용자별 UX 상태 저장
- `plusfriendUserKey` 레거시 매핑 정리·마이그레이션

이 항목은 카카오 권한·심사, 개인정보 고지, 보존 기간, 기존 데이터 현황, 별도 SQL이 필요할 수 있습니다. 사용자 승인과 마이그레이션 계획 없이 도입하지 않습니다.
