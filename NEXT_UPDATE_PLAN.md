# 다음 업데이트 계획

현재 기준은 `V22.8.10-HOME-PERFORMANCE-LEAN-PACKAGE`입니다. 다음 버전도 파일을 다시 불리지 않고, 운영에서 확인된 병목만 단계적으로 줄입니다. 분석 화면, 영수증 등록, 카카오 그룹 규격, 가계부·계정 보안 경계는 재현 오류나 명시 요청이 없으면 보호합니다.

## V22.8.11 우선 · 운영 성능 계측과 DB 인덱스 판정

### 목표

- Cloudflare 실제 `/my`, `/app`, `/my/households` 응답시간의 p50·p95 확인
- Supabase에서 느린 쿼리, 테이블 행 수, 실행 계획 확인
- V22.8.10의 9회 홈 요청이 실제 운영에서도 유지되는지 확인
- 첫 방문과 캐시 재사용 방문을 분리해 측정
- 기능별 응답 크기와 오류율을 개인정보 없이 집계

### SQL 또는 index.js 판정

| 확인 결과 | 다음 조치 |
|---|---|
| 쿼리 횟수·HTML 생성이 병목 | `src/index.js`만 개선, SQL 없음 |
| `household_members`의 `user_id` 조회가 순차 스캔 | 멱등 인덱스 SQL 검토 |
| 월간 `transactions` 범위 조회가 순차 스캔 | household·date 복합 인덱스 SQL 검토 |
| DB는 빠르고 네트워크만 느림 | 정적 자원·응답 압축·리전 점검, SQL 없음 |
| 근거가 불충분 | 변경하지 않고 계측 연장 |

후보 인덱스 이름이나 SQL은 실행 계획 확인 전 확정하지 않습니다. 실제로 필요하면 `CREATE INDEX IF NOT EXISTS`, 적용 전후 `EXPLAIN`, 잠금·디스크 영향, 롤백 한계를 한 문서에 작성합니다.

## V22.8.12 후보 · 홈 2단계 로딩

첫 화면에 꼭 필요한 예산·오늘 금액·최근 기록만 먼저 보내고, 고급 필터나 긴 기록 목록은 사용자가 열 때 불러오는 방식을 검토합니다.

- 첫 화면의 주 정보와 뒤늦게 불러도 되는 정보 구분
- JavaScript가 꺼져 있어도 기본 기록 조회 가능
- 가계부 권한을 API에서 다시 확인해 다른 가계부 데이터 노출 방지
- 로딩·실패·재시도 상태를 명확히 표시
- 동일 월 데이터를 중복 조회하지 않도록 요청 캐시 경계 설계

예상 판정은 `src/index.js` 교체만이며 SQL은 없습니다. 새 API가 기존 필드로 충분하지 않을 때만 다시 판단합니다.

## V22.8.13 후보 · 소스 구조 경량화와 단일 배포 파일 자동 생성

현재 운영 편의를 위해 `src/index.js` 한 파일 배포를 유지하되, 개발 원본은 기능별 모듈로 나누고 검증된 단일 파일을 자동 생성하는 구조를 검토합니다.

- 인증·가계부·거래·카카오·화면·정적 자원 모듈 경계 정의
- 빌드 결과와 원본의 함수·라우트 누락 검증
- 분석·영수증 보호 해시를 빌드 전후 각각 확인
- 읽을 수 없는 과도한 minify보다 재현 가능한 tree-shaking 우선
- 사용자 전달 ZIP에는 최종 `src/index.js`와 현재 문서·검증만 포함

이 단계는 코드 구조 변경이므로 회귀 범위가 큽니다. 기능 추가와 같은 버전에 섞지 않습니다. SQL·환경변수·OpenBuilder 변경은 예상하지 않습니다.

## V22.9.0 후보 · 소유권 이전 후 나가기

현재 소유자는 바로 나갈 수 없습니다. 참여자가 있는 가계부에서 새 소유자를 선택하고 이전과 나가기를 원자적으로 처리하는 흐름입니다.

- owner가 정확히 한 명인지 DB 제약 감사
- pending·blocked·viewer를 새 소유자로 선택하지 못하도록 규칙 확정
- 동시 요청과 마지막 소유자 이탈 방지
- 그룹 연결 관리 권한의 연속성 확인

기존 원자적 RPC가 없으면 신규 SQL·RPC가 필요할 가능성이 높습니다. 이 경우 SQL 적용·검증 후 Worker를 교체하며 `index.js`만 먼저 배포하지 않습니다.

## 매 버전 시작 전 결정표

| 점검 | `예`일 때 | `아니오`일 때 |
|---|---|---|
| 새 DB 객체·필드가 필요한가? | 멱등 SQL·적용 순서·검증·롤백 작성 | SQL 없음 명시 |
| 느린 쿼리가 실행 계획으로 확인됐는가? | 최소 인덱스만 별도 SQL로 검토 | 추측성 인덱스 추가 금지 |
| Worker 동작이 바뀌는가? | 전체 `src/index.js` 교체 | 문서 전용 여부 명시 |
| 새 환경변수·Secret이 필요한가? | 이름·필수 여부·콘솔 작업 분리 | 변경 없음 명시 |
| Kakao Developers가 바뀌는가? | Redirect URI·동의항목 절차 작성 | 기존 설정 유지 |
| OpenBuilder가 바뀌는가? | 블록별 적용표·롤백 작성 | 기존 설정 유지 |
| 보호 화면에 영향이 있는가? | 전용 승인·회귀 검증 추가 | 기존 해시 유지 |
| ZIP에 현재 배포와 무관한 파일이 있는가? | 통합 이력 문서로 옮기고 제외 | 현재 구성 유지 |
