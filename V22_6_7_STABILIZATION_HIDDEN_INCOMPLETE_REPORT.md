# V22.6.7 공개 기능 안정화·미완성 기능 숨김 보고서

## 결론

소비 카드/밈과 카드 실적/혜택은 아직 사용자에게 제공할 완성도가 아니므로 메뉴·카카오 응답·직접 URL에서 숨겼습니다. 그 외 무료 공개 기능은 유지하면서 영수증, 반복 거래, 정산, 자동 리포트의 중복·권한·가계부 범위·부분 실패 처리를 보강했습니다.

현재 릴리스 식별자는 `V22.6.7-STABILIZATION-HIDDEN-INCOMPLETE`입니다.

## 숨긴 범위

- 소비 카드/밈: 만들기, 이미지, 보관함, 랭킹, 통계, 공유, 좋아요, 콘텐츠 운영 페이지와 관련 쓰기 API
- 카드 실적/혜택: `/card-benefits`
- 메뉴, 스마트 도구, 운영 점검 링크에서 관련 항목 제거
- 카카오에서 `밈` 요청 시 숨긴 주소를 주지 않고 이번 달 요약·오늘 기록 보기로 안내
- 모든 숨김 응답은 `404`, HTML `noindex,nofollow`, `x-robots-tag: noindex, nofollow`
- 과거 기능 플래그만 켜져 있어도 재노출되지 않음. 내부 QA에서만 `INCOMPLETE_FEATURE_QA_ENABLED=1`과 개별 기능 플래그를 함께 지정해야 열림

상세 경로는 `HIDDEN_ROUTE_AUDIT_V22_6_7.md`에 정리했습니다.

## 공개 무료 기능 안정화

### 영수증 기록

- 명시적 확인 체크, 실제 달력 날짜, 양수 금액, 쓰기 권한을 모두 확인
- 같은 가계부·날짜·금액에서 영수증 문자 또는 상호가 같은 영수증은 구성원이 달라도 중복 저장하지 않음
- 인식 실패 이유와 직접 수정 경로 유지
- 영수증 이미지는 브라우저 OCR에만 사용하며 서버 저장 없음

### 반복 거래 자동화

- owner/admin의 명시적 확정 전에는 규칙을 만들지 않음
- 같은 후보의 재제출과 같은 달 자동 거래를 중복 방지
- 가계부 목록과 반복 규칙 조회를 엄격하게 처리하고 조회 실패 시 임의의 빈 결과로 성공 처리하지 않음
- 한 가계부나 한 규칙이 실패해도 나머지를 계속 처리하고 `applied`, `deduplicated`, `skipped`, `failed`를 분리 집계
- 일부 실패 시 cron HTTP 207과 운영 이벤트를 남김
- 정기 거래 작업이 일부 실패해도 자동 리포트 작업은 계속 실행

### 고급 정산

- 동일·비율·인원수·품목별 방식의 1원 단위 합계 보존 검증
- 허용되지 않은 모드는 저장하지 않음
- 같은 가계부·월·방식·메모의 완료 요청이 5분 안에 재전송되면 한 번만 저장
- 완료 이력은 선택 가계부 안에서만 조회·저장

### 주간·월간 리포트

- owner/admin만 자동 생성 설정 변경 가능
- 주간 또는 월간 주기를 하나도 선택하지 않으면 자동 생성 비활성화
- 설정 키의 가계부 ID만 조회 범위로 신뢰하며, JSON 본문의 가계부 ID가 다르면 `invalid` 처리
- 가계부가 다른 거래·설정·스냅샷이 섞이지 않도록 격리 검증
- 인쇄·PDF 저장과 카카오톡 공유용 문구는 무료 유지

## UI·UX 결과

- 메뉴와 스마트 도구에는 현재 사용할 수 있는 기능만 표시
- 공개 기능이 무료라는 점과 준비가 끝나지 않은 기능은 숨긴다는 원칙을 명확히 표시
- 360×800과 390×844에서 각각 18개 주요 경로 렌더링
- 가로 넘침, 화면 밖 요소, 중복 ID, 입력 라벨 누락, 고정 하단 메뉴의 사업자 정보 가림, 숨긴 기능 링크 노출 모두 0건
- 개인 비밀번호 화면은 `저장하고 이전 화면으로`, `취소하고 이전 화면으로`를 유지해 경로 이탈 방지

## 남은 운영 확인

- 실제 iPhone Safari와 Android Chrome에서 각 1회 최종 확인
- 운영 Supabase 데이터로 같은 영수증 오탐 여부와 반복 거래 대량 처리 시간 관찰
- Cloudflare Cron에서 `failed` 또는 `invalid`가 0이 아닌 경우 운영 이벤트 확인
- 소비 카드/밈과 카드 실적/혜택은 별도 완성도·콘텐츠·정확도 검증 전까지 활성화 금지
