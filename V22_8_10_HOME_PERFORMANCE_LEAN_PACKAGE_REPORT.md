# V22.8.10 홈 성능·경량 패키지 보고서

## 문제 확인

V22.8.9 누적 ZIP은 과거 보고서·검증 로그·SQL·체크섬·학습 원본을 모두 한 폴더에 모아 374개 항목, 비압축 4,445,584 bytes가 됐습니다. 배포에 필요한 최신 코드와 이력 보존 자료가 섞여 찾기 어렵고 반복 파일이 많았습니다.

개인 홈은 고정 QA 데이터에서도 다음 병목이 있었습니다.

- `/my`: DB 요청 7회
- `/app`: DB 요청 16회
- 홈 HTML: 106,841 bytes
- 가계부 목록과 참여자 프로필의 N+1 조회
- settings 키별 반복 조회
- 짧은 거래 목록 뒤 빈 페이지 한 번 추가 조회
- 공통 CSS·JavaScript를 개인 HTML마다 반복 삽입

## 구현

### 데이터 요청

- 가계부 ID와 사용자 ID를 PostgREST `in.(...)`로 묶어 조회
- 홈 settings 4종을 한 쿼리로 조회
- 예산 fallback과 예산 테이블을 병렬 처리
- pagination 반환량이 요청량보다 적으면 즉시 종료
- 바로 리디렉션하는 `/my` 경로의 미사용 프로필 조회 제거

### 화면 응답

- 공통 CSS: `/assets/mobile-home-v22810.css`
- 공통 JavaScript: `/assets/mobile-home-v22810.js`
- 두 자원은 DB를 사용하지 않고 1년 immutable 캐시
- 개인 데이터 HTML은 `no-store` 유지
- CSP의 기존 `self` 정책 안에서만 로드

## 측정 결과

| 항목 | 변경 전 | 변경 후 | 감소 |
|---|---:|---:|---:|
| `/my` DB 요청 | 7 | 4 | 42.9% |
| `/app` DB 요청 | 16 | 9 | 43.8% |
| 개인 홈 HTML | 106,841 B | 24,490 B | 77.1% |

첫 방문에는 공통 CSS 약 51.7KB와 JavaScript 약 30.5KB를 한 번 받습니다. 이후에는 immutable 캐시를 재사용하므로 같은 버전에서 개인 HTML에 반복 포함되지 않습니다. 실제 시간은 운영 DB·리전·기기에 따라 달라지므로 다음 버전에서 p50·p95와 실행 계획을 확인합니다.

## SQL 판정

- 필수 신규 SQL: 없음
- 기존 SQL 재실행: 없음
- 신규 필드·RPC: 없음
- 이번 변경은 기존 기본키와 settings key unique를 사용
- 추가 인덱스는 운영 실행 계획에서 병목이 확인될 때만 별도 검토

## 보호 범위

- 분석 클라이언트·렌더러 해시 동일
- 영수증 구현 변경 없음
- 카카오 그룹 규격 변경 없음
- 계정 로그인 비밀번호와 가계부 수명주기 분리 유지
- 삭제·연결해제 그룹 저장 0건 유지

## 패키지 정리

현재 ZIP에는 최신 배포 코드, 최신 검증, 적용 문서, 통합 이력, 카카오·UX 프로젝트 메모만 넣습니다. 과거 원본은 V22.8.9 이전 보관본에 남아 있으며 V22.8.10 업데이트에는 필요하지 않습니다.
