- [ ] `CRON_SECRET`을 운영 고유값으로 설정했다.
- [ ] 기존 기능 긴급 중지 변수를 사용 중이면 무료 개방 정책에 맞게 해제 여부를 확인했다.

## 배포

- [ ] `src/index.js`를 V22.6.6 파일로 교체했다.
- [ ] Worker를 저장·배포했다.
- [ ] `/health`가 `V22.6.6-FREE-FEATURE-ROLLOUT`을 반환한다.
- [ ] `/menu`에 무료 스마트 기능 링크가 표시되고 프리미엄 등급 문구가 없다.
- [ ] `/my/premium` 기존 링크가 무료 스마트 도구 화면으로 열린다.

## 기능 확인

- [ ] 테스트 영수증을 읽고 인식 결과를 수정한 뒤 1건만 저장된다.
- [ ] 같은 영수증을 다시 저장하면 중복 사유가 안내된다.
- [ ] 반복지출 후보는 확정 전 저장되지 않고 확정 후 규칙이 1개만 생긴다.
- [ ] 동일·비율·인원수·품목별 정산 결과가 각각 계산된다.
- [ ] 주간·월간 리포트 설정과 인쇄·PDF 저장이 동작한다.
- [ ] 카드 혜택 화면에 예상값·약관 확인 안내가 함께 보인다.
- [ ] 소비 카드 저장·보관함·공유가 선택 가계부 안에서 동작한다.

## 권한·격리

- [ ] owner/admin/member/viewer 계정으로 조회와 변경 권한을 각각 확인했다.
- [ ] viewer가 자동화 설정, 정산 완료, 카드 삭제를 할 수 없다.
- [ ] 한 계정이 참여한 여러 가계부의 거래·카드·리포트가 섞이지 않는다.
- [ ] 참여하지 않은 가계부 ID를 URL이나 POST로 보내면 거부된다.

## 자동 실행

- [ ] Cloudflare Cron `10 0 * * *` 또는 운영 시간표가 유지되어 있다.
- [ ] `/cron/recurring/apply?key=...` 수동 테스트가 중복 없이 완료된다.
- [ ] `/cron/reports/generate?key=...&force=1` 수동 테스트가 완료된다.
- [ ] 리포트가 자동 생성되지만 사용자 동의 없이 외부 메시지를 발송하지 않는다.

- [ ] Android Chrome 또는 카카오 인앱브라우저에서 같은 흐름을 확인했다.
- [ ] 하단 메뉴, 키보드, 파일 선택기, 긴 금액·가계부 이름이 겹치지 않는다.
