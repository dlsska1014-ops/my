# V22.7.1 자산 대시보드 검토·안정화 보고서

## 반영 범위

- 자산: 입출금·현금·페이·저축·투자·가상자산·연금·부동산·자동차·기타
- 부채: 대출 잔액을 자산과 분리하고 순자산에서 차감
- 결제수단: 신용·체크카드는 자산 잔액에 더하지 않음
- 수정: 이름, 종류, 금융사, 잔액/평가액, 메모, 합계 포함 여부
- 권한: 가계부 owner/admin만 추가·수정·삭제
- 개인정보: 12자리 이상 금융번호로 보이는 입력 차단

## 원본 검토에서 바로잡은 항목

1. 이전 V22.6 계열에서 분기된 전체 파일은 사용하지 않고 V22.7.0 인증·원자성 안정화본에 자산 기능만 병합했습니다.
2. 대출과 카드가 자산 합계에 포함되지 않도록 종류별 계산 규칙을 고정했습니다.
3. 동시에 저장할 때 JSON 목록을 덮어쓰는 위험을 줄이기 위해 DB advisory lock 기반 원자 변경 함수를 추가했습니다.
4. 가계부 삭제 시 `asset_history:{household_id}` 설정도 함께 정리하도록 보완했습니다.
5. 미완성 카드 실적·혜택 및 밈 기능은 기본 비노출 상태를 유지했습니다.

## 배포 순서

1. 기존 V22.7.0 DB라면 `schema_v22_7_1_asset_dashboard.sql`을 추가 실행합니다.
2. 신규 DB라면 기존 스키마 및 `schema_v22_7_0_auth_atomicity.sql` 적용 후 위 파일을 실행합니다.
3. Worker를 배포하고 `/health`의 버전이 `V22.7.1-ASSET-DASHBOARD-STABILITY`인지 확인합니다.
4. owner/admin과 일반 참여자 계정으로 `/payment-methods` 권한을 각각 확인합니다.
