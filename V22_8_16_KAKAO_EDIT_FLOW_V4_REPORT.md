# V22.8.16 카카오 수정 플로우 V4 보고서

## 배경 — 재현된 버그

실사용 로그에서 다음 무한루프가 재현되었습니다.

```
사용자: 수정 03번    → 봇: 메뉴 출력 (정상)
사용자: 지출자 변경  → 봇: 같은 메뉴 재출력 (버그)
사용자: 지출자       → 봇: 같은 메뉴 재출력 (버그)
```

원인 분석 결과 세 가지가 복합적으로 존재했습니다.

1. 파서 문제: 세션 후속 입력을 `extractKakaoEditPatch`가 `금액 13000원` 같은 정확한 정규식 패턴만 매칭하고 그 외 전부 실패 처리
2. 실패 처리 문제: 패치 추출 실패 시 무조건 같은 메뉴를 재출력하고 실패 횟수를 세지 않음 (`kakaoEditMenuText` 재출력 분기)
3. 안내 문구 문제: 봇이 예시로 안내한 `지출자 변경`을 파서가 인식하지 못함

## 조치 — 지침서 이행 내역

수정플로우-최종판-구현지침서의 작업 순서(8장)를 그대로 따랐습니다.

1. 기존 수정 플로우 경로 전부 제거: `handleKakaoDailyEditFlow`, `extractKakaoEditPatch`, `kakaoEditMenuText`, 스펜더 서브플로우, 구 세션 상태 (경로 이원화 금지)
2. 동봉 모듈 `edit-parser-v4.js`를 `src/index.js`에 그대로 이식 (파서·상태머신·selfTest·loopFuzzTest)
3. 세션 저장: 기존 `accountbook_settings` 키-값 방식 유지, 키는 방+사용자 조합, TTL 5분, 지침서 스키마 필드 유지. 텔레메트리 `unrecognized_inputs`는 선택 SQL로 문서화
4. 라우팅 순서 재구성(3장): ① `수정 NN번 [나머지]` ② `삭제 NN번` ③ 유효 세션 보유 시 메시지 전체를 `handleEditMessage`로 (세션 체크가 기록 저장·기타 명령보다 앞) ④ 기록 저장 ⑤ 그 외
5. 문구 교체(1-1장): 저장 응답 하단 안내, V4 메뉴, `결제자`→`지출자` 통일. 봇 안내 예시 전부 selfTest 케이스로 고정
6. config 주입(4장): 구성원·등록 결제수단·분류를 매 요청 실데이터로 구성해 값 옵션·검증 규칙으로 사용
7. 수락 기준 1~13 자동화 (`validation/validate-kakao-edit-flow.mjs`, 114개)
8. 검증 하네스에 selfTest·loopFuzzTest 포함 — 실패 시 배포 차단

## 무한루프 방지 장치 (5장)

- 실패 문구 회전(L6) + 실패 3회 종료 + 총 8턴 상한(I9)
- 전송 직전 가드(5-2): 직전 봇 메시지와 동일하면 변형 강제, 2회면 세션 강제 폐기, `duplicate_reply_guard` 텔레메트리
- reply 전송과 세션 upsert/삭제의 원자적 쌍(I6): `sendKakaoEditReplyV4` 단일 경로
- 이해 실패의 기본 분기가 메뉴 출력으로 이어지는 경로 0개(I8): 소스 불변식 검증으로 고정
- 카카오 반복가드가 세션 중 동일 입력(`ㅁㄴㅇㄹ` 연속 등)을 삼키지 않도록 세션 힌트로 우회
