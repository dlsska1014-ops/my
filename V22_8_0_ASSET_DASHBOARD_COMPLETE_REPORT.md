# V22.8.0 자산 대시보드 완성·안정화 보고서

## 결론

클로드가 설계한 자산 화면의 핵심 기능을 V22.7.3 안정판에 통합했습니다. 기존 카카오 WebView 저장, 가계부 전환·추가, 기록 수정·삭제, 지출자 처리와 권한 모델은 유지하고, 자산 데이터 저장과 월별 이력만 별도 원자성 RPC로 보강했습니다.

배포는 가능합니다. 단, Worker보다 먼저 `schema_v22_8_0_asset_dashboard_complete.sql`을 적용해야 자산 변경과 월별 순자산 이력이 하나의 DB 트랜잭션으로 저장됩니다.

## 반영한 기능

- 현재 순자산, 자산 합계, 부채 합계, 월 카드 사용액 요약
- 미결제 신용카드 사용액을 반영한 참고 순자산
- 자산 구성 도넛과 최근 6개월 순자산 추이
- 입출금·현금, 저축, 투자, 연금·보험, 부동산·실물, 기타 자산, 대출·부채, 카드·결제수단 그룹
- 잔액 빠른 수정과 이름·종류·발급사·메모 상세 수정
- 빠른 항목 추가와 거래에서 발견한 미등록 결제수단 연결
- 잔액 갱신 시점 표시와 오래된 잔액 안내
- 월별 자산 스냅샷 24개월 보관

## 안정화·보안

- 소유자·관리자만 자산 추가·수정·삭제 가능, 일반 참여자는 조회만 가능
- 가계부 ID를 서버 범위에서 다시 확인하고 다른 가계부 데이터 혼합 차단
- 가계부 단위 DB advisory lock과 행 잠금으로 동시 변경 직렬화
- 자산 목록과 월별 순자산 이력을 같은 트랜잭션에서 저장
- 공백 차이를 무시한 중복 이름 차단과 최대 200개 제한
- 금액은 0원 이상, 항목당 최대 9조원으로 제한
- 계좌번호·카드번호 전체로 보이는 12자리 이상 숫자 저장 차단
- 자산 페이지 `noindex,nofollow` 처리
- 화면에 내부 사용자 UUID, 원시 DB 오류, Secret 값 미노출
- 미완성 밈·소비 카드 및 카드 실적·혜택 화면은 기존 정책대로 숨김

## UI·UX 검토

- 390×844, 320×700 모바일 브라우저에서 18개 핵심 화면을 각각 렌더링
- 총 36화면·108캡처에서 가로 넘침 0건, 중복 ID 0건, 라벨 없는 컨트롤 0건
- 좁은 화면에서 필터와 잔액 수정 폼을 1열로 전환
- 넓은 결제수단 표는 페이지 전체를 밀지 않고 표 안에서만 좌우 스크롤
- 키보드 포커스 표시, 상태·오류용 `role`, 필터용 접근성 라벨 추가
- 공유 가계부에 자산 이름과 금액이 보인다는 안내와 민감정보 입력 금지 안내 추가

## 남은 운영 확인

이 패키지에서는 실제 운영 Supabase에 SQL을 실행하거나 Cloudflare Worker를 배포하지 않았습니다. 스테이징 또는 테스트 가계부에서 아래 항목을 최종 확인해야 합니다.

1. 자산 추가 후 같은 달 이력 생성
2. 잔액 빠른 수정 후 그래프·합계 갱신
3. 일반 참여자에게 변경 폼이 보이지 않음
4. 다른 가계부로 전환했을 때 자산 목록이 섞이지 않음
5. 카카오 인앱 브라우저에서 기록 수정과 가계부 전환·추가 정상 동작

## 롤백

문제가 있으면 Worker만 이전 V22.7.3 코드로 되돌릴 수 있습니다. 새 SQL은 기존 테이블 구조를 파괴하지 않고 새 RPC 함수만 추가하므로 그대로 남겨도 V22.7.3 동작에 영향을 주지 않습니다.
