# V22.6.9 보안·지출자·개인정보 핫픽스 보고서

## 결과

릴리스 식별자는 `V22.6.9-SECURITY-SPENDER-PRIVACY-HOTFIX`입니다. 사용자가 제보한 보안 설정 이탈, 지출자 수정·표시 누락, 내부 완료 코드 노출, 불필요한 시작가이드 문구, 고유 사용자 ID 노출을 수정했습니다. V22.6.8의 데이터 무결성 보호와 무료 기능 정책, 미완성 밈·카드 실적 숨김 정책은 유지합니다.

## 구현 내용

### 보안 설정 복귀

- 관리자 인증 폼에 검증된 내부 `return_to`를 보존합니다.
- 인증 실패 시에도 대상 경로를 잃지 않고 같은 인증 화면에 머뭅니다.
- 인증 성공 시 `/`가 아니라 요청한 보안 설정 경로로 복귀합니다.
- 관리자 비밀번호를 변경한 뒤 세션을 삭제하지 않고 새 세션을 발급해 `/settings?msg=password_updated`를 유지합니다.
- 개인 비밀번호 설정은 가계부 옵션·메뉴의 가계부 ID, 월, 관리 패널 앵커를 유지합니다.
- `//외부주소`, 줄바꿈 삽입, 로그인·로그아웃 순환 경로는 안전한 내부 기본 경로로 대체합니다.

### 기록 지출자

- 모바일 최근 기록에 수입은 `수입자`, 지출은 `지출자` 이름을 표시합니다.
- 수정/삭제 펼침 영역에 가계부 참여자 선택 항목을 추가했습니다.
- 소유자·관리자는 참여자 변경이 가능하고 일반 구성원은 자기 기록의 지출자를 임의 변경할 수 없습니다.
- 서버는 제출된 사용자 ID가 현재 가계부 참여자인지 다시 확인하며, 다른 가계부·외부 사용자 ID는 거부합니다.
- 빠른 입력의 기본 지출자는 현재 로그인 참여자로 설정하고 선택값이 없을 때의 미지정 기록을 줄였습니다.
- PC 기록 표와 수정 모달의 기존 지출자 표시·선택 기능도 함께 회귀 검증했습니다.

### 피드백 문구와 개인정보

- `member_alias_updated`를 `참여자 이름을 수정했습니다.`로 변환합니다.
- 시작가이드 하단의 앱 비교·참고 문구를 제거했습니다.
- 참여자 카드, 일반/고급 정산 표, 관리자 가계부 참여자 표에서 고유 사용자 ID·사용자키의 화면 표기를 제거했습니다.
- 참여자는 표시 이름과 권한으로 구분합니다.
- 인증된 수정 폼의 내부 식별값은 요청 처리를 위해 남지만 서버 권한 검증의 근거로 신뢰하지 않습니다.
- 운영 전용 카카오 진단의 사용자키는 마스킹하고, 개인정보 안내에는 처리되는 식별 데이터 종류를 투명하게 유지합니다.

## 무료·숨김 정책

- 영수증, 정기거래, 정산, 리포트, 분석, 예산, 백업·가져오기는 일반 무료 기능입니다.
- 소비 카드/밈과 카드 실적/혜택은 미완성 상태이므로 메뉴에 표시하지 않고 직접 URL도 404로 막습니다.
- 결제, 구독, 프리미엄 사용량 제한은 연결하지 않았습니다.

## 데이터베이스

V22.6.9는 Worker 코드 핫픽스이며 새 테이블·열·인덱스가 없습니다. V22.6.8 운영 무결성 마이그레이션을 이미 적용했다면 추가 SQL을 실행하지 않습니다.

## 다음 안정화 단계

1. 운영 배포 직후 실제 카카오 인앱브라우저에서 보안 설정 → 인증 → 설정 복귀를 확인합니다.
2. 두 명 이상 참여한 테스트 가계부에서 지출자를 바꾸고 기록·분석·정산의 이름 반영을 확인합니다.
3. 실제 iPhone/Android 키보드가 열린 상태에서 수정 펼침 영역과 저장 버튼 가림을 확인합니다.
4. 장기적으로는 모든 변경 POST에 CSRF 토큰, 중요 설정 변경 감사 로그, 관리자 세션 강제 만료 기능을 추가합니다.
5. 사용자 ID를 HTML 내부에도 전혀 싣지 않아야 하는 정책이 생기면 짧은 수명의 서명된 멤버 참조 토큰으로 교체합니다.
